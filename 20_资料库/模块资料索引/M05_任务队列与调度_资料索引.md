# M05 任务队列与调度资料索引

## 当前策略
M05 只围绕一条主线展开：先把 P01 的调度内核做扎实，再把它映射到 P03 这类 AI workload 平台。

当前正式入口是：

```text
学习地图 -> 章节教材 -> 复现推进表 -> E05 实验
```

当前主教材是：

```text
10_学习模块/M05_任务队列与调度/M05_任务队列与调度_章节教材.md
```

`M05_任务队列与调度_适配教材.md` 只作为旧版总览导读保留；真正逐章学习、手写代码、跑实验和复盘时，以章节教材和复现推进表为准。

资料优先级按“能直接帮助你写出实验和结论”的顺序排：

1. 先看调度最小概念和 Python 数据结构。
2. 再看高校课程里关于 CPU scheduling / resource allocation 的基础章节。
3. 然后看指标、尾延迟和高峰负载资料。
4. 最后再看 Kubernetes / Kueue / Volcano 这类真实系统怎么组织队列与配额。

## 转化要求

- 每读完一条资料，至少要落成一个实验步骤、知识卡片或 README 结论。
- 资料必须为“你能写出代码和结论”服务，不接受只收藏不转化。

## 建议学习顺序

1. 先读章节教材第 0-1 章，明确学习方式和调度直觉。
2. 完成 [[40_实验练习/E05_调度实验/E05-R0-R1 学习边界与最小模型复现|E05-R0-R1]] 的 R0 部分，用自己的话写清 M05 / P01 关系和当前不做什么。
3. 再读 OSTEP Scheduling 的 FIFO、SJF、响应时间、公平性相关部分。
4. 按章节教材第 2 章完成 E05-R0-R1 的 R1 部分，手写 Task / Worker。
5. 按章节教材第 3 章手写 FIFO，并完成 E05-01。
6. 读 Python `heapq` 的 priority queue 最小用法，再完成 Priority 和 SJF，对应章节教材第 4-5 章与 E05-02。
7. 读 Prometheus Histograms 里 histogram、summary、P95/P99 的概念，再学第 6 章指标。
8. 按章节教材第 7 章完成 E05-03，高峰负载只解释队列积压、尾延迟和策略取舍。
9. 第 8 章再进入 Cost-aware，对照 E05-04，暂时只做成本分数和权重直觉。
10. 第 9 章使用 E05-04 的 `task_type_breakdown` 做分组分析，重点解释谁被改善、谁被牺牲。
11. 第 10 章使用 E05-04 的 aging 结果，理解最大等待保护如何降低极端等待以及它的代价。
12. 第 11 章使用 Worker 数量实验，理解增加 worker 如何降低 P95/P99，以及为什么 utilization 会下降。
13. 第 12 章整理项目总结与复盘，把已经亲手复现的模型、策略、指标和实验结论写成自己的 README 草稿、实验报告草稿和简历表达草稿。
14. Kueue / Volcano / Kubernetes Scheduler 先作为概念映射资料，不读源码，不提前扩展真实平台。
