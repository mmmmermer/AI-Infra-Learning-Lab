# M06 数据库缓存与异步任务资料索引

## 当前策略

M06 资料服务于 P03 的任务持久化和异步执行，不做数据库大全，也不做中间件横向选型。
当前主线是 PostgreSQL task + outbox、dispatcher、Redis Streams consumer group、ACK/reclaim 和
owner/version/lease fencing；RQ/Celery 仅用于对比。

第一轮资料使用顺序：

```text
M06 学习地图
-> M06 适配教材
-> 本资料索引按需查官方资料
-> E06 实验
-> P03 持久化异步闭环
```

|---|---|---|---|---|---|---|

| PostgreSQL Transaction Isolation | https://www.postgresql.org/docs/current/transaction-iso.html | 官方文档 | 必读 | 并发正确性 | 区分隔离级别、可见性与 serialization failure | 教材第 2A 章；PostgreSQL 迁移边界 |
| PostgreSQL Explicit Locking / Deadlocks | https://www.postgresql.org/docs/current/explicit-locking.html#LOCKING-DEADLOCKS | 官方文档 | 必读 | 锁与恢复 | 理解行锁顺序、deadlock victim、完整事务 rollback/retry | 教材第 2A 章；可选隔离环境 demo |

| Google SRE: Addressing Cascading Failures | https://sre.google/sre-book/addressing-cascading-failures/ | 权威工程资料 | 必读 | 重试风暴 | 学有限重试、指数退避、jitter、deadline 和 overload 边界 | 教材第 6A/7 章；retry-storm lab |

## 和教材章节的对应关系

| 教材章节 | 优先资料 | 使用方式 |
|---|---|---|
| 第 2 章：SQL 和任务状态表 | PostgreSQL Tutorial | 查 SQL、表、插入、查询、更新 |
| 第 2A 章：数据库正确性、查询计划与并发 | PostgreSQL Constraints/EXPLAIN/Isolation/Locking + SQLite Transactions | 把 schema、访问路径和并发异常转成负向测试与恢复证据 |

| 第 6 章：异步任务队列 | Redis Streams | 查 XADD、consumer group、ACK 和 pending |
| 第 6A 章：分布式失效语义 | Patterns of Distributed Systems + Google SRE | 按 C0-C4 推导丢失/重复/恢复，并运行 retry-storm 对照 |
| 第 7 章：失败重试和幂等 | Redis Streams + P03 reference | 查 reclaim、CAS/fencing 和终态 ACK |
| 第 8 章：任务结果、日志和 metrics | PostgreSQL Tutorial | 转化成可查询字段和聚合 |
