# M11 科研方法与实验设计学习地图

## 怎么读这个模块

把 M11 当成“如何把 P01/P03 的实验写成可信研究训练”的写作课，不要当成泛泛科研方法清单。

阅读主线是：工程现象提出问题，写假设，设置 baseline，控制变量，记录指标，设计 Rubric，做人工作业式复核，分析失败，最后形成 RQ01 实验报告和导师沟通材料。

没有真实实验数据时，只写问题、设计和假设；不要为了让材料好看而写结论。

## 在总路线中的位置

M11 负责把工程项目转成科研训练能力。

它不单独讲泛泛科研方法，而是围绕 P01/P03、M05 调度实验、M08 监控压测和 RQ01 研究问题，训练你提出问题、设置 baseline、控制变量、设计指标、分析结果和写实验报告。

主线是：

```text
P01 / P03 工程现象
-> M05 调度策略实验
-> M08 压测与指标
-> RQ01 研究问题
-> 实验设计
-> 实验报告
-> 导师沟通材料
```

## 要解决的问题

- 如何从工程现象抽象出研究问题？
- 如何设计 baseline？
- 如何控制变量并解释结果？
- 如何把 P95/P99、worker utilization、queue length 写成实验分析？
- 如何把 RAG/Agent 输出质量写成 Rubric 和人工评测记录？
- 如何避免没有数据时虚构结论？
- 如何形成导师联系材料？

## 学习目标

- [ ] 能写出 RQ01 的研究问题和假设。
- [ ] 能设计 FIFO / Priority / SJF / Cost-aware baseline。
- [ ] 能区分自变量、因变量、控制变量。
- [ ] 能选择 average、P95/P99、throughput、queue length、worker utilization、error rate 等指标。
- [ ] 能把 M05 实验记录转成科研实验记录。
- [ ] 能把 M08 压测数据转成报告证据。
- [ ] 能设计一个小型 Rubric，记录人工评测、分歧和规则迭代。
- [ ] 能写失败分析、局限和下一步计划。
- [ ] 能形成诚实的导师沟通版本。

## 核心内容

| 内容 | 学到什么程度 | 对应出口 |
|---|---|---|
| 研究问题 | 能把工程现象转成可验证问题 | RQ01 |
| 假设 | 能写可被支持或否定的判断 | RQ01 实验设计 |
| baseline | 能解释 FIFO/Random/Priority/SJF | M05 |
| 变量控制 | 能固定任务流、worker 数、随机种子 | 实验设计 |
| 指标 | 能解释 P95/P99、utilization、queue length | M08 |
| Rubric | 能拆评分维度、写标注规则和验收标准 | P03 EvaluationTask |
| 人工评测 | 能记录 review_status、reviewer_note 和分歧原因 | 60_科研训练 / P03 |
| 实验记录 | 能记录设置、结果、异常、局限 | 60_科研训练 |
| 失败分析 | 能把失败转成下一轮实验 | RQ01 |
| 报告结构 | 能写摘要、方法、结果、讨论、局限 | 实验报告 |
| 导师材料 | 能说明问题、进展、局限、请教点 | 写作草稿 |

## 推荐学习顺序

1. 读 [[M11_科研方法与实验设计_适配教材|M11 适配教材]] 第 1-2 章，写 RQ01 研究问题和假设。
2. 读第 3-4 章，设计 baseline 和变量表。
3. 读第 5 章，对齐 M05/M08 指标。
4. 读第 6-7 章，把 P01/M05 实验记录转成科研记录和失败分析。
5. 读第 8 章，写 RQ01 初版实验报告结构。
6. 读第 9 章，整理导师沟通材料。

## 检查标准

- [ ] 能写出一个清晰研究问题。
- [ ] 能设计 baseline。
- [ ] 能做对比实验。
- [ ] 能写实验记录和报告结构。
- [ ] 能写出一份小型 Rubric，并说明评分维度为什么合理。
- [ ] 能区分“人工观察”“模型判断”“实验结论”。
- [ ] 能解释 P95/P99 和 worker utilization。
- [ ] 能说明实验局限和下一步。
- [ ] 能生成导师联系材料草稿。
