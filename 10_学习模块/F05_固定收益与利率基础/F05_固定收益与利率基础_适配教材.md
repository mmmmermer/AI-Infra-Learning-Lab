## 本模块最值得理解的难点

F05 的难点不是折现公式，而是利率、价格、收益率、现金流之间的关系很容易混。

| 难点 | 为什么难 | 工程痛点 | 后续影响 |
|---|---|---|---|
| coupon 和 yield 不是一回事 | coupon 是合约现金流，yield 是价格反推的收益率 | 字段名相似但含义不同 | 债券定价记录会错 |
| 利率和价格反向 | 折现率上升，现值下降 | 初学者容易只记结论不懂原因 | duration 学不稳 |
| YTM 是数值求解 | 不是直接给出的简单字段 | 需要记录求解方法和误差 | fixed_income_task 需要 error_type |
| duration 只是近似 | 利率变化大时线性近似不够 | 只用 duration 会偏离完整重估 | 需要 convexity，并保留完整重估基线 |
| 利率数据有频率和口径 | FRED 不同 series 含义不同 | 数据源不清会污染实验 | Q01/Q03 解释变弱 |

学习 F05 的核心习惯是：

```text
每个固定收益计算都必须记录现金流、贴现率、频率、数据来源、参数口径和非投资建议边界。
```

## 一条贯通学习线

F05 不从复杂利率模型开始，而从一条现金流时间线开始：

```text
债券条款
-> 现金流表
-> 折现现值
-> 由价格反解 yield
-> 利率敏感性
-> fixed_income_task 记录
```

这条线的重点是：固定收益不是“公式很多”，而是每个数字都必须能追溯到现金流、时间和利率口径。只要现金流表错了，后面的 YTM、duration、convexity 都会错。

## 可迁移的原则

1. 债券先看现金流，再看公式。coupon、principal、maturity、frequency 决定现金流表，折现只是把现金流映射到今天。
2. yield 不是 coupon。coupon 是合约约定，YTM 是由价格反推的收益率；两者混淆会让 fixed_income_task 的输入输出失真。
3. 利率数据必须有 series 口径。不同期限、频率和来源的利率不能随意替换，否则折现结果和敏感性分析都不可复现。

## 踩坑现场

> 你用一个年化利率直接折现半年付息债券，结果看似合理，但频率错了。正确做法是先把现金流日期、付息频率、折现频率和利率来源写清楚，再做现值或 YTM 计算。

## 第 1 章：债券是一组现金流

### 1.1 本章解决什么问题

债券最简单的理解方式是：未来按约定时间收到一组现金流。

例如一只面值 100、年 coupon 5%、两年到期的债券，可以简化成：

| 年份 | 现金流 |
|---|---:|
| 1 | 5 |
| 2 | 105 |

第二年包含 coupon 5 和本金 100。

把它画成时间线：

```text
t=0          t=1          t=2
今天          收 5          收 105
价格?         票息          票息 + 本金
```

固定收益学习的第一步，就是能从条款写出这条时间线。不要急着算 YTM；如果现金流时间线写不出来，求解出来的收益率没有意义。

### 1.2 基本术语

| 术语 | 含义 |
|---|---|
| face value | 到期归还的本金 |
| coupon rate | 票息率 |
| coupon payment | 每期票息现金流 |
| maturity | 到期时间 |
| yield | 用来折现现金流的收益率口径 |
| price | 未来现金流折现后的现值 |

### 1.3 最小代码

```python
def coupon_bond_cashflows(
    face_value=100.0,
    coupon_rate=0.05,
    maturity_years=2.0,
    payments_per_year=2,
):
    total_periods = maturity_years * payments_per_year
    if not float(total_periods).is_integer():
        raise ValueError("maturity must align with payment frequency")

    period_count = int(total_periods)
    coupon = face_value * coupon_rate / payments_per_year
    cashflows = []
    for period in range(1, period_count + 1):
        amount = coupon + (face_value if period == period_count else 0.0)
        cashflows.append((period, period / payments_per_year, amount))
    return cashflows
```

这里的 `coupon_rate` 是名义年票息率，`payments_per_year=m` 决定每期票息为 `face_value * coupon_rate / m`。现金流同时保存期数 `period` 和年时间 `time_years`，因为贴现指数用期数，Macaulay duration 的单位用年。真实债券还会涉及结算日、计息天数、应计利息和交易日历；第一轮必须把这里标成简化现金流，而不能把 `m=2` 的半年付息误写成年付息。

### 1.4 从条款到字段

固定收益 task 的输入字段不是随便设计的，它们来自债券条款：

| 条款 | 字段 | 影响 |
|---|---|---|
| 面值 | `face_value` | 最后一期本金 |
| 票息率 | `coupon_rate` | 每期票息 |
| 到期年限 | `maturity` | 现金流期数 |
| 付息频率 | `payment_frequency` | 每期现金流和时间单位 |
| 收益率/折现率 | `yield_rate` | 现值计算 |
| 复利口径 | `compounding` | 决定年收益率如何换成每期利率 |
| 结算和计息 | `settlement_assumption` / `day_count` | 决定应计利息和现金流时间 |

如果 `payment_frequency` 缺失，就不能判断 coupon 是一年一次还是半年一次。这个字段不是细节，是计算定义的一部分。

### 1.5 常见错误

- 把 coupon rate 当成 yield。
- 忘记最后一期归还本金。
- 不记录付息频率。
- 把教学债券现金流写成真实投资建议。

### 1.6 本章来源

- [U.S. TreasuryDirect：Understanding Pricing and Interest Rates](https://www.treasurydirect.gov/marketable-securities/understanding-pricing/)：
  核对票面金额、票息、到期现金流与价格/收益率的关系；本章现金流仍是付息日结算的简化 fixture。
- [MIT OpenCourseWare 15.401 Lecture Notes](https://ocw.mit.edu/courses/15-401-finance-theory-i-fall-2008/pages/lecture-notes/)：
  Fixed-Income Securities 部分用于核对从合约现金流进入现值的学习顺序。

## 第 2 章：折现和债券现值

### 2.1 本章解决什么问题

未来现金流今天值多少钱？这就是现值问题。

本教材采用名义年收益率 `y`、每年复利 `m` 次。第 `k` 期现金流的现值和 dirty price 为：

```text
periodic_yield = y / m
PV_k = CF_k / (1 + y / m)^k
dirty_price = sum(PV_k)
```

因此 `y=0.04, m=2` 表示每半年使用 `0.02` 的周期收益率，并不等于有效年收益率恰好为 4%；对应有效年收益率是 `(1 + 0.04 / 2)^2 - 1`。教材后面的 YTM、duration 和 convexity 必须沿用同一个 `m`，不能一处半年复利、另一处按年复利。

### 2.2 最小代码

```python
def present_value(cashflows, annual_yield, payments_per_year=2):
    if payments_per_year <= 0:
        raise ValueError("payments_per_year must be positive")
    if annual_yield <= -payments_per_year:
        raise ValueError("periodic discount factor must be positive")

    periodic_yield = annual_yield / payments_per_year
    return sum(
        amount / ((1 + periodic_yield) ** period)
        for period, _time_years, amount in cashflows
    )

cashflows = coupon_bond_cashflows(
    face_value=100,
    coupon_rate=0.05,
    maturity_years=2,
    payments_per_year=2,
)
dirty_price = present_value(cashflows, annual_yield=0.04, payments_per_year=2)
```

### 2.3 金融直觉

贴现率越高，未来现金流今天越不值钱。因此在其他条件不变时，利率上升通常会让债券价格下降。

一个手算观察：

| 现金流 | 折现率 | 现值直觉 |
|---|---:|---|
| 第 1 年收 5，第 2 年收 105 | 4% | 未来现金流按较低利率折回今天 |
| 第 1 年收 5，第 2 年收 105 | 8% | 同样现金流折回今天更低 |

所以“债券价格和利率反向”不是要硬背，而是因为同样的未来钱，在更高折现率下今天值更少。

### 2.4 Clean price、dirty price 和应计利息的最小边界

上面的现金流折现结果是 dirty price，也叫 full price。市场报价常用 clean price，两者满足：

```text
dirty_price = clean_price + accrued_interest
```

第一轮实验明确假设“结算日恰好在付息日”，因此 `accrued_interest=0`，此时 `clean_price=dirty_price`。这个相等关系只属于该教学 fixture，不能推广到两个付息日之间的真实结算。非付息日结算至少需要 `settlement_date`、上/下一个付息日和 `day_count`；当前 `finance_reference` 尚未实现这条生产级路径，应显式返回“超出当前实现边界”，不能默认为 0。

### 2.5 工程痛点

必须记录：

```text
cashflow_schedule
annual_yield
payments_per_year
compounding_assumption
settlement_assumption
day_count
dirty_price
clean_price
accrued_interest
```

付息频率、复利频率和现金流时间单位必须一致。教材只实现离散复利；如果数据源给出有效年收益率、连续复利率或不同 day-count 的收益率，必须先做有来源的口径转换，不能只改字段名。

### 2.6 小练习

用同一组现金流分别代入 `annual_yield=0.04` 和 `annual_yield=0.08`，保持 `payments_per_year` 不变并记录：

| annual_yield | dirty_price | explanation |
|---:|---:|---|
| 0.04 | 代码输出 | 较低折现率 |
| 0.08 | 代码输出 | 较高折现率 |

验收标准：能用自己的话解释为什么第二行价格更低，并说明这不是投资建议，只是折现机制演示。

## 第 3 章：YTM 的直觉和数值求解

### 3.1 本章目标与前置

前置：能使用第 1 章现金流表和第 2 章 `present_value()`，并理解目标价格采用 dirty price、
结算日在付息日的简化口径。

YTM 可以理解为：给定债券价格和未来现金流，反推出让现值等于价格的收益率。

```text
price = sum(CF_k / (1 + y / m)^k)
```

这里的 `y` 就是要求解的到期收益率。

换句话说，现值计算是：

```text
给 yield -> 算 price
```

YTM 求解是反过来：

```text
给 price -> 找 yield
```

它不是债券条款直接给出的 coupon，也不是市场观点本身，而是一个由价格和现金流共同决定的数值解。

### 3.2 最小二分求解

<!-- textbook-code: role=fragment env=python-3.13 network=off -->
```python
def solve_ytm(
    cashflows,
    target_price,
    payments_per_year=2,
    low=-0.20,
    high=1.00,
    tol=1e-10,
    max_iter=200,
):
    if target_price <= 0:
        raise ValueError("target_price must be positive")
    if low >= high:
        raise ValueError("low must be below high")

    def objective(yield_rate):
        return (
            present_value(cashflows, yield_rate, payments_per_year)
            - target_price
        )

    f_low = objective(low)
    f_high = objective(high)
    if f_low * f_high > 0:
        raise ValueError("YTM root is not bracketed by [low, high]")

    for _ in range(max_iter):
        mid = (low + high) / 2
        f_mid = objective(mid)
        if abs(f_mid) < tol or abs(high - low) < tol:
            return mid
        if f_low * f_mid <= 0:
            high = mid
        else:
            low = mid
            f_low = f_mid

    raise RuntimeError("YTM solver did not converge")
```

该函数依赖第 2 章的 `present_value()`，因此是可组合片段，不是独立程序。

### 3.3 Worked example：观察二分区间如何收缩

固定年付息现金流 `[(1, 1.0, 5), (2, 2.0, 105)]`、`payments_per_year=1` 和目标
dirty price `101`。定义残差 `f(y)=PV(y)-101`：

| 候选年 YTM | `f(y)` | 判断 |
|---:|---:|---|
| 4.0000% | 0.886095 | 现值高于目标，YTM 还要提高 |
| 5.0000% | -1.000000 | 现值低于目标，根位于 4%-5% |
| 4.5000% | -0.063666 | 上界继续下移 |
| 4.4500% | 0.030702 | 下界继续上移 |
| 4.4660% | 0.000489 | 已接近根，但仍继续到容差满足 |

<!-- textbook-code: role=fragment env=python-3.13 network=off -->
```python
cashflows = [(1, 1.0, 5.0), (2, 2.0, 105.0)]
target_price = 101.0
ytm = solve_ytm(
    cashflows,
    target_price,
    payments_per_year=1,
    low=0.04,
    high=0.05,
    tol=1e-10,
)
residual = present_value(cashflows, ytm, 1) - target_price
print(f"ytm={ytm:.12f} residual={residual:.3e}")
```

<!-- textbook-code: role=output -->
```text
ytm=0.044662591629 residual=-6.460e-09
```

点估计必须和残差一起保存。`4.466259%` 只是在当前现金流、价格、复利频率和结算假设下的
数值解，不是持有期实现收益保证。

### 3.4 工程痛点

YTM 求解可能失败或不稳定。要记录：

```text
target_price
solver_method
tolerance
max_iter
converged
```

数值求解的核心是“逐步逼近”。二分法不是金融魔法，它只是不断试一个 yield，看算出的 price 是高于还是低于目标 price，再缩小搜索区间。

如果目标价格或现金流不合理，求解可能不收敛。P03 里不能只返回空值，应该记录 `ytm_not_converged`。

### 3.5 常见错误

- 把 coupon rate 当 YTM。
- 不记录求解方法。
- 不记录 tol 和 max_iter。
- price/cashflow 单位不一致。

### 3.6 独立练习

保持同一现金流和 `payments_per_year=1`，把目标 dirty price 改为 `98`：

| 记录项 | 要求 |
|---|---|
| `target_price` | 98 |
| `solver_method` | bisection |
| `tol` | 1e-6 |
| `max_iter` | 100 |
| `converged` | 代码运行后填写 |
| `ytm` | 代码运行后填写 |

验收标准：`converged=True`，YTM 约为 `0.0609228474`，重算价格与 98 的绝对残差小于
`1e-6`；并能说明为什么目标价格由 101 降到 98 时，当前现金流下的 YTM 上升。

### 3.7 验收与失败恢复

除第 3.6 节的数值与残差外，还要验证失败路径：把区间故意设为 `[0.20, 0.30]` 时，
根不在区间内，必须得到
`YTM root is not bracketed`，而不是返回区间端点。恢复方法是先计算两端残差并扩大到真正
异号的区间；回归检查同时覆盖正常收敛、未括根和非法目标价格。

YTM 求解的完成证据不是“返回一个数字”，而是参数、括根区间、收敛状态和价格残差。下一章

## 第 4 章：久期、凸性和价格近似

### 4.1 本章目标与前置

前置：能用第 2 章完整重估同一组现金流，并理解第 3 章 YTM 是当前 dirty price 对应的
单一离散复利收益率口径。

duration 和 convexity 用来描述“在当前现金流和收益率口径不变时”，债券 dirty price 对同一到期收益率 `y` 平行冲击的局部敏感性。它们不是对所有利率、信用利差和现金流变化的统一预测。

### 4.2 Macaulay duration 和 modified duration

令 `PV_k` 是第 `k` 期现金流现值、`t_k=k/m` 是以年计的现金流时间、`P=sum(PV_k)`：

```text
Macaulay duration = sum(t_k * PV_k) / P
modified duration = Macaulay duration / (1 + y / m)
```

- Macaulay duration 是现值权重下的平均收款时间，单位是年。
- Modified duration 才直接进入价格对收益率的局部一阶近似。
- 两者都依赖现金流、`y`、`m` 和结算假设；不能只写一个没有单位和口径的 `duration`。

对小的年收益率变化 `Delta y`：

```text
Delta P / P approximately - modified_duration * Delta y
```

例如 1 bp 必须写成 `0.0001`，而不是 `1` 或 `0.01`。

### 4.3 解析 convexity 和二阶近似

在本教材的离散复利口径下，固定现金流债券的解析 convexity 为：

```text
convexity =
  sum(CF_k * k * (k + 1) / (1 + y / m)^(k + 2))
  / (P * m^2)
```

加入凸性后二阶近似为：

```text
Delta P / P approximately
  - modified_duration * Delta y
  + 0.5 * convexity * (Delta y)^2
```

正凸性的普通固定票息债券中，二阶项能修正只用直线近似造成的偏差。但“加了 convexity”仍然是局部近似；冲击较大、现金流带权利条款或收益率曲线非平行移动时，应完整重估。

### 4.4 有限差分交叉验证

解析公式不能只靠抄写。令 `P_0=P(y)`、`P_-=P(y-h)`、`P_+=P(y+h)`，用足够小且保持折现因子有效的 `h` 做中心差分：

```text
effective_duration = (P_- - P_+) / (2 * P_0 * h)
effective_convexity = (P_- + P_+ - 2 * P_0) / (P_0 * h^2)
```

小 `h` 下，effective duration 应接近 modified duration，effective convexity 应接近解析 convexity。差异明显时，优先检查 `m`、收益率单位、现金流时间和冲击大小。

教材不重复粘贴整份答案实现；公式的可执行参考和测试位于：

- [[40_实验练习/GF10_金融工程全阶段实验候选/finance_reference/finance_reference/fixed_income.py|finance_reference fixed_income.py]]
- [[40_实验练习/GF10_金融工程全阶段实验候选/finance_reference/tests/test_fixed_income.py|finance_reference 固定收益测试]]
- [[40_实验练习/GF10_金融工程全阶段实验候选/GF05-02 久期和凸性实验|GF05-02 久期和凸性实验]]

### 4.5 Worked example：近似和完整重估

下面是教材级校验 fixture，不是市场结论：面值 100、票息率 5%、5 年到期、半年付息、名义年 YTM 4%，结算日在付息日。基准 dirty price 为 `104.491293`，Macaulay duration 为 `4.498904` 年，modified duration 为 `4.410690`，convexity 为 `22.923146`。

| 收益率上移 | 完整重估价格 | 完整重估 `Delta P/P` | 只用 duration | duration + convexity |
|---:|---:|---:|---:|---:|
| 1 bp (`0.0001`) | 104.445217 | -0.044095% | -0.044107% | -0.044095% |
| 200 bp (`0.02`) | 95.734899 | -8.380023% | -8.821380% | -8.362917% |

这张表给出两个检查点：小冲击时两种近似都接近完整重估；冲击变大时，一阶误差明显，二阶近似虽改善但仍不能替代完整重估。GF05-02 还要由学习者亲手计算下移冲击、有限差分指标和误差，不得把本表当作完整实验记录。

### 4.6 怎么选择方法

| 目的 | 方法 | 当前边界 |
|---|---|---|
| 解释现值加权收款时间 | Macaulay duration | 不是价格变化百分比 |
| 快速估计很小的平行收益率冲击 | Modified duration | 一阶局部近似 |
| 修正曲率 | Duration + convexity | 二阶局部近似 |
| 给教学债券生成验收真值 | 完整现金流重估 | 仍依赖同一现金流和收益率假设 |

### 4.7 工程痛点

duration/convexity 容易被写成绝对预测。它们只是给定现金流和利率假设下的敏感性指标。

记录：

```text
annual_yield
payments_per_year
compounding
settlement_assumption
price_type
derivative_shock_h
scenario_yield_shock
price_base
price_up
price_down
macaulay_duration_years
modified_duration
analytic_convexity
effective_duration
effective_convexity
approximation_error
```

`derivative_shock_h` 是有限差分求局部导数的小步长，`scenario_yield_shock` 是业务情景冲击；两者不能混成同一个字段。另一个常见错误是把收益率曲线某一点的变化当成所有现金流都按同一个 YTM 平行变化，本教材的单一 YTM 模型必须明确写出这一限制。

## 第 5 章：利率数据和 fixed_income_task

<!-- textbook-content: type=design-note -->

### 5.1 FRED 数据口径

[Federal Reserve Economic Data（FRED）](https://fred.stlouisfed.org/) 这类公开数据源
可以用于学习利率序列。FRED 是圣路易斯联储维护的经济数据聚合服务，不是某一种
“无风险利率”的名称；具体含义仍由 series 页面中的来源机构、期限、单位和注释决定。
第一轮只记录：

```text
series_id
source_url
frequency
download_time
units
date_range
```

不要把公开利率序列直接写成投资建议。

常见的学习误区是看到一个利率 series 就直接拿来折现。更稳的做法是先问：

| 问题 | 例子 |
|---|---|
| 期限是什么？ | 1 年、2 年、10 年国债收益率不是同一个概念 |
| 单位是什么？ | 百分数、年化、日频观测 |
| 频率是什么？ | 日、周、月，是否和债券现金流频率匹配 |
| 是否有修订或缺失？ | 公开宏观数据可能存在缺失、假日和发布时间差异 |

第一轮只做数据口径记录，不做复杂曲线构建。

