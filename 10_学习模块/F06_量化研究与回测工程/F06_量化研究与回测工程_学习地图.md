# F06 量化研究与回测工程学习地图

## 怎么读这个模块

F06 是金融工程第二主线里和计算机、工程、AI 结合最紧的模块之一。

它不是“找赚钱策略”，而是学习一套可复现研究流程：

```text
数据
-> 信号
-> 仓位
-> 收益
-> 成本
-> 风险
-> 报告
-> 失败分析
```

第一轮只做最小回测，不做实盘交易、不做高频、不做策略收益宣传。重点是工程可复现、数据不泄漏、记录清楚、指标可解释。

## 要解决的问题

- 量化研究里的 signal、position、return 分别是什么？
- 为什么回测很容易因为未来数据、幸存者偏差、交易成本缺失而失真？
- bid/ask、spread、slippage、成交假设为什么会影响回测结果？
- 一个最小回测脚本应该有哪些模块？
- 如何记录 baseline、变量、控制变量和失败原因？
- 回测任务为什么适合异步执行和指标监控？

## 学习目标

- [ ] 能用价格序列构造一个最小信号。
- [ ] 能从 signal 转成 position，再计算 strategy return。
- [ ] 能加入 transaction cost，观察结果变化。
- [ ] 能解释 bid/ask spread、slippage、turnover 和成交假设。
- [ ] 能说明 look-ahead bias、survivorship bias、data snooping 的含义。
- [ ] 能写一份回测记录，而不是只看收益曲线。
- [ ] 能把回测任务抽象成 P03 的 `backtest_task`。

## 第一轮学习产物

1. 一个最小回测脚本：`load_data -> make_signal -> make_position -> calculate_returns -> report_metrics`。
2. 一份参数记录：窗口长度、交易成本、时间范围、数据来源。
3. 一个成本对比表：无成本 vs 有成本。
4. 一份失败分析：哪里可能泄漏未来数据，哪里可能过拟合。
5. 一个 P03 `backtest_task` 草图。
6. 一个市场摩擦说明：记录 spread、slippage、turnover、rebalancing_frequency 和成交假设。

## 检查标准

- [ ] 能解释 signal 和 position 的区别。
- [ ] 能说明为什么不能用未来价格生成当前信号。
- [ ] 能加入交易成本，并说明它如何影响结果。
- [ ] 能说明交易成本、滑点和换手率为什么会改变回测结论。
- [ ] 能列出至少四种回测偏差检查项。
- [ ] 能写出回测记录，而不是只贴收益曲线。
- [ ] 能说明回测结果不构成投资建议。
