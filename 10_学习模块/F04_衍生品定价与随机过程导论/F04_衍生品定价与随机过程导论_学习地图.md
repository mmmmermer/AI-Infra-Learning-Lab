# F04 衍生品定价与随机过程导论学习地图

## 怎么读这个模块

把 F04 当成金融工程第二主线的“定价模型入口”，不要一上来啃随机微积分大书。

第一轮只抓三件事：

```text
衍生品是什么
-> payoff 和定价输入是什么
-> 如何用最小代码实现 Black-Scholes / Monte Carlo 定价
```

本模块的目标不是让你立刻做复杂期权交易，而是让你理解金融工程里“模型、假设、参数、数值方法、误差和计算成本”之间的关系。它和 AI Infra 的连接点在于：Monte Carlo pricing 这类任务天然可以变成 P03 里的 `pricing_task`，进入队列、记录运行时间、失败类型和结果。

## 在总路线中的位置

```text
F00 金融市场与资产基础
-> F01 概率统计与数学基础
-> F04 衍生品定价与随机过程导论
-> Q03 期权定价与 Monte Carlo 实验
-> F08 金融工程任务与 AI Workload 接口
-> P03 pricing_task
```

它和 M05/M08 的关系：

```text
Monte Carlo 定价任务
-> simulation_count / runtime_ms / queue_wait_ms
-> 调度策略和资源消耗
-> P95/P99 与 worker utilization
```

## 要解决的问题

- forward、futures、options 分别是什么？
- payoff diagram 为什么比背定义更重要？
- Black-Scholes 模型的输入和输出是什么？
- Monte Carlo pricing 为什么适合做成计算型 workload？
- 定价模型里的参数、假设和误差应该如何记录？
- 哪些内容第一轮必须会，哪些随机过程内容可以暂缓？

## 学习目标

- [ ] 能解释 call、put、strike、maturity、underlying、volatility、risk-free rate。
- [ ] 能画出欧式 call/put 的 payoff 形状。
- [ ] 能说清 Black-Scholes 的输入、输出和主要假设。
- [ ] 能用 Python 写一个最小 Black-Scholes 定价函数。
- [ ] 能用 Monte Carlo 模拟估计欧式期权价格，并记录随机种子和模拟次数。
- [ ] 能说明模型输出不是投资建议，也不是真实交易报价。
- [ ] 能把定价任务抽象为 P03 的 `pricing_task` 草图。

## 第一轮学习产物

1. 一张 payoff 图：call 和 put 至少各一张。
2. 一个 Black-Scholes 函数：输入 `S, K, r, sigma, T, option_type`，输出 price。
3. 一个 Monte Carlo 定价脚本：记录 `seed`、`simulation_count`、`runtime_ms`。
4. 一个 P03 `pricing_task` 草图：写清 input、output、metrics、error_type。
