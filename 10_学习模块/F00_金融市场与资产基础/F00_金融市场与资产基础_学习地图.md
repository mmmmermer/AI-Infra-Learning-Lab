# F00 金融市场与资产基础学习地图

## 怎么读这个模块

把 F00 当成金融工程支线的“语言和场景入口”，不要当成投资学百科。

阅读主线是：先知道股票、债券、基金、衍生品和宏观数据分别是什么，再理解价格、收益率、利率、风险、流动性、期限和信用这些基础口径，最后知道这些数据未来如何进入 Q01 金融数据实验、M12 金融文档 RAG 和 P03 金融 workload。

第一轮只建立金融市场直觉和数据口径，不做投资建议，不分析买卖机会。

正式学习时先读 [[10_学习模块/F00_金融市场与资产基础/F00_金融市场与资产基础_适配教材|F00 金融市场与资产基础适配教材]]，再进入 F02/F03 的数据和风险实验。

## 在总路线中的位置

F00 属于金融工程支线的入口模块。

它服务于：

```text
金融市场基础
-> 金融数据字段和口径
-> F02 金融时间序列处理
-> F03 风险指标和投资组合
-> Q01 金融数据分析与风险指标实验
-> P03 金融工程 workload
```

它和 M12 的边界是：

```text
F00 负责解释金融资产和数据口径。
M12 负责金融文档 RAG / Agent 场景。
两者在 SEC EDGAR、财报、公告和金融指标解释处交叉。
```

## 要解决的问题

- 金融市场里常见资产类别分别是什么？
- 股票、债券、基金、期货、期权这些数据为什么不能混在一起理解？
- 价格、收益率、利率、期限、信用风险、流动性分别描述什么？
- 金融数据来自哪里，哪些是市场价格，哪些是财报，哪些是宏观数据？
- 为什么后续做 RAG、风险指标、回测、P03 task 前必须先确认数据口径？

## 学习目标

- [ ] 能解释股票、债券、基金、期货、期权的基本区别。
- [ ] 能说明价格、收益率、利率、波动率、流动性和信用风险的含义。
- [ ] 能区分市场价格数据、财报数据、宏观数据和公告文本。
- [ ] 能说明 adjusted close、filing、series、observation 等数据入口的作用。
- [ ] 能为 Q01 写出一份数据字段说明。
- [ ] 能说明为什么本模块不做投资建议和实盘交易。

## 第一轮学习产物

学完 F00 后，不要求写长篇金融学笔记，只要求产出三件小东西：

1. 一张资产类别对比表：股票、债券、基金、期货、期权分别写清“数据字段、主要风险、常见来源、后续用途”。
2. 一页金融数据口径说明：解释 price、return、interest rate、filing、series、observation、adjusted close。
3. 一个 P03 任务草图：说明未来 `finance_data_task` 或 `risk_metric_task` 可以从哪些公开数据来源取数。

合格标准不是背定义，而是看到一个金融数据字段时，能判断它大概属于什么数据、该查什么来源、后续能不能用于实验。

## 推荐学习顺序

1. 先读本学习地图，明确 F00 只负责市场和数据口径。
2. 对照 F00 资料索引，先看 Yale ECON 252 的市场直觉部分。
3. 再查 MIT 15.401 中 present value、fixed income、equities 的基础章节。
4. 查 SEC EDGAR APIs 和 FRED API，理解公开数据入口。
5. 写一页数据字段说明，为 F02 的 Python 处理做准备。
