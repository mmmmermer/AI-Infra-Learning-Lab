## 本模块最值得理解的难点

F03 的难点不在于公式很多，而在于“风险指标很容易被误读成投资结论”。

| 难点 | 为什么难 | 工程痛点 | 后续影响 |
|---|---|---|---|
| 风险和收益不是同一个东西 | 高收益样本可能伴随高波动和大回撤 | 只看收益会误判策略或资产 | 报告和简历表达容易虚高 |
| VaR 不是最大亏损 | 它只是分位数口径 | 忘记置信水平和样本范围 | 风险解释错误 |
| 最大回撤依赖路径 | 同样收益分布，顺序不同回撤不同 | 只保存 summary 会丢路径信息 | 回测和报告不可解释 |
| 组合风险不是风险相加 | 相关性和协方差决定共同波动 | 多资产合并和对齐容易出错 | portfolio_task 字段不完整 |
| Sharpe ratio 容易被滥用 | 它依赖样本、频率、无风险利率和波动 | 不同实验不可比 | 面试解释容易被追问 |
| 有效前沿不是投资建议 | 它是给定假设下的风险收益示意 | 约束和输入变化会改变结果 | 不能包装成最优投资组合 |

本模块的学习重点是：

```text
每个风险指标都必须带着数据范围、计算口径、参数和限制一起出现。
```

## 一条贯通学习线

F03 不按“风险指标清单”来学，而按“清洗后的收益率如何变成风险报告”来学：

```text
F02 cleaned returns
-> 单资产风险
-> VaR / max drawdown
-> 组合权重
-> 协方差和相关性
-> 风险收益曲线
-> P03 risk_metric_task / portfolio_task
```

这条线的重点是：风险指标必须被放回样本、权重、频率和限制里理解。F03 的产出不是投资结论，而是能被 GF03 实验记录、Q01/Q02 候选项目和 P03 task 复用的风险解释。

## 可迁移的原则

1. 风险指标是解释工具，不是投资结论。VaR、volatility、max drawdown、Sharpe 都只能在给定样本和假设下辅助理解风险。
2. 组合风险来自共同波动，不是单资产风险相加。相关性、协方差、权重顺序和数据对齐，是 portfolio_task 最容易出错的地方。
3. 所有风险报告都要同时写限制。样本长度、资产范围、频率、无风险利率、置信水平和非投资建议声明，必须跟结果一起出现。

## 踩坑现场

> 你算出一个组合 Sharpe 很高，于是想写“组合表现优秀”。这已经越界了。合格写法应该是：在某个样本区间、某种频率和某个无风险利率假设下，该组合的 Sharpe 为 X；该结果不代表未来表现，也不构成投资建议。

## 第 1 章：风险指标不是投资结论

### 1.1 本章解决什么问题

F03 的第一条边界是：风险指标用于描述样本，不用于承诺未来。

你可以说：

```text
在这段自造/公开样本里，按照这个口径计算，max drawdown 是 X，VaR 是 Y。
```

不能说：

```text
这个资产未来最大只会亏 Y。
```

### 1.2 金融直觉

风险指标像体检指标。它能告诉你样本中出现过什么情况，但不能保证未来不会发生更极端的事情。

一个指标必须和它的上下文一起读：

- 数据来自哪里。
- 时间范围是什么。
- 频率是什么。
- 指标怎么计算。
- 样本是否足够。
- 有哪些限制。

### 1.3 工程场景

P03 的风险任务不能只保存一个数字。它至少要保存：

```text
input_json:
  asset_list
  date_range
  return_field
  parameters

result_json:
  metrics
  limitations
  disclaimer
```

### 1.4 常见错误

- 把历史风险指标写成未来预测。
- 只保存指标，不保存参数。
- 不说明样本数据是自造还是公开来源。
- 在项目表达里写成“实现投资风险控制系统”，但没有真实系统和数据记录。

### 1.5 小练习

把下面一句话改成合格表达：

```text
这个资产 95% VaR 是 2%，所以很安全。
```

合格表达应包含：样本范围、计算口径、置信水平、限制和非投资建议说明。

## 第 2 章：单资产风险指标

### 2.1 本章解决什么问题

在谈组合之前，先要会描述单个资产的风险。

第一轮最小指标：

- 平均收益。
- 收益率标准差。
- rolling volatility。
- max drawdown。
- historical VaR。

### 2.2 数学/统计口径

平均收益：

```text
mean_return = mean(r_t)
```

波动率：

```text
volatility = std(r_t)
```

Historical VaR：

```text
VaR_95 = - quantile(returns, 0.05)
```

Max drawdown：

```text
wealth = product(1 + r_t)
drawdown = wealth / cumulative_peak - 1
max_drawdown = min(drawdown)
```

### 2.3 最小代码

```python
confidence_level = 0.95
tail_probability = 1 - confidence_level

mean_return = returns.mean()
volatility = returns.std()
historical_var = -returns.quantile(tail_probability)

wealth = (1 + returns).cumprod()
peak = wealth.cummax().clip(lower=1.0)  # 把初始净值 wealth_0=1 纳入峰值
drawdown = wealth / peak - 1
max_drawdown = drawdown.min()
```

回归夹具必须包含“首期收益为 `-0.10`”的序列，并断言首期 drawdown 为 `-0.10`。否则
`cummax()` 会把样本第一条 wealth 误当成历史起点，静默漏掉从初始本金到首期净值的回撤。

### 2.4 工程痛点

同一个指标可能因为口径不同而不同：

- `std()` 是否使用样本标准差。
- VaR 输出是正数损失还是负数收益。
- drawdown 是否从 1 的累计净值开始。
- 数据频率是日、周还是月。
- 是否年化。

所以实验记录里必须保存：

```text
frequency
return_method
confidence_level
var_sign_convention
annualized_or_not
sample_date_range
```

## 第 3 章：组合权重和组合收益

### 3.1 本章解决什么问题

组合不是把资产放在一起就结束了。组合的核心是权重。

如果两个资产在区间 `t` 的简单收益率分别是 `r_A,t` 和 `r_B,t`，区间开始前已经持有的
权重是 `w_A,t-1` 和 `w_B,t-1`，组合收益是：

```text
r_p,t = w_A,t-1 * r_A,t + w_B,t-1 * r_B,t
```

这里的权重必须早于收益区间确定。对数收益适合跨时间相加，但不能直接按这个式子做资产间
加权来替代真实组合简单收益。

### 3.2 金融直觉

权重表示你把多少比例放在每个资产上。第一轮只考虑最简单情况：

```text
w_A + w_B = 1
w_A >= 0
w_B >= 0
```

这意味着不考虑杠杆和卖空。

### 3.3 最小代码与再平衡假设

下面的常数权重代码隐含“每一期开始都恢复到 60%/40%”，也就是定期再平衡，不是一般的
买入持有路径：

<!-- textbook-code: role=fragment env=python-3.13 network=off -->
```python
weights = {"A": 0.6, "B": 0.4}

rebalanced_portfolio_return = (
    weights["A"] * returns_df["A"]
    + weights["B"] * returns_df["B"]
)
```

买入持有时权重会随价格漂移。若初始权重为 60%/40%，第一期 A 收益为 10%、B 收益为
0%，组合收益是 6%，期末权重变为：

```text
w_A,end = 0.6 * 1.10 / (0.6 * 1.10 + 0.4 * 1.00) = 0.622642
w_B,end = 0.377358
```

第二期若不再平衡，就必须使用漂移后的 62.2642%/37.7358%，不能继续套 60%/40%。

### 3.4 工程痛点

组合收益计算看起来简单，但真实工程里会遇到：

- 两个资产日期不对齐。
- 某个资产历史数据更短。
- 权重是否每天再平衡没有说明。
- 用期末权重去乘同一期收益，形成前视偏差。
- 把对数收益直接做横截面加权，当成组合简单收益。
- 权重和是否等于 1。
- 是否允许负权重。

第一轮必须记录：

```text
asset_list
weights
weight_sum
rebalance_assumption
join_method
```

## 第 4 章：协方差、相关性和组合风险

### 4.1 本章解决什么问题

组合风险不是资产风险的简单加权平均。资产之间是否一起涨跌，会影响组合波动。

两资产组合方差：

```text
var_p = w_A^2 var_A + w_B^2 var_B + 2 w_A w_B cov_AB
```

### 4.2 金融直觉

如果两个资产总是一起涨跌，组合风险很难被分散。如果一个涨时另一个不一定涨，组合波动可能降低。

这就是相关性的意义。

### 4.3 最小代码

```python
cov_matrix = returns_df[["A", "B"]].cov()
weights_vector = [0.6, 0.4]

portfolio_variance = (
    weights_vector @ cov_matrix.values @ weights_vector
)
portfolio_volatility = portfolio_variance ** 0.5
```

### 4.4 Worked example：相关性上升时，分散化可能失效

沿用 F01 的教学矩阵和 `[0.6, 0.4]` 权重：

```text
var_A = 0.0004
var_B = 0.0009
cov_AB = 0.0001
base portfolio volatility = 1.833%
```

压力情景不是预测，而是问“如果共同波动明显增强，会怎样”。把 `cov_AB` 提高到 `0.0005`，
其他输入保持不变：

```text
stressed variance
  = 0.6^2 * 0.0004
  + 0.4^2 * 0.0009
  + 2 * 0.6 * 0.4 * 0.0005
  = 0.000528

stressed volatility = sqrt(0.000528) = 2.298%
```

单资产波动没有改变，组合波动仍从 1.833% 上升到约 2.298%。这说明历史低相关不能被当成
稳定保护；压力值是人为情景，也不能写成未来一定发生。

### 4.5 可复现反例：列顺序变化但权重没有变化

如果 `returns_df` 从 `[A, B]` 变成 `[B, A]`，而 `weights_vector` 仍是 `[0.6, 0.4]`，矩阵
乘法不会报错，却悄悄改变了经济含义。修复方式是使用 `weight_by_asset` 按 `asset_order`
重建向量，并用交换列顺序的回归测试证明结果不变。

### 4.6 工程痛点

协方差矩阵非常依赖输入数据：

- 缺失值处理方式不同，协方差会变。
- 样本时间范围不同，相关性会变。
- 极端值会显著影响协方差。
- 资产数量增加后，矩阵维度和权重顺序必须对齐。

所以 `portfolio_task` 必须记录：

```text

weights_order
covariance_method
date_range
missing_data_policy
base_covariance_matrix
stress_covariance_matrix
stress_rationale
```

### 4.7 常见错误

- 只看单资产波动率，忽略相关性。
- 权重顺序和收益率列顺序不一致。
- 不说明样本时间范围。
- 把低相关性直接解释成无风险。

## 第 5 章：Sharpe ratio 的使用边界

### 5.1 本章解决什么问题

Sharpe ratio 常被用来描述“单位风险获得多少超额收益”。第一轮只需要理解它的直觉和限制。

### 5.2 数学口径

简化形式：

```text
Sharpe = mean(return - risk_free_rate) / std(return)
```

如果第一轮不用无风险利率，可以记录：

```text
risk_free_rate_assumption: 0 for learning exercise
```

### 5.3 Worked example：频率不一致时，Sharpe 没有可比性

假设某教学策略的日均超额收益为 `0.0005`，日收益标准差为 `0.01`：

```text
daily Sharpe = 0.0005 / 0.01 = 0.05
annualized Sharpe under iid square-root convention
  = 0.05 * sqrt(252)
  approximately 0.794
```

如果另一个报告直接给出月频 Sharpe，却没有说明是否年化，就不能与 `0.794` 比较。这里的
`sqrt(252)` 还依赖独立同分布等简化假设；自相关、重叠收益和非平稳性会破坏简单年化解释。

若无风险利率是年化 3%，也不能直接从日收益减去 `0.03`。第一轮可以明确采用简化日化
`0.03 / 252`，或使用匹配期限的实际序列，但必须记录转换规则。

### 5.4 可复现反例：把月频和日频结果放在同一排行榜

错误报告只保留 `strategy_name` 和 `sharpe`，丢掉 frequency、annualization、risk-free
series 和 sample range。排序看似客观，实际比较了不同量纲和样本。

修复后，任何比较表至少包含：

```text
return_frequency
annualization_factor
risk_free_source
risk_free_conversion
sample_start
sample_end
observation_count
sharpe_method
```

### 5.5 工程痛点

Sharpe ratio 的问题是很容易被拿来比较不同实验，但比较前必须确认：

- 数据频率是否一致。
- 是否年化。
- 无风险利率如何处理。
- 样本时间范围是否一致。
- 是否存在极端值。

### 5.6 常见错误

- 不说明无风险利率。
- 用小样本 Sharpe 做结论。
- 不同频率 Sharpe 直接比较。
- 把高 Sharpe 写成投资能力。

### 5.7 独立练习

给定日均收益 `0.0007`、日标准差 `0.012`、年化无风险利率 2.5%：

1. 明确你的日化无风险利率简化规则。
2. 计算日频超额收益 Sharpe 和按 `sqrt(252)` 简化年化的结果。
3. 把样本量从 252 改成 20，说明公式数值能算并不代表估计可靠。
4. 列出至少三个使简单年化失真的条件。

验收要求：单位和频率全程一致；无风险利率转换可复算；结果旁必须保存样本量、假设和限制。

### 5.8 检查标准

- [ ] 能解释 Sharpe ratio 的直觉。
- [ ] 能说明它依赖样本和频率。
- [ ] 能说明当前阶段不把 Sharpe 写成投资建议。
- [ ] 能在频率一致的前提下计算并记录简化年化 Sharpe。
- [ ] 能识别“数值可算”和“估计可靠”不是同一件事。

## 第 6 章：两资产风险收益曲线

### 6.1 本章解决什么问题

当权重从 0 到 1 变化时，组合收益和组合风险会形成一条曲线。这个实验能直观看到权重和相关性如何影响组合。

### 6.2 最小流程

```text
准备两个资产的收益率
-> 对齐日期
-> 枚举权重 0%, 10%, ..., 100%
-> 计算组合收益
-> 计算组合波动
-> 画 risk-return 曲线
```

### 6.3 Worked example：三个权重点不是三个推荐方案

假设 A、B 的日均收益分别为 `0.0004`、`0.0002`，协方差矩阵为：

```text
[[0.0004, 0.0001],
 [0.0001, 0.0009]]
```

枚举三个 A 权重：

| A 权重 | B 权重 | 组合日均收益 | 组合日波动率 |
|---:|---:|---:|---:|
| 0% | 100% | 0.020% | 3.000% |
| 50% | 50% | 0.030% | 1.936% |
| 100% | 0% | 0.040% | 2.000% |

中间点波动较低来自这组协方差假设，不是“50/50 永远最优”。改变样本、相关性、成本、约束
或目标后，曲线会改变。

### 6.4 从风险度量到最小风险管理闭环

风险管理不能停在画图。第一轮至少形成：

```text
识别风险：相关性上升、集中度、数据失真
-> 度量：基础波动和压力波动
-> 限额：教学阈值及其理由
-> 动作：降低集中度、暂停结果发布或要求复核
-> 监控：输入、指标、越限和处置记录
-> 复核：情景是否合理、数据是否更新、限制是否仍成立
```

限额是教学治理规则，不是从三个点自动推导的投资建议。学习者必须写出触发条件、负责人和
处置证据，不能只圈出曲线上“看起来最好”的点。

### 6.5 工程痛点

这类图最容易被误读成“最佳组合推荐”。第一轮只把它当作学习实验：

- 不使用真实资金。
- 不推荐权重。
- 不承诺收益。
- 不写“最优组合”结论。
- 只解释权重变化和风险收益关系。

### 6.6 独立练习

枚举 A 权重 `0%, 25%, 50%, 75%, 100%`，输出基础和 `cov_AB=0.0005` 压力情景两张表：

1. 比较哪个权重对相关性压力最敏感。
2. 设计一个只用于教学的波动率或集中度复核阈值。
3. 写出越限后的动作和记录字段。
4. 说明为什么阈值、动作和结果都不构成投资建议。
