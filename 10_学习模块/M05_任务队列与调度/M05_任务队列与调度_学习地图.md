- 任务队列为什么是调度系统的入口？
- FIFO、Priority、SJF 分别优化什么，又牺牲什么？

- 为什么平均值不够，P95/P99 尾延迟为什么重要？

## 核心内容

### 第一层：最小调度模型

- Task：任务 id、到达时间、预计耗时、优先级、状态。
- Worker：worker id、可用时间、执行中的任务。
- Queue：等待执行的任务集合。
- Scheduler：从队列里选择下一个任务。
- Metrics：记录等待时间、完成时间、尾延迟、资源利用率。

### 第二层：调度策略

- FIFO：先到先服务，作为 baseline。
- Priority：优先级高的任务先执行。
- SJF：预计耗时短的任务先执行。
- Round Robin：理解即可，后续再深入。
- Cost-aware：把耗时、token、worker 占用、优先级组合成成本分数，作为进阶实验。

### 第三层：实验与评价

- baseline 对比
- 高峰负载
- 长短任务混合
- 高低优先级混合
- 单 worker 与多 worker
- 平均值与 P95/P99
- 公平性与饥饿风险

### 第四层：云原生映射

- Kubernetes Scheduler：Pod 如何被调度到 Node。
- Kueue：面向 Kubernetes batch workload 的队列、准入、资源配额。
- Volcano：面向 AI / 大数据 / HPC 场景的 batch scheduling、gang scheduling、queue resource management。

当前阶段只做概念映射，不直接读源码，不直接改 Kubernetes 调度器。

## 推荐学习顺序
1. 先读本学习地图，确认 M05 的边界：本模块只负责任务、队列、worker、策略、指标和实验解释，不负责 FastAPI、数据库、Kubernetes 源码或真实 GPU 调度。
2. 再读 [[10_学习模块/M05_任务队列与调度/M05_任务队列与调度_章节教材|M05 章节教材]] 第 0-1 章，先把“P01 是参考答案”和“调度不是简单排序”这两个前提立住。
3. 打开 [[10_学习模块/M05_任务队列与调度/M05_任务队列与调度_复现推进表|M05 复现推进表]]，按 R0-R9 安排每轮亲手复现、实验观察和复盘输出。
4. 先完成 [[40_实验练习/E05_调度实验/E05-R0-R1 学习边界与最小模型复现|E05-R0-R1]]，把学习边界、P01 定位、Task / Worker 最小模型写成自己的记录。
5. 按章节教材第 2 章，亲手写 Task / Worker / Queue 最小模型，并解释 `submit_time`、`start_time`、`finish_time`、`available_at`。
6. 按章节教材第 3 章，亲手实现 FIFO / 单 worker 模拟，并完成 E05-01，把 FIFO baseline 跑通。
7. 按章节教材第 4-5 章，实现 Priority 和 SJF，并用同一组任务比较 FIFO / Priority / SJF。
8. 按章节教材第 6 章，补 average、P95/P99、worker utilization 等指标，再解释三种策略的差异。
9. 按章节教材第 7 章进入高峰负载实验，完成 E05-03，重点解释低负载和高峰负载下策略差异为什么不同。
10. 按章节教材第 8-10 章学习 Cost-aware、分组分析和 aging，理解权重偏好、谁被牺牲、最大等待保护的代价。
11. 按章节教材第 11 章完成 E05-05，独立实现 worker pool 并解释 P95 与 utilization 的共同变化。
12. 按章节教材第 12 章完成 E05-06，先建立本人证据表，再决定哪些内容可以转成 README 或简历表达。
13. 最后再看 Kueue / Volcano / Kubernetes Scheduler 的概念映射，理解自己的小项目如何通向云原生调度方向。
