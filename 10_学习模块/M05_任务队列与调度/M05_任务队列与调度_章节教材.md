## 学习主线

本教材围绕一条主线展开：

```text
先理解调度问题 -> 再建立任务和 worker 模型 -> 写出 baseline -> 比较策略取舍 -> 用指标和实验解释结果
```

P01 Mini Scheduler 在这里不是直接拿来当成果展示的项目，而是：

```text
M05 教材配套参考答案 / 实验样板 / 后续亲手复现的对照材料
```

学习时要先亲手复现核心逻辑，再对照 P01。README、简历表达和图表素材只有在你完成复现并能解释结果之后，才适合转化成自己的项目表达。

当前阶段暂时不扩展：

- FastAPI
- 数据库
- Redis / Celery
- Kubernetes
- 真实 RAG 请求
- 复杂 GPU 调度

这些内容会放到 M02 / M06 / M08 / M09 / P03 等后续模块。M05 当前只负责把调度内核学扎实。

## Python 版本说明

当前主学习环境固定为 Python 3.13。进入项目目录后使用项目虚拟环境：

```powershell
py -3.13 -m venv .venv
.\.venv\Scripts\Activate.ps1
python --version
```

本教材统一使用现代类型标注：

```python
float | None
list[Task]
dict[str, list[Task]]
```

旧代码顶部的 `from __future__ import annotations` 可以保留，但不再作为支持 Python 3.8/3.9 的理由。版本异常时先检查解释器和虚拟环境，不通过修改类型标注掩盖环境漂移。

## 每章学习闭环

每一章都按同一个学习闭环推进：

```text
读概念 -> 手写最小代码 -> 跑小样例 -> 观察结果 -> 对照 P01 -> 写复盘
```

每章尽量包含：

- 本章目标：学完后应该会什么。
- 概念讲解：为什么要学这一点。
- 工程场景：它在 AI / RAG / Agent / 平台系统里对应什么问题。
- 关系图：对象和数据流如何连接。
- 代码跟做：你要亲手写的最小代码。
- 运行命令或运行方式：如何验证代码。
- 观察问题：看结果时应该关注什么。
- 常见错误：提前避开容易踩的坑。
- 参考答案位置：复现后到 P01 的哪里对照。
- 复盘问题：把“会运行”变成“会解释”。

这套闭环的目的，是防止学习变成“只看资料”或“只复制代码”。你需要把每章内容转化成自己的理解、代码和解释。
