# F07 金融机器学习与模型风险学习地图

## 怎么读这个模块

F07 是金融工程第二主线中和 AI/机器学习最直接相关的模块。

但它的重点不是“用深度学习预测股价”，而是理解金融 ML 为什么容易失败：

```text
标签设计错误
-> 特征泄漏
-> 时间切分错误
-> 过拟合
-> 无 baseline
-> 无失败分析
-> 无模型验证文档
```

第一轮只做小样本、可解释、可复现的模型风险训练。不要追求模型复杂度。

## 在总路线中的位置

```text
F02 金融时间序列
-> F03 风险指标
-> F06 回测工程
-> F07 金融机器学习与模型风险

```

## 要解决的问题

- 金融 ML 为什么不能随机打乱时间序列训练测试集？
- 标签、特征、预测目标分别是什么？
- baseline 为什么比复杂模型更重要？
- 什么是数据泄漏、过拟合、样本外失效？
- 模型风险如何记录和复盘？
- 什么是 baseline/challenger、压力测试、模型卡和人工复核？

## 学习目标

- [ ] 能构造一个简单金融分类或回归任务。
- [ ] 能使用时间顺序切分训练集和测试集。
- [ ] 能建立一个简单 baseline。
- [ ] 能设计 baseline/challenger 对比。
- [ ] 能识别至少三类数据泄漏风险。
- [ ] 能写模型卡式说明：数据、特征、标签、指标、失败样例、限制。
- [ ] 能做一个简单 stress test 或 out-of-sample 检查。
- [ ] 能记录模型指标、失败样例和限制。
- [ ] 能说明金融 ML 输出不能直接写成投资建议。

## 第一轮学习产物

1. 一个小型模型任务说明：预测什么、输入什么、不能用什么。
2. 一个时间切分示例：train/test 按时间顺序分开。
3. 一个 baseline 对比表。
4. 一份泄漏检查清单。
5. 一份模型风险说明：这个模型不能说明什么。
6. 一张模型卡：数据来源、特征、标签、切分方式、指标、失败样例、适用边界。

## 检查标准

- [ ] 能解释为什么不能随机打乱金融时间序列。
- [ ] 能写出最小 baseline。
- [ ] 能解释 baseline 和 challenger 的区别。
- [ ] 能指出至少三种数据泄漏。
- [ ] 能写一份模型卡式记录。
- [ ] 能说明 stress test 和 out-of-sample 检查为什么重要。
- [ ] 能说明指标只是模型表现，不是投资承诺。
- [ ] 能把模型评测记录连接到 M11 实验设计。

## 暂时不深入

- 不做深度学习模型堆叠。
- 不做无 baseline 的模型比较。
- 不做收益预测宣传。
- 不做高频数据建模。
- 不把模型结果写成交易建议。
