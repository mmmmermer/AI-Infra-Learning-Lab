# M00 工具链与计算机基础学习地图

## 怎么读这个模块

把 M00 当成“以后自己能跑项目、查报错、保存进度”的生存手册，不要当成零散电脑知识清单。

第一遍只抓四件事：路径和命令行、Python 环境、Git、报错和测试。每学一小节，都问自己：这能不能帮我独立运行 P01 或排查一个错误？

资料和工具不用一次学全。遇到项目里真实用到，再回到对应小节查。

## 在总路线中的位置

这是基础薄弱阶段的前置模块。它不直接产出 AI Infra 项目功能，但决定你后面能不能独立跑项目、改代码、看报错、写测试和复盘问题。

如果 M00 没打牢，后面的 FastAPI、RAG、Agent、Docker、K8s 很容易变成“每一步都靠别人救火”。

## 要解决的问题

- 能不能独立配置 Python 项目环境？
- 能不能看懂命令行报错？
- 能不能用 Git 保存和回退自己的学习进度？
- 能不能理解 HTTP API 是怎么请求和响应的？
- 能不能观察进程树、退出码和清理，并修复一次权限失败？
- 能不能把请求失败定位到 DNS、TCP、TLS 或 HTTP，而不是统称“网络问题”？
- 能不能用日志和测试定位问题？
- 能不能把“AI 生成的代码”变成自己能维护的代码？

## 学习内容

### 工具链

- Windows 路径、工作目录、环境变量
- PowerShell 常用命令
- Git：init、clone、add、commit、branch、diff、log
- Python 虚拟环境
- pip / uv / requirements
- VS Code 或 Cursor 调试
- `.env` 配置文件

### 编程基础

- 变量、函数、类、模块、包
- list / dict / set / tuple
- 文件读写
- JSON
- 异常处理
- 类型标注
- logging

### Web 基础

- HTTP 请求和响应
- GET / POST / PUT / DELETE
- 状态码
- headers
- JSON body
- REST API 基本设计
- curl / Postman / httpx

### 工程基础

- 项目目录结构
- README 怎么写
- pytest 最小测试
- 错误复现步骤
- 运行日志和错误日志
- 最小可复现问题

### 系统基础

- PID/PPID、退出码、环境变量和 stdout/stderr
- POSIX SIGTERM、Windows 平台差异与优雅清理
- 文件/父目录写权限与失败恢复
- DNS -> TCP -> TLS -> HTTP 请求路径
- 线程、进程与异步的详细选择留给 M01
- 队列的直觉
- 端口和本地服务
- 数据库、缓存、消息队列分别解决什么问题

## 检查标准

- [ ] 能在命令行进入项目目录并运行 Python 文件
- [ ] 能独立创建虚拟环境并安装依赖
- [ ] 能用 Git 做一次完整提交
- [ ] 能解释一个 HTTP 请求包含什么
- [ ] 能读懂常见 Python 报错
- [ ] 能写 3 个 pytest 测试
- [ ] 能用日志定位一次 bug
- [ ] E00-06 的 11 项测试通过，能解释超时回收与 Windows signal fixture 的证据边界
- [ ] 能用单变量证据区分 DNS/TCP/TLS/HTTP，并说明 500 不等于网络不可达

- Linux 内核细节
- 复杂 shell 脚本
- Git rebase 和高级协作流
- 操作系统完整课程
- 复杂网络协议
- 分布式一致性算法

这一模块的目标不是“学完计算机基础”，而是先拿到自学工程项目所需的最低工具能力。
