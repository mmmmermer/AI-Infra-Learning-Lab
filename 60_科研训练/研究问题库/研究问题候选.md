# 研究问题候选

## 当前主问题

在 RAG/Agent 请求类型差异较大的场景下，如何通过任务调度降低尾延迟并提升 worker 利用率？

主项目入口：[[60_科研训练/研究项目/RQ01_RAG_Agent请求调度尾延迟/00_研究问题|RQ01 RAG/Agent 请求调度尾延迟]]

## 候选问题

### 1. 不同调度策略对 RAG/Agent 请求尾延迟的影响

- baseline：FIFO / Priority / SJF / Random
- 指标：P95/P99、平均等待时间、吞吐
- 场景：短问答、长文档总结、报告生成

### 2. 多租户配额对公平性和吞吐的影响

- baseline：无配额 / 固定配额 / 动态配额
- 指标：用户等待时间、失败率、资源利用率

### 3. 成本感知调度对 token 成本和延迟的权衡

- baseline：只看优先级 / 同时考虑成本
- 指标：成本、延迟、吞吐

### 4. 云原生扩缩容对高峰 AI workload 的影响

- baseline：固定 worker / HPA
- 指标：冷启动时间、P95、资源利用率

### 5. 边缘节点资源受限下的 AI 任务分配

- baseline：本地执行 / 云端执行 / 混合策略
- 指标：延迟、成功率、成本
