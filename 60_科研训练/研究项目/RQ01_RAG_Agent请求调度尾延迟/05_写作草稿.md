# RQ01 写作草稿

## 当前一句话

我已完成一组可重放的 30-seed 合成调度 pilot，研究服务时间预测误差如何影响非抢占调度的尾等待与公平性；当前观察限定于合成模拟，P03 服务回放和真实 RAG/Agent 场景结论尚未完成。

## 导师沟通版本

老师您好，我正在围绕混合 AI workload 的外层任务调度做一项入门研究训练。

我目前把问题收窄为：在非抢占、多 worker 的队列模型中，服务时间预测误差和负载强度会怎样影响 predicted SJF 相对 FIFO 的 P95/P99 queue wait，以及短任务改善与长任务等待之间的公平性权衡。oracle SJF 只作为不可部署的上界，不与 predicted SJF 混写。

工程上，我已经修订 Mini Scheduler 的状态、指标和 predicted/oracle 语义，并实现开放环 workload、三档预测误差、common random numbers、30-seed 配对 bootstrap CI、分任务类别 slowdown/SLO violation 和可校验 artifact。下一步是补 Pareto/burst、最终 seed 数决策，并接入 P03 固定请求回放。

希望请教您两个问题：这个问题是否适合作为系统方向的复现加扩展训练？在 prior art 和公平性指标上，哪些工作或评价口径应优先补齐？

## 简历口径

当前只能写“研究训练/正在实现”：

> 正在构建面向混合 AI 请求的可重放调度实验框架，已完成 FIFO、priority、predicted/oracle SJF 和 cost-aware 的离散事件参考实现与自动测试，并设计预测误差、重尾负载、配对重复和公平性分析 protocol；真实服务回放与统计结论仍在进行中。

在多 seed 合成实验和 P03 回放完成前，不写：

- “降低 P99 xx%”。
- “显著优于 FIFO”。
- “提升 RAG/Agent 平台性能”。
- “提出新型调度算法”。
