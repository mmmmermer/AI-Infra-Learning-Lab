# RQ01 研究问题

> 当前状态（2026-07-11）：`E2 synthetic pilot reference / E3-E4 blocked / no scenario conclusion`。已完成 30 seeds、三档预测误差、FIFO/predicted/oracle SJF 的开放环配对 pilot，并保存 54,000 条 workload 与 162,000 条逐任务结果。该证据只支持指定合成模拟条件下的观察，不支持 RAG/Agent、生产性能或策略普遍优越性结论。

## 暂定题目

混合 AI 请求负载下，服务时间预测误差如何影响非抢占调度的尾等待、公平性与资源利用率？

“RAG/Agent”是目标场景，不是当前已验证的数据来源。第一阶段只研究抽象 workload；只有 P03 能采集真实阶段耗时后，题目才可收窄到 RAG/Agent。

## 主研究问题

在固定 worker 数量、开放环到达序列和混合服务时间分布下，与 FIFO 相比，`predicted_sjf` 在不同预测误差和负载强度下如何改变：

1. 全体任务的 P95/P99 queue wait。
2. 短、中、长任务各自的 P95/P99 和最大等待。
3. slowdown、SLO violation 和长期等待任务比例。
4. throughput、worker utilization 和最大 ready queue length。

## 次研究问题

- `oracle_sjf` 与 `predicted_sjf` 的差距随预测误差如何变化？
- aging 或显式最大等待保护能否降低长任务饥饿风险？
- worker 数量增加时，尾延迟改善与利用率下降之间的权衡是什么？
- 合成 workload 上的观察能否在 P03 固定请求回放中复现？

## 可检验假设

| 编号 | 假设 | 反证条件 |
|---|---|---|
| H1 | 在高负载且预测误差较低时，predicted SJF 相对 FIFO 降低整体平均等待和短任务 P95。 | 配对置信区间包含无改善，或多数负载条件下方向相反。 |
| H2 | 预测误差增大时，predicted SJF 与 oracle SJF 的差距扩大。 | 差距不随误差增大，或变化主要由其他因子解释。 |
| H3 | predicted SJF 可能改善短任务，但恶化长任务 max wait、P99 或 slowdown。 | 分组指标未出现该权衡，或 aging 后无可测差异。 |
| H4 | 增加 worker 数量降低 queue wait 尾部，但在固定 workload 下可能降低 utilization。 | 配对结果没有这一方向，或系统瓶颈转移导致相反结果。 |

假设不是结论。最终报告必须保留无效、相反或不确定结果。

## 操作性定义

| 名称 | 定义 |
|---|---|
| queue wait | `start_time - submit_time` |
| turnaround | `finish_time - submit_time` |
| slowdown | `turnaround / actual_service_time`，零服务时间任务不纳入 |
| predicted SJF | 只使用提交时可获得的 `estimated_duration` |
| oracle SJF | 使用不可部署的 `actual_duration`，仅作上界/诊断基线 |
| tail | 固定 percentile 方法后的 P95/P99；当前 P01 使用 nearest-rank |
| starvation proxy | max wait、per-class P99、slowdown P99、超过预注册等待阈值的比例 |

## 研究边界

- 第一阶段是离散事件模拟，不是生产服务 benchmark。
- 当前调度为非抢占式；SRPT/抢占调度属于另一问题。
- worker 同质，暂不建模 GPU 型号、KV cache、batching 和数据局部性。
- `estimated_duration` 的生成机制必须显式记录，不能默认等于真实耗时。
- oracle 只能作为诊断上界，不能写成可部署方案。
- 不把模拟 improvement 外推为 RAG 质量、模型吞吐或真实用户体验提升。
