# 金融工程高质量资料与路线审计

## 1. 本文定位

本文不是金融工程教材，也不是资料收藏夹。它的职责是为金融工程支线做第二层审计：

```text
权威课程与项目结构
-> 本库 F 系列模块设计
-> GF 实验出口
-> Q 项目出口
-> X 交叉项目与 P03 workload
```

当前结论：

```text
金融工程支线可以建设，
但当前不应一次性创建 F00-F08 全套教材。
正确推进方式是：
先固化权威资料来源和能力映射，
再只建设 F00-F03 的学习地图和资料索引第一版。
```

## 2. 审计原则

金融工程支线必须遵守现有学习库规则：

- 不替代 AI Infra 主线。
- 不塞进 M12 金融投研 AI 场景。
- 不写投资建议。
- 不承诺收益。
- 不把回测结果包装成真实投资能力。
- 不收录无法验证来源的策略经验。
- 不用资料数量制造完整感。
- 每条资料必须能转成模块、实验、项目或表达。

当前支线建设优先级：

```text
路线质量
> 资料可信度
> 自学可读性
> 工程实验出口
> P03 交叉出口
> 教材篇幅
```

## 3. 当前路线是否需要重构

现有 `F00-F08` 结构总体合理，不需要推倒重来。

它和顶级金融工程项目的共性基本一致：

- 金融市场与资产基础。
- 数学、概率、统计、优化。
- 金融数据与时间序列。
- 投资组合与风险管理。
- 衍生品与定价。
- 固定收益与利率。
- 量化研究与回测。
- 金融机器学习与模型风险。
- 工程化任务接口。

需要优化的不是模块名称，而是每个模块的建设方式：

| 当前问题 | 影响 | 处理方式 |
|---|---|---|
| 路线草案已经有，但资料未按模块拆解 | 后续写教材容易凭空扩写 | 先做资料映射，再写 F00-F03 学习地图 |
| F00-F08 都有目标，但缺少“必读/选读/查阅”层级 | 容易一次性学太多 | 第一轮只激活 F00-F03 |
| 项目出口有 Q01-Q04，但还未定义最小实验 | 容易变成概念路线 | 先用 GF00-GF03 实验支撑 Q01/Q02 |
| 和 P03 有连接，但还不够字段化 | 交叉项目容易泛泛而谈 | 在 F08 再定义 task 类型、输入、输出、metrics |

## 5. F00-F08 资料与能力映射

### F00 金融市场与资产基础

学习定位：

```text
让学习者知道金融数据和金融资产到底在描述什么，
避免后面直接拿价格序列做计算却不知道口径。
```

推荐资料：

- 必读：Yale ECON 252 Financial Markets。
- 必读：MIT 15.401 中 Introduction、Present Value、Fixed-Income、Equities。
- 查阅：SEC EDGAR APIs。
- 查阅：FRED API。

核心知识：

- 资产类别：股票、债券、基金、期货、期权。
- 市场参与者：发行人、投资者、交易所、做市商、监管机构。
- 基础口径：价格、收益率、利率、期限、风险、流动性、杠杆。
- 数据来源：市场价格、财报数据、宏观数据、公告文本。

实验出口：

- GF00-01 收益率和波动率计算。
- GF00-02 股票/债券/指数数据字段对比。

工程出口：

- 为 Q01 提供字段说明和数据口径。
- 为 M12 金融文档 RAG 提供概念边界。

暂不深入：

- 高频市场微观结构。
- 实盘交易策略。
- 投资建议。

实验出口：

## 6. 第一轮实际建设建议

第一轮不要补全 F00-F08。

建议只推进：


- F00-F03 是金融工程支线的入口层。

- 它们能和 M12、M03、P03 形成早期交叉。
- 它们不会过早引入随机微积分、复杂期权、回测平台、金融 ML。

暂不建设：

```text
F04-F08 适配教材

Q03/Q04 项目工作台
X02 详细实现
```

这些内容等 AI Infra 主线和 F00-F03 有基础后再激活。

## 7. F00-F03 第一版资料索引应如何写

资料索引不是摘要，而是来源管理。

每个 F 模块资料索引第一版只需要 5-8 条资料：

```text
2 条必读公开课或官方文档
2 条查阅型工具文档
1 条数据源文档
1-2 条选读或长期参考
```

每条资料必须标注：

- 资料名称。
- 链接。
- 类型。
- 状态：必读 / 选读 / 查阅 / 候选 / 暂缓。
- 对应模块。
- 对应实验。
- 对应项目。
- 为什么收录。
- 暂时不读哪些部分。

## 8. F00-F03 学习地图应如何写

学习地图不写成长教材。

每个学习地图必须回答：

- 这个模块解决什么能力缺口。
- 前置能力是什么。
- 学完能做什么实验。
- 哪些资料是必读。
- 和 Q01/Q02/P03 的关系。
- 暂时不深入什么。
- 检查标准是什么。

学习地图的质量标准：

```text
读完后知道为什么学
知道先读哪几份资料
知道做哪几个实验
知道学到什么程度算合格
知道哪些东西先不要碰
```

## 11. 质量风险与防偏移

### 风险 1：路线变成金融百科

表现：

- 每个金融名词都想收进来。
- F00-F08 同时写教材。
- 期权、债券、回测、ML 都写得很大。

处理：

```text
第一轮只做 F00-F03。
F04-F08 只保留路线级说明。
```

### 风险 2：路线变成量化策略收藏

表现：

- 收集策略文章。
- 强调收益曲线。
- 忽视风险、回撤、成本和数据泄漏。

处理：

```text
不收录无来源策略经验。
每个实验必须记录风险、假设、数据来源和失败情况。
```

### 风险 3：路线和 P03 断开

表现：

- 金融工程单独做 notebook。
- AI Infra 主线单独做调度平台。
- 两边没有共同任务模型。

处理：

```text
从 F03 开始就把风险指标设计成 task。
从 F06 开始把回测设计成异步任务。
从 F08 开始接入 P03。
```

### 风险 4：学习负担过大

表现：

- 同时学 Python、后端、调度、金融、统计、回测。
- 每周入口过多。

处理：

```text
每周只激活一条路线。
金融工程当前只做建设准备。
真正学习时先 F00-F02，不碰 F04-F08。
```
