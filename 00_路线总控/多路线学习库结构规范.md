## 1. 为什么要做多路线结构

当前学习库已经从单纯资料库升级为一个完整学习系统：

```text
路线总控
-> 学习模块
-> 资料库
-> 知识卡片
-> 实验练习
-> 项目产出
-> 科研训练
-> 就业简历
-> 计划复盘
```

现在新增金融工程方向时，不能把它直接塞进 M12，也不能复制一套新库。正确目标是：

```text
一个大学习库
+ 多条路线
+ 共享资料/实验/项目区
+ 严格边界和激活规则
```

这样既能保留 AI Infra 主线的清晰度，又能把金融工程作为长期方向分支管理。

## 3. 路线层级设计

本库采用三层路线：

```text
主线：AI Infra / AI Workload Platform
支线：金融工程与量化金融
交叉线：AI + 金融工程应用
```

### 3.1 主线：AI Infra

定位：

```text
用 RAG/Agent 构造真实 AI workload，
用后端系统承载它，
用调度器管理它，
用监控压测评估它，
再逐步迁移到云原生和推理服务平台。
```

对应模块：

```text
M00-M12
```

核心项目：

```text
P01 Mini Scheduler
P03 AI Workload Platform
RQ01 RAG/Agent 请求调度尾延迟
```

### 3.2 支线：金融工程与量化金融

定位：

```text
学习金融市场、概率统计、时间序列、投资组合、衍生品、风险管理和回测工程，
形成金融工程基本分析和实验能力。
```

对应模块建议：

```text
F00-F08
```

核心项目建议：

```text
Q01 金融数据分析与风险指标实验
Q02 投资组合与回测实验
Q03 期权定价与 Monte Carlo 实验
```

### 3.3 交叉线：AI + 金融工程

定位：

```text
把金融文档、金融数据分析、风险指标和回测任务，
转成 P03 可调度、可记录、可评测的 AI/数据 workload。
```

对应模块：

```text
M03 / M04 / M08 / M11 / M12
F02 / F03 / F06 / F07
```

核心项目建议：

```text
X01 金融文档 RAG 与事实核验
X02 金融工程任务 Workload 接入 P03
```

## 6. 学习激活规则

### 6.1 每周只激活一条路线

允许：

```text
本周主激活：AI Infra
金融工程：只读一节或不动
```

或：

```text
本周主激活：金融工程
AI Infra：只维护项目记录，不新增教材
```

不允许：

```text
同时推进 M03/M04/P03 和 F02/F03/Q02
```

### 6.2 金融工程默认冻结到 AI Infra 第一阶段稳定后

当前推荐仍然是：

```text
M00 -> M01 -> M05 -> P01
```

金融工程支线先建设路线和资料索引，不进入大规模学习。

### 6.3 交叉线只能在两个基础都具备后激活

例如：

```text
金融文档 RAG = M03 基础 + M12 场景
金融风险指标 workload = F03 风险指标 + M06/M08/P03 工程能力
回测任务调度 = F06 回测 + M05/P03 调度
```

## 7. 资料收录规则

金融工程资料也必须遵守现有 `资料筛选与学习规则`。

优先级：

1. 顶级高校公开课。
2. 权威项目或官方文档。
3. 经典教材配套公开资料。
4. 少量高质量工程文档。

每条资料必须有转化出口：

```text
F 模块教材
-> GF 实验
-> Q 项目
-> X 交叉项目
-> 简历/面试/科研表达
```

不收录：

- 炒股经验贴。
- 无来源策略分享。
- 只讲收益不讲风险和回撤的文章。
- 无法复现的数据或结论。
- 投资建议类材料。

## 8. 模块建设标准

F 系列模块也采用现有标准。

普通模块：

```text
学习地图 + 适配教材
```

核心实操模块：

```text
学习地图 + 章节教材 + 实验推进表
```

第一阶段建议只建：

```text
学习地图 + 资料索引
```

等确定要学习时再写适配教材，防止空转式补库。

## 9. 与 M12 的边界

M12 保持原定位：

```text
金融投研 AI 场景
= 金融文档 RAG / Agent / 引用 / 风险提示 / P03 workload
```

M12 不负责：

- 投资组合理论。
- 衍生品定价。
- 随机过程。
- 时间序列建模。
- 量化回测。
- 风险模型深挖。

这些进入 F 系列。

当前最优方案：

```text
保留一个大学习库。
新增路线层级。
AI Infra 仍是主线。
金融工程作为方向支线。

```

这个方案能减少冗余，同时保留长期扩展能力。
