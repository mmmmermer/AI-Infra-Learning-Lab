## 一、结论

这批材料已经不再只是资料链接集合。金融时间语义、RAG 权限边界、Kubernetes 受控故障、
推理指标和科研统计等部分已有成熟教材章节。但按 Microsoft Learn、Google、Kubernetes、
Python 和 GitHub Docs 的官方公开写作规范与发布惯例复核后，**全库仍应标记为结构化教材草稿，
不能整体标记为 `content-reviewed` 或可直接发布的自学教程**。

原因不是“还没人试学”，而是作者侧内容本身仍有三层缺口：

1. **初始事实与契约阻断已完成修订，但教程发布闭环尚未清零**：金融 rolling/drawdown、回测执行时钟与
   成本、YTM task、F08 权限字段、Agent 审批、数据库异步一致性、监控空样本和当前 P03 契约
   已完成修订；M09-M12 的安全、指标、统计和证据边界也已补正，固定时间表 sender 已补成
   通过测试的 reference。大量其他 tutorial/how-to 的可运行闭环仍未完成。
2. **Wave 1 结构归类已经完成，但显式标签不等于内容通过**：202 章均已人工归入
   `instructional/workbook/reference/design-note/appendix`，22 份教材入口也已补齐受众、前置、
   版本、用时、产物、完成口径和学习顺序；这只消除了材料用途混淆，不能证明教程闭环正确。
3. **Wave 2 已完成两批局部闭环，但首次教学证据仍不均匀**：两批共增强十三个章次，补入
   固定输入、中间状态、实际输出、错误恢复、不同输入练习、验收和就近来源。158 个
   `instructional` 章中，显式 worked example 现为 61 章、章内来源为 39 章；全 202 章的总结
   为 25 章、下一步为 18 章。章内 1415 个围栏中只有 49 个已标角色，34 个 Mermaid 中只有
   6 个具有显式可访问性元数据，stateful cleanup 也仍未系统治理。

本轮没有把以下情况记为教材内容缺陷：尚无学习者提交、尚未进行 3 至 5 人试学、没有前后测、
没有一周或四周保持数据、没有记录学习者用时或求助次数。排除这些证据后，以上结论仍然成立。

## 二、检查边界

### 2.1 本轮检查什么

- 内容是否声明受众、范围、非范围、前置知识、环境、资源和版本。
- 章节是否清楚区分概念解释、教程、单任务 how-to、参考、设计稿和答案。
- 概念是否同时说明定义、机制、直觉、适用条件和不适用边界。
- 示例是否给出必要输入、设置、关键中间状态、输出和解释。
- 操作是否能按顺序复制，命令、输出、占位符和预期结果是否分开。
- 故障是否包含现象、诊断、原因、修复和修复后检查；有持久状态时是否有清理。
- 代码语义、金融计算、统计方法、安全边界和跨章契约是否正确。
- 版本、状态和 canonical source 是否一致，是否把设计稿冒充实现。
- 练习题面、正文依据、参考答案和验收标准在内容层面是否一致。
- 图表、标题、列表、表格、警告和等价文本是否达到发布候选要求。

人工阅读分两组完成：F00-F08 共 63 章，M00-M12 共 139 章，合计 202 章；确定性矩阵与
人工阅读使用同一章节边界。矩阵用于防漏章，人工复核负责语义和领域判断。

### 2.2 本轮不检查什么

- 学习者是否亲手执行或提交产物。
- 目标学习者完成率、用时、求助次数、卡点或放弃率。
- 前测、后测、迁移任务和延迟保持。
- 章节数量或结构信号是否产生了实际学习效果。
- 是否通过 Microsoft、Google、Kubernetes、Python、GitHub 或其他机构认证。

排除的是学习者效果证据，不是作者侧验证。教程声称代码可运行、步骤可完成或安全边界正确时，
作者仍须用 reference、测试和人工审稿核实。

### 3.2 不强行统一的规则

- Microsoft Learn 展示时长，GitHub Tutorial 模型不要求预计完成时间，因此**时长不是门禁**。
- Google、GitHub 和 Kubernetes 对代码行宽或占位符 house style 不完全相同，因此只要求
  仓库内一致、占位符已解释、代码无需危险横向复制，不指定唯一格式。
- Kubernetes 明确称其 style guide 为 guidelines；本报告不把某个项目的排版偏好提升为事实标准。
- 教程不等于参考大全。Python 官方信息架构反而证明精确参考可以简洁，不需要强行补练习。

### 4.3 建议项

- 对高频误区同时给正确例、反例和错误机制。
- 示例从最小可行例渐进到真实场景，而不是从定义直接跳到完整生产系统。
- 一句聚焦一个主要意思，一段聚焦一个主题；长章先给概要，再渐进披露。
- 可选材料显式标注，下一步保持少量、可执行和有先后关系。
- 预计时长、统一占位符或固定行宽仅作为仓库风格选择，不计为通用质量门禁。

## 七、发布阻断明细

> 本节保留 2026-07-12 初始审计快照，便于追溯为什么修改。已完成项的现行状态见 7.5；
> 不应继续把已修复描述当作当前文件缺陷。

### 7.1 金融计算与数据契约

1. **F02 多资产 rolling 错用整列窗口**：第 5 章直接对长表 `return.rolling()`，会跨 symbol
   边界。第 6 章 wealth/drawdown 同样未按资产分组（F02:849-852、936-941）。
2. **F02/F03/F06 最大回撤漏初始净值 1**：若首期收益为负，当前 `cumprod().cummax()` 会把
   首期当作峰值并报告 0 回撤；教材正文提醒基准问题，但示例代码未实现。
3. **F02 数据形态不连续**：前几章在 `date` 索引、`symbol/date` MultiIndex 和普通列之间
   切换，读者不能按章顺序复制运行。
4. **F06 执行时钟和成本不自洽**：正文称次日开盘成交、当日收盘平仓/计价，只计算
   open-to-close 收益，却用跨日 `position.diff()` 计成本。若每日平仓，应计每日进出成本；
   若跨夜持有，则不能忽略 close-to-next-open 收益（F06:182-193、341-352）。
5. **F05 YTM task 缺目标价格**：task 只输入 `yield_rate` 却把 `ytm` 列为可选输出。反解 YTM
   必须有 `target_price`，否则只是从自算价格还原原输入。
6. **F07 示例契约不稳定**：一处 `pct_change()` 未固定 `fill_method=None`；切分代码依赖未定义
   `model_df/test`；`confusion_matrix()` 未固定 labels，单类数据会改变输出形状。

### 7.2 权限、安全和状态

1. **F08 允许客户端自报 `permission_group`**，与 P03 明确禁止在 `input_json` 放 tenant、user、
   permission 字段并要求服务端 principal snapshot 的现行契约冲突（F08:543-555；P03:80-90）。
2. **F08 契约漂移**：文件仍写 P03 v0.1，现行 reference 已为 v0.3；状态图漏
   `retrying -> queued`；统一 envelope 与各 task 的扁平字段互相冲突。
3. **M04 session 保存破坏 owner scope**：示例用 `(user_id, session_id)` 读取，却只按
   `session_id` 保存，与正文承诺的跨用户隔离矛盾，可能覆盖另一用户的同名 session
   （M04:1048-1079）。
4. **M04 审批不能只建状态**：`waiting_approval`、approver 字段和重新入队并不足以证明审批
   安全；必须绑定待批准对象/版本/动作、授权范围和一次性决策，并用条件更新防重复或陈旧审批。
5. **M06 前后模型自相矛盾**：前文正确教授 `tasks + outbox` 同事务，后文又出现直接
   `create task -> enqueue` 和无 CAS/lease 的 worker。后者会重新引入双写裂缝和重复副作用，
   相关章节修复前应暂降 design-note（M06:433-452、987 附近）。

### 7.3 监控、API 和教程承诺

1. **M08 Prometheus 标签自相矛盾**：示例直接使用 `request.url.path` 作为 label，后文又明确
   禁止高基数 path。动态 task/document ID 会造成标签爆炸，应使用路由模板（M08:419-455）。
2. **M08 空样本百分位返回 0**：`percentile([])=0` 把“没有观测”伪装成“零延迟”，应返回
   null/undefined 并携带 sample count（M08:595-602）。
3. **M08 仍依赖旧 `/rag/query`、`/health` 和 Prometheus 文本 `/metrics` 假设**，而当前 P03
   主闭环是 `/tasks`，`/metrics` 返回受鉴权保护的 JSON；固定回放只有 manifest 设计，没有
   可运行 sender，不能称完整 how-to。
4. **F08 第 7 章标题称完整 walkthrough**，实际只有占位 JSON，没有 schema 验证、worker、
   预期响应、查询结果或故障恢复，违反 B01/B04。
5. **M12 引用和权限不足**：URL/chunk ID 不足以稳定定位监管文件；每个关键点一个 source 不等于
   source 支持该主张；只保存 `permission_group` 未定义检索前强制过滤；citation count 会奖励
   无关引用，不能作为质量指标。

### 7.4 环境型内容的隐含前置
