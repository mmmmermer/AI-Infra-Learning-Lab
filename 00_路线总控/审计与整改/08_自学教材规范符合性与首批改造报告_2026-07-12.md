## 一、执行结论

当前路线已经从“资料齐全、结构较完整”推进到“部分章节具有可追踪的自学反馈闭环”，但
全库仍只能判定为**部分符合自学教材要求**，不能整体标记为 `reviewed` 或“可直接无人指导
自学”。最主要的原因不是外链不足，而是教学完整度仍不均匀：有些章节已有概念、worked
example、反例、独立练习和验收；另一些章节仍主要承担资料索引、字段字典、工程说明或
设计记录的职责。

首批改造取得了四项实质进展：

1. 五份教材都显式区分了 `instructional`、`workbook`、`reference`、`design-note` 和
   `appendix` 中适用的内容类型，避免把设计稿或资料页冒充教学正文。
2. F00、F01、F03 和 M03 增加了固定输入、中间状态、可复算结果、失败反例、独立变式与
   可判定标准；M09 增加了 `observe -> break -> diagnose -> fix -> verify` 的实验路径。
3. 金融章节把折现、VaR/ES、协方差、相关性压力、Sharpe 频率和组合治理从名词解释推进到
   数值推演，并明确了不能从教学样本推出真实投资结论。
4. M03 把抽象 embedding 连接到具体向量，同时把 RAG 权限边界收紧为检索前过滤；M09
   明确区分已验证 reference、学习者本人复现和仍未实现的进阶设计。

但下列证据仍然缺失，因而阻止全库升级：

- 没有目标学习者试学记录，不能证明章节在无人讲解时真的可学。
- 没有前测/后测、未见迁移任务汇总和延迟保持数据，不能证明学习效果与保持效果。
- 答案隔离尚未全库完成，M05 仍是最明显的高优先级对象，F00、M09 等也需继续抽查。
- 发布态 HTML 尚未完成 WCAG 相关的 axe-core/Pa11y、键盘、放大和读屏验证。
- 22 份现行教材尚未全部按内容类型重分类；schema 1.3 已分出首批 40 章，其余 162 章仍
  明确为 `unclassified`，不能据此给出全库教学完成率。
- 六个显式 `runnable` Python 围栏已按当前标记实跑通过，但尚未建立带哈希允许清单的
  沙箱/执行器；M03 的浮点精确断言已修复为容差比较。

### 2.2 证据等级

为避免把“写进正文”误写成“已经学会”，本报告使用以下证据等级：

| 等级 | 含义 | 能证明什么 | 不能证明什么 |
|---|---|---|---|
| E1 静态教学证据 | 正文中存在目标、解释、示例、反例、练习和验收 | 教学环节已经写明 | 内容能运行或学习者能独立完成 |
| E2 确定性复算证据 | 固定输入可由代码或手算复现，结果与正文一致 | 局部计算和代码语义可核对 | 端到端实验与真实场景有效性 |
| E3 reference 运行证据 | 锁定环境中的参考实现或实验已运行并留存 artifact | 给定环境下 reference 可执行 | 学习者本人完成或迁移成功 |
| E4 学习者复现证据 | 目标学习者独立执行并保存过程、错误和产物 | 无人讲解路径具有初步可学性 | 学习效果会长期保持 |
| E5 学习效果证据 | 前后测、未见迁移和延迟保持有记录 | 学习增益、迁移与保持 | 对所有人群和环境普遍有效 |

当前首批五模块主要达到 E1-E2；M09 的既有 E09-01/02/03 reference 达到 E3，但新增章节
中的学习者操作仍停留在 E1，不能借用 reference 状态冒充 E4。全库没有 E4-E5 汇总证据。

## 三、公开原则到仓库检查项的映射

自学教材没有一个可以仅凭 Markdown 扫描就宣告通过的单一标准。本库采用多个公开框架中
与当前任务相关的原则，并把它们转换为可审计的本地检查项。

| 外部原则 | 本库采用的含义 | 仓库检查项 | 当前证据与边界 |
|---|---|---|---|
| [Quality Matters Higher Education Rubric](https://www.qualitymatters.org/qa-resources/rubric-standards/higher-ed-rubric) | 目标、评测、材料、活动和技术相互对齐 | 可观察目标；练习覆盖目标；产物、阈值或 Rubric；技术只在服务目标时使用 | 07 契约已定义闭环；语义对齐仍需人工复核；没有 QM 官方评审 |
| [IES/WWC 学习实践指南](https://ies.ed.gov/ncee/wwc/PracticeGuide/1) | worked example 与问题求解交替；连接抽象与具体表示；用测验促进提取；安排间隔复习 | 固定输入和中间状态；同构练习；条件变化的迁移题；后续章节混入旧概念；延迟检查 | 首批已补示范和迁移题；跨章提取与延迟保持尚无系统证据 |
| [CAST UDL Guidelines](https://udlguidelines.cast.org/) | 提供多种参与、表征和行动/表达方式，并明确目标、挑战与支持 | 文字、数值、表格、代码、真实观察；可选产物；故障恢复；核心与进阶分层 | 首批表征方式增加；尚未完成全库学习障碍、替代路径和目标人群试学检查；不声称 UDL 合规 |
| [Open Textbook Library Review Rubric](https://open.umn.edu/opentextbooks/reviews/rubric) | 检查完整性、准确性、清晰度、一致性、模块化、组织、界面与可访问性 | 概念与来源核对；统一术语；章级闭环；内容类型；可检索导航；章长和重复候选 | 05/06/07 已建立检查路径；不同模块完整度仍不均匀 |
| [WCAG 2.2](https://www.w3.org/TR/WCAG22/) | 对最终发布形态检查可感知、可操作、可理解和兼容性 | 图表等价文本；颜色非唯一编码；键盘；放大；读屏；公式 MathML；axe-core/Pa11y | 原始 Markdown 只能发现候选；当前没有发布态一致性证据，不能声明 WCAG conformance |

外部页面可访问、域名权威或工具零报错，都不等于教材满足这些原则。每一项结论必须落到
仓库中的目标、文本、代码、测试、artifact 或学习者记录。

## 四、全库基线与解释限制

06 报告中的章级分析器快照为：22 份现行教材、202 个可识别章节；显式目标 150 章、显式
示例 48 章、反例/常见错误 123 章、练习 111 章、验收 128 章、来源小节 28 章。另有 139 章
同时具有显式目标和至少一种评测证据，11 章有目标但没有显式练习/验收，40 章两者都未
显式标记。

这些数字是**首批改造前使用 schema 1.2 得到的候选排序快照**，不能解释为：

- “只有 48 章有任何示例”；同义标题或未显式标记的示例可能没有被识别。
- “168 章没有依据”；正文可能通过模块级索引间接引用来源。
- “有练习标题就具备有效评测”；题面可能紧邻答案，或没有未见输入和可判定标准。
- “202 章都是教学正文”；来源索引、学习顺序、接口参考和设计说明仍混在统计范围内。

因此本报告不计算“规范符合率”。schema 1.3 已在保留上述兼容视图的同时，完成首批章级
主类型试点：

| 当前章级主类型 | 章节数 | 当前解释 |
|---|---:|---|
| `instructional` | 33 | 首批五本中明确承担首次教学的章节 |
| `design-note` | 5 | task、P03 接口或迁移路线 |
| `reference` | 1 | M03 外部资料与主张映射 |
| `appendix` | 1 | M03 学习顺序和范围导航 |
| `unclassified` | 162 | 其余 17 本尚未迁移，不猜测类型 |

33 个 `instructional` 章节的显式标题证据为：目标 33、示例 16、反例 26、练习 24、
验收 29、章内来源 10。M09 的 workbook 是章内混合小节，当前章级统计不单列。缺标题只生成
人工复核候选，有标题也不证明语义对齐、答案隔离或学习有效性。

## 五、首批五模块逐项复核

### 5.1 F00 金融市场与资产基础

#### 改造前问题

- 利率、债券和数据来源章节更接近概念清单、字段字典与网址汇编。
- “利率上升、价格下降”缺少固定输入、中间计算和期限敏感性的具体表示。
- 跨资产直接对价格列做 `pct_change()` 会遗漏股息、票息、分配、期货换月和期权结算。
- 宏观数据只看 `observation_date`，没有形成 release、availability、vintage 的防泄漏不变量。
- 数据来源“权威”与“适合当前问题、时间可用、许可可说明”没有被分开。

#### 已经完成的改造

- 显式标记第 1-5 章为 `instructional`，第 6 章的未实现接口为 `design-note`，来源映射为
  `reference`，导航内容为 `appendix`。
- 增加一年和五年零息现金流在 5%/6% 要求收益率下的 worked example，展示公式、输入、
  中间结果和适用边界。
- 增加信用利差的拆分直觉，并明确“2 个百分点信用利差”不能直接解释为“2% 违约概率”。
- 增加跨资产现金流表、total return 最小公式、连续期货换月与回溯调整边界。
- 增加 point-in-time 反例、`availability_time <= decision_time` 不变量、来源选择示例、
  数据卡和使用不同输入的独立练习。
- 增加 total return、Treasury、期货、adjusted 字段、SEC 与 FRED 的正文来源映射。

#### 本地验证证据

- E1：第 4、5 章均已有预测问题、worked example、反例、独立练习和检查标准。
- E2：使用 Python 3.13.2 复算，`100/1.05=95.238095`、`100/1.06=94.339623`；五年值为
  `78.352617` 和 `74.725817`，与正文一致。
- E1：独立练习改用两年期 4%/5% 输入，避免直接复制 worked example 的完整答案。

#### 仍缺什么

- 尚无学习者提交的数据卡、时间线和两年期折现实验记录。
- point-in-time 断言在 F00 只是验收不变量，实际字段构造和端到端数据实验仍依赖 F02/GF。
- 票息、应计利息、久期、凸性和曲线折现仍应在 F05 验证，不能由零息示例替代。
- 06 报告已指出 F00 其他练习仍有答案靠近风险，需要继续做全模块答案隔离抽查。
- 数据供应方许可、再分发限制和实际下载 provenance 尚未形成统一台账。

**当前判定**：`部分符合，达到 E1-E2；不能标记为学习者已完成。`

### 5.2 F01 概率统计与数学基础

#### 改造前问题

- VaR 章节主要依赖 `quantile()` 调用，没有展示排序、秩、插值、尾部集合和符号口径。
- 小样本 nearest-rank 与 pandas 默认线性插值可能不同，但原文不足以解释结果差异。
- 组合方差只有矩阵公式和最终代码，缺少三个贡献项的逐步展开。
- 权重与协方差矩阵的资产顺序错位不会触发形状错误，原章节缺少语义回归测试。
- IID 标准误和样本协方差的适用边界说明不足。

#### 已经完成的改造

- 增加 10 个固定收益样本的 nearest-rank VaR/ES 手算和 pandas 线性插值对照。
- 增加“符号看似正确但尾部方向取反”的反例、尾部诊断字段和 12 个未见样本练习。
- 增加固定协方差矩阵的三部分方差贡献、组合波动手算、标签化权重修复和条件变化迁移题。
- 新增 Python 代码块使用 `textbook-code: role=runnable` 标记。
- 增加 IID 标准误、HAC/Newey-West、VaR/ES 和 shrinkage 的来源与使用边界。

#### 本地验证证据

- E2：固定 10 样本的 pandas 5% 线性插值分位数复算为 `-0.0575`，对应正数损失 VaR
  `0.0575`；进入尾部的样本只有 `-0.08`，教学 ES 为 `0.08`。
- E2：协方差矩阵 `[[0.0004, 0.0001], [0.0001, 0.0009]]` 与权重 `[0.6, 0.4]`
  得到方差 `0.000336`、波动率 `0.018330303`，与正文一致。
- E1：独立练习要求输出排序样本、尾部集合、分位数方法、符号约定和资产顺序，不接受只贴
  最终 `numpy` 数值。

#### 仍缺什么

- 没有学习者对 12 样本 VaR/ES 和 50/50 权重迁移题的实际提交。
- HAC 和 shrinkage 当前只建立入口与边界，没有形成可运行的比较实验。
- 分布、尾部和置信区间仍缺可复算图表及等价文本；这也是 06 报告列出的图解优先项。
- 新增围栏已标角色，但全文件其他代码块尚未全部完成角色分类和允许清单执行。
- 没有跨章间隔提取和延迟保持证据。

**当前判定**：`部分符合，局部达到 E1-E2；统计推断进阶部分仍是边界说明。`

### 5.3 F03 投资组合与风险管理

#### 改造前问题

- 组合收益公式没有充分强调权重必须在收益区间开始前确定，也没有揭示常数权重代码隐含再平衡。
- 相关性主要作为定义出现，缺少“相关性上升导致分散化减弱”的固定压力情景。
- Sharpe 缺少日频/月频、年化因子和无风险利率转换的可复算示例。
- 风险收益曲线容易停留在画图或“找最佳点”，没有连接到限额、动作、监控和复核。

#### 已经完成的改造

- 增加 `w_(t-1)` 权重时点、60/40 买入持有后的 0.622642 权重漂移和前视偏差反例。
- 增加 `cov_AB=0.0005` 的相关性压力 worked example，以及资产顺序错位反例和回归要求。
- 增加日均超额收益 `0.0005`、日波动 `0.01` 的简化年化 Sharpe 示例，并明确
  `sqrt(252)` 和年化无风险利率日化的假设。
- 增加三个权重点的风险收益表、基础/压力情景独立练习，以及
  “识别 -> 度量 -> 限额 -> 动作 -> 监控 -> 复核”的最小治理闭环。
- 增加风险收益、组合理论、再平衡、协方差和 ES/VaR 的正文来源映射。

#### 本地验证证据

- E2：压力协方差下组合方差复算为 `0.000528`，波动率为 `0.022978251`，即约 2.298%。
- E2：基础协方差下 50/50 组合波动率为 `0.019364917`，即约 1.936%。
- E2：日频 Sharpe `0.05 * sqrt(252)` 复算为 `0.793725393`，与正文约 0.794 一致。
- E1：验收要求协方差增加时组合方差不得下降，标签重排后结果保持不变，并禁止把压力情景
  写成概率预测或投资建议。

#### 仍缺什么

- 全部数值仍来自合成教学参数，不是经过样本外验证的市场风险模型。
- 压力值的选择、限额和动作尚无真实治理场景校准，不能写成生产风险管理制度。
- 组合风险曲线仍缺由可复算脚本生成的正式图、来源说明和等价文本。
- 没有学习者完成五个权重点、双情景表和越限处置记录的证据。
- 交易成本、约束、估计误差和权重漂移的端到端回测仍需 GF 实验承接。

**当前判定**：`部分符合，局部达到 E1-E2；不构成投资模型有效性证明。`

### 5.4 M03 RAG 工程

#### 改造前问题

- embedding 章节能够描述链路和排序适配器，但 score 的来源仍是黑箱，抽象概念没有连接到
  可手算的具体表示。
- 零向量、并列排序和词表外查询缺少明确失败行为，容易产生“有分数就是有证据”的错觉。
- 权限过滤曾允许把检索后过滤与检索前过滤并列为第一轮方案；无权限 chunk 一旦进入 prompt，
  最终删除引用仍不能消除泄漏风险。
- P03 接入仍属于目标架构，不能冒充已经完成的端到端 RAG 服务。

#### 已经完成的改造

- 增加三维固定关键词向量，把点积、范数、cosine score 和排序全部展开。
- 增加离线 Python 示例、零向量反例、词表外失败状态、稳定次级排序和未见 query 迁移题。
- 明确教学关键词向量不代表真实 embedding 模型，模型/维度/距离函数必须进入版本记录。
- 将权限主路径收紧为 metadata 检索前过滤；检索后检查只保留为纵深防御和故意失败对照。
- 显式把第 8 章 P03 接口标为 `design-note`，不再把接口草图混作已实现实验。

#### 本地验证证据

- E2：Python 3.13.2 复算的三条 score 为 `0.9999999999999998`、`0.4999999999999999`
  和 `0.0`；教学解释中的 1.0、0.5、0.0 在数值容差内正确。
- E2：词表外查询产生零向量，并按正文设计触发 `ValueError("zero_vector")`。
- E1：权限章节明确要求无权限 chunk 不进入候选集合，实验记录同时检查候选集合和最终回答。

#### 已发现并修复的验证问题

初版 runnable 示例曾使用：

```python
assert ranked == [("c1", 1.0), ("c3", 0.5), ("c2", 0.0)]
```

Python 3.13.2 实际浮点结果不是精确的 1.0/0.5，因此该断言会失败。当前正文已经改为逐项
精确比较 chunk ID，并对 score 使用 `math.isclose(..., abs_tol=1e-12)`。使用 Python 3.13.2
按正文逻辑重跑后返回码为 0，排序为 `c1 -> c3 -> c2`。这证明当前局部示例通过，不替代
真实 embedding 端到端实验。随后又按同一显式标记执行全库 6 个 `runnable` Python 围栏，
全部通过；由于执行器没有沙箱，进入自动门禁前仍需固定代码块哈希允许清单。

- 缺真实 embedding 模型、固定 corpus、查询集、expected source 和检索质量/成本联合实验。
- 缺权限注入测试的实际 artifact；概念规则正确不等于向量库、缓存和 prompt 链路均不会泄漏。

- 没有目标学习者完成未见 query、零向量诊断和权限实验的记录。

**当前判定**：`部分符合；解释、安全边界和局部 runnable 已增强，端到端与学习者证据仍缺失。`

## 九、复核门槛

### 单章可标记 `reviewed` 的最低条件

- 目标、边界、worked example、反例、独立练习和验收语义完整。
- 练习至少保留未见输入、迁移题或隐藏测试，不在题面直接暴露完整答案。
- 可执行代码在匹配环境通过；故意失败示例按预期失败并有恢复路径。
- 公式、版本、统计、安全和金融主张有正文级来源与边界。
- reference 状态与学习者本人完成状态没有混用。
- 至少有一次目标学习者试学证据；否则最多标记为 `reference verified / learner pending`。

### 全库可标记“适合直接自学”的附加条件

- 核心路径全部完成内容类型分类和目标-练习-验收人工对齐。
- 核心实验有跨模块端到端 reference，且有学习者独立复现样本。
- 高风险练习完成答案隔离，状态由单一 manifest 生成。
- 有前后测、未见迁移和至少一次延迟保持检查。
- 发布态完成适用的可访问性自动与人工检查。
- 来源、许可、版本和数据 provenance 台账达到可发布要求。

当前仓库尚未达到这一全库门槛。
