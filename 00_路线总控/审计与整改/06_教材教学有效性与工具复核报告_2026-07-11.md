## 一、范围校正与结论

第一层质量检查把 23 个教材文件作为同一组统计。章节级复核确认，其中实际是：

- 22 份现行教材。
- 1 份位于 `M05/99_归档` 的停用旧稿。
- 现行教材共有 202 个可识别章节；21 份按 H2 章节识别，M05 的 13 个独立 H1 章节由
  兼容索引聚合为一本逻辑教材。
- 截至 2026-07-12 首批自学教材改造后，现行教材包含 1500 组代码围栏和 318 个表格、
  图片或 Mermaid 项；2026-07-11 快照为 1465/309。

因此后续成熟度、来源和教学闭环统计必须排除归档旧稿。简单的全文件关键词扫描也不能
继续承担“缺目标、缺示例、缺验收”的结论：M01/M02 原本都存在学习检查，F01-F03 也有
最小代码，只是标题词汇不同或证据不够机器可判定。

本轮真正确认的问题不是“教材空白”，而是四类闭环不足：

1. 概念讲了，但没有从公式、代码、实验到可判定结果。
2. 实验能运行，但正文契约和 reference 实现不一致。
3. 有结果表，但统计单位、不确定性或负载模型不足以支撑结论。
4. 有资料索引，但正文主张没有映射到具体权威来源。

## 二、教材还值得检查的方向

| 检查方向 | 核心问题 | 主要证据 | 自动化边界 |
|---|---|---|---|
| 目标与评测对齐 | 学习目标是否被练习和验收真正覆盖？ | 目标 ID、练习 ID、产物、阈值/Rubric | 结构可查，语义对齐需人工复核 |
| 前置与学习顺序 | 是否跳过必要知识、环境或实验？ | 前置诊断、依赖图、失败入口 | 可查链接和状态，不能自动判断认知难度 |
| 概念解释 | 是否同时有定义、直觉、机制和边界？ | 正例、反例、适用范围、常见误解 | 关键词只能生成候选 |
| 示例递进 | 是否从最小例子递进到跨模块项目？ | 固定输入、中间状态、输出、错误修复 | 代码角色未标记前不能全库执行 |
| 练习可判定性 | 是否能客观判断“做对了”？ | 命令、预期结构、已知答案、容差 | 清单存在不等于验收充分 |
| 调试与失败恢复 | 学习者出错后能否定位和回归？ | 最小复现、错误分类、回归测试 | 需要真实失败记录 |
| 代码与实验闭环 | 文档步骤是否真的可运行？ | 锁文件、测试、Compose/kind/GPU 证据 | 代码块工具不能替代端到端环境 |
| API/数据契约 | 字段、单位、状态和错误是否跨文档一致？ | schema、contract tests、reference code | 事实冲突可阻断 |
| 安全与隐私 | 权限、Secret、日志和输入边界是否正确？ | 泄漏测试、Gitleaks、owner/tenant 过滤 | 工具无发现不等于架构安全 |
| 统计有效性 | 实验单位、重复、效应量和区间是否正确？ | 多 seed、配对差值、CI、失败/排除规则 | 单次运行不能变成科研结论 |
| 性能测量语义 | 闭环/开放环、吞吐、尾延迟是否混淆？ | offered load、actual start、backlog、阶段边界 | 压测工具默认值不能替代协议 |
| 金融计算 | 时间、现金流、复利、标签和回测是否使用正确口径？ | 已知答案、有限差分、PIT 时间、泄漏断言 | 必须由领域测试和人工共同核对 |
| 版本与生命周期 | 当前基线、兼容下界和历史复现是否混用？ | 锁文件、官方 EOL、`last_verified` | “不是最新版”不等于错误 |
| 来源与许可 | 关键主张是否可追溯，资料能否合法使用？ | claim -> source -> locator、license | URL 可访问不等于支持该主张 |
| 认知负荷 | 单章是否过长、一次引入过多概念？ | 章长、围栏密度、章末压缩总结 | 不套用英文可读性公式 |
| 图表与可访问性 | 动态流程、时间线和数值关系是否可见？ | Mermaid、可复算曲线、替代文字、单位 | 图多不等于教得好 |
| 重复与术语漂移 | 重复是螺旋复习还是维护债务？ | 近重复候选、术语表、字段映射 | 不自动删除重复段落 |
| 状态与成果归属 | reference 是否被误写为本人完成？ | 多维状态、运行记录、学习者 artifact | 状态失真应阻断 |

上述检查规则已固化到 `07_教材章节质量契约.md`，后续可再升级为 Schema，而不是依赖
模型猜测章节语义。

### 3.3 明确不采用为门禁

- `textstat 0.7.13` 没有中文公式，不能把英文 Flesch/SMOG 套到中文技术教材。
- `cntext 2.2.1` 发布包的中文可读性分支存在词数累计错误，并继续使用英文公式。
- `AlphaReadabilityChinese` 缺少可审计源码和许可证。
- FActScore 面向英文知识库的生成事实精度；RefChecker 已归档。
- 句向量、NLI 和 LLM 事实检查只能给人工复核候选，不能证明概念正确。
- 已归档的中文 `textlint` preset 不再引入；避免和 Vale 建两套重复术语基础设施。

## 四、本轮已经完成的教材补强

### 4.1 M11：从现象补到可复核结论

新增最小统计推断章节，明确：

- task 行不是独立实验单位，RQ01 第一轮使用 seed/run。
- common random numbers 和 seed 级配对差值。
- 绝对/相对效应量、配对 t 区间和 percentile bootstrap 边界。
- P95/P99 的 run 内样本量与 run 间不确定性。
- primary/secondary/exploratory 和多重比较边界。
- 读取 RQ01 artifact 的可复算代码、已知结果和 `1e-3` 验收容差。

### 4.2 M08：修正压测负载模型

明确 Locust `HttpUser + between` 是闭环用户模型，不是固定 RPS；新增：

- closed-loop、open-loop 和 fixed replay 的区别。
- offered load、actual start 和分层 achieved throughput。
- warm-up、steady-state、cool-down。
- coordinated omission、压测机落后和饱和点。
- 阶梯负载的唯一判定区间练习。

k6、Vegeta 和 wrk2 只作为开放环候选工具，没有被写成已执行实验。

### 4.3 F05 与 F04：补全金融计算和随机误差

F05 已从“价格上下冲击直觉”补到：

- Macaulay duration、modified duration 和解析 convexity。
- 有限差分交叉验证与二阶 `Delta P/P` 近似。
- 小冲击近似和完整重估的确定性数值表。
- 付息频率、复利口径、clean/dirty price 和 accrued interest 边界。
- GF05-01/GF05-02 与已验证 `finance_reference` 的映射。

F04 已补：

- `q=0`、连续复利和风险中性漂移 `r`，明确它不是实际收益预测。
- 局部 RNG、Monte Carlo 标准误、近似 95% CI 和多 seed 重复。
- Black-Scholes 同模型教学 oracle、put-call parity 已知答案。
- `1/sqrt(N)` 收敛检查和可判定练习。

### 4.4 F02 与 F07：补时间语义和泄漏边界

F02 已补跨资产收益率可比性、简单/对数收益聚合、UTC/交易所时区、DST、market
session date，以及 observation/release/availability/vintage 时间的 point-in-time 边界。

F07 已补特征构造后的 NaN 清洗契约、标签信息区间、重叠 horizon、非等间隔交易日，
并明确 `TimeSeriesSplit(gap=horizon)` 不是自动防泄漏开关。阈值只能在 validation 选择，
final test 在冻结方案后使用一次。

### 4.5 M01、M02 与 M07：修正工程契约

M01 已补教学简化字段到 E01/P01 规范字段的映射、`dict -> Task -> asdict -> JSON`、
pytest 最小复现和回归流程，以及命令、固定输入、预期结果组成的验收。E01-04 仍明确是
待创建的学习者任务，不是已有 reference。

M02 已对齐 E02 的 `estimated_duration_ms`、服务端 UTC `created_at`、TaskStatus 和稳定错误
口径；改用 Pydantic v2 `model_dump()`，补 `extra="forbid"`、coercion/strict 边界、
BackgroundTasks 与持久队列的区别，以及“UUID 不是授权”。当前 reference 仍诚实标记为
route + repository，service 是下一步。

M07 的 Docker 示例已从 Python 3.11 对齐到已验证的 Python 3.13.2 镜像与 digest，增加
非 root UID 验收，并明确 digest 更新必须伴随重建和测试。

### 4.6 F06：从名词清单补到可执行偏差反例

- 修正“close 生成信号 + shift 一行”仍隐含同收盘价成交的问题，统一为
  `close t 决策 -> open t+1 成交 -> open-to-close t+1 收益`。
- 教材、GF06-01/02/03 和 GF08-02 同步记录 decision/fill/return interval。
- 增加 point-in-time universe 对 survivor-only 的固定反例，已知累计收益为
  `-0.6149` 与 `0.0303`。
- 增加 200 个零均值候选的 best-of-many 反例，固定 Python 3.13 结果为训练 Sharpe
  `2.218`、封存测试 `-0.145`；明确数值只是回归夹具，不是理论结论。
- `backtest_task` 增加 universe、candidate count、test access 和 evidence artifact。

### 4.7 M10：统一推理指标、容量和压测契约

- 用 `t_submit/t_backend_send/t_first_content/t_last_content` 区分 outer queue、backend
  TTFT 和 e2e TTFT，首内容排除 header/空 chunk。
- TPOT 固定为 `(t_last - t_first)/(N - 1)`，单 token 请求为 null；ITL 与请求级 TPOT 分开。
- 分开逐请求生成速率、input/output token throughput、requests/s 和 SLO goodput。
- 增加 prefill/decode、三类 batching、低负载/饱和/过载区间。
- 增加 `2 * layers * kv_heads * head_dim * bytes` 公式；32/8/128/BF16 已知答案为
  128 KiB/token、8192 token = 1 GiB，并说明 GQA、block 和 runtime overhead。
- 对齐 E10 reference 的 5/0.05/2.0 合成 profile；E10-01/02 是 verified reference，
  E10-03 仍是 blocked/unverified，AIPerf 只是后续真实实验候选。

### 4.8 金融来源、组合权重和 artifact 契约

F00、F01、F03、F06、F08 已全部增加正文 URL 和“主张/依据/简化/不能推出什么”映射。
同时修复：

- F00 price return/total return、跨资产现金流和期货换月口径。
