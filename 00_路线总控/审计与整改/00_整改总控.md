## 状态模型

单一“已完成”标签不能同时表达教材、代码、实验和本人完成情况。模块、实验和已复核文件分别记录：

| 维度 | 可选值 |
|---|---|
| 盘点状态 | `unreviewed` / `rule-reviewed` / `manual-reviewed`；未复核时其他结论保持未知 |
| 内容成熟度 | `planned` / `draft` / `content-reviewed`；`unassessed` 只用于覆盖清单 |
| 教学就绪 | `not-assessed` / `partial` / `instructional-ready` |
| 实现状态 | `not-applicable` / `absent` / `partial` / `runnable-task` / `executable` |
| 验证状态 | `unverified` / `verified` |
| 成果归属 | `reference` / `learner-reproduced` / `learner-owned` |
| 学习者验证 | `not-evaluated` / `pending` / `learner-validated` |
| 风险 | `blocker` / `high` / `medium` / `low`；`unknown` 只用于尚未复核的覆盖清单 |

只有同时具备正确教材、可执行实现和验收证据的实验，才能显示为 `verified`。AI 生成或整理的参考答案默认属于 `reference`。
`content-reviewed` 不要求学习者试学，`learner-validated` 也不能掩盖内容错误；未注明状态轴时，
不再使用含混的“教材已验证”。
