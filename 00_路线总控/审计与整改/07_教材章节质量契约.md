# 教材章节质量契约

> 状态：`content-reviewed working contract`
> 适用范围：`10_学习模块` 中 22 份现行教材；归档教材只做安全和断链检查。
> 原则：自动化检查结构和证据，人工复核概念、教学有效性与领域结论。

本契约使用三个相互独立的教材状态轴：

| 状态轴 | 回答的问题 | 证据来源 |
|---|---|---|
| `content-reviewed` | 作者侧内容是否正确、完整、可执行并达到发布要求 | 正文、代码、reference、测试、来源和人工审稿 |
| `instructional-ready` | 章节是否具备目标、示例、练习、反馈和清理的作者侧自学闭环 | 章节结构、任务、答案隔离、排错和人工教学审稿 |
| `learner-validated` | 目标学习者能否据此独立学习并迁移 | 试学、学习者产物、前后测、迁移或保持数据 |

三个状态轴不能相互自动升级。`content-reviewed` 不等于教学闭环，`instructional-ready` 不要求
学习者本人试学，`learner-validated` 也不能反过来掩盖事实、代码、安全或金融计算错误。未明确
状态轴时，不使用含混的“已验证教材”表述。

## 一、模块级入口

每份现行教材应在正文前部明确：

| 字段 | 最低要求 |
|---|---|
| 学习边界 | 写清第一轮要学、暂不深入和不能据此声称什么 |
| 前置诊断 | 给出知识、环境和可执行命令，而不只列前置模块名 |
| 学习产物 | 指向具体文件、实验记录、测试输出或报告 |
| 完成口径 | 区分 verified reference、learner reproduction 和 learner-owned |
| 版本基线 | 标明当前验证版本、兼容下界或历史复现版本的角色 |
| 学习顺序 | 核心路径和选修支线分开，超长模块给预计时间与退出点 |
| 内容类型 | 区分教学正文、工程手册、设计说明、附录和答案，不把它们混算为同类章节 |

模块入口不要求把每章内容重复一遍，但必须让学习者在开始前知道“做什么、产出什么、
怎样算完成”。

## 二、章节级闭环

每章至少形成下面的可追踪闭环。小节标题可以按领域调整，但语义角色应明确。

| 角色 | 必须回答的问题 | 合格证据 |
|---|---|---|
| 目标 | 学完后能做出什么可观察行为？ | 使用解释、实现、比较、诊断、验证等动作词 |
| 前置 | 需要已经会什么或准备什么？ | 链接到具体章节、命令、数据或环境检查 |
| 概念 | 定义、直觉和机制分别是什么？ | 至少有一个边界或不适用场景 |
| worked example | 正确路径如何一步步完成？ | 固定输入、中间状态、输出和解释 |
| 反例 | 初学者最可能怎么错？ | 错误现象、原因、修复和回归检查 |
| 练习 | 学习者要独立完成什么？ | 不直接暴露完整答案；写明产物位置 |
| 验收 | 如何判断完成，而不是“看懂了”？ | 命令、固定输入、预期结构/数值、容差或 Rubric |
| 来源 | 关键定义、公式、版本和强断言依据什么？ | 具体 URL/引用和 locator，不只链接模块索引 |

“本章解决什么问题”可以承担目标角色，“小练习 + 检查标准”可以承担评测角色。自动化
只检查显式证据，不根据同义词缺失直接判定正文错误。

### 2.1 内容类型必须显式

同一知识库可以保留不同用途的材料，但每份材料必须明确自己的教学承诺：

| 类型 | 用途 | 必须具备 | 不得冒充 |
|---|---|---|---|
| `instructional` | 学习者第一次建立概念和能力 | 机制解释、worked example、独立练习和反馈 | 只列字段、命令或链接的资料页 |
| `workbook` | 学习者动手练习 | 任务、starter、固定输入、验收和失败恢复 | 已完成 reference 或答案 |
| `reference` | 完成后查阅事实、接口和实现 | 可检索结构、版本、来源和边界 | 零基础教学正文 |
| `design-note` | 记录尚未实现的架构和取舍 | 假设、状态、风险和激活条件 | 已执行实验或可运行案例 |
| `solution` | 完成后核对答案 | 对应练习 ID、解释和替代解法 | 学习者未作答前的默认正文 |
| `appendix` | 来源、学习顺序、术语或状态说明 | 清晰导航和维护信息 | 核心概念章节 |

如果一个文件混合多种类型，至少在小节标题或机器可读元数据中标明角色。章节分析报告必须
把 `appendix/reference/design-note/solution` 与 `instructional` 分开统计，避免用附录数量稀释
教学正文缺陷，也避免把接口完整度误写成教学完整度。

当前章级分析器使用下面的稳定语法。文档默认值放在首个章节之前：

```html
<!-- textbook-content: default=instructional -->
```

章节覆盖值必须放在对应章标题之后、下一个小节标题之前：

```markdown
## 第 8 章：目标接口

<!-- textbook-content: type=design-note -->
```

允许类型只有本节表中的六种。未知值、同章冲突值或缺少显式标记都归为 `unclassified`，
不能静默继承默认值来掩盖错误。当前统计粒度是章级；同章混合的 workbook/参考小节需在人读
类型表中说明，迁移到小节级 Schema 前不单独计入章级类型数量。

### 2.2 自学反馈闭环

面向无人实时讲解的章节，还必须满足下面要求：

1. 进入新概念前给一个预测题、前置自测或具体问题情境，激活已有知识。
2. 抽象定义必须连接至少一个具体数值、状态变化、输入输出或真实对象观察。
3. worked example 展示中间状态和解释；随后安排同构练习，再安排至少一个条件变化的迁移题。
4. 脚手架应逐步撤除：`完整示范 -> 部分 TODO -> 独立实现 -> 新场景迁移`，不能一直照抄完整代码。
5. 反例必须尽量包含错误现象、根因、修复和回归证据，而不只是“不要这样做”的提醒。
6. 章内必须给形成性反馈；不能连续多个核心章节只阅读，最后才在模块末尾第一次验收。
7. 反馈可以是已知答案、属性/不变量、测试、容差、Rubric 或诊断树；“能理解”“能说明”不能
   单独作为唯一验收。
8. 章末安排短提取练习，并在后续章节混入旧概念变式；仅重复阅读不算复习设计。
9. 环境型章节提供 `observe -> break -> diagnose -> fix -> verify` 路径和安全清理步骤。
10. 核心主线与高风险进阶内容分层，给预计时间、暂停点和替代路径，避免一次引入生产级全部复杂度。

## 三、目标与评测映射

进入结构化元数据阶段后，每个高风险章节使用稳定 ID：

```yaml
objectives:
  - id: M11-O5.6-1
    behavior: 识别实验单位并解释伪重复
    condition: 给定 seed/run/task 三层数据
    criterion: 正确把 seed/run 设为第一轮推断单位
    level: analyze
exercises:
  - id: M11-E5.6-1
    covers: [M11-O5.6-1]
    evidence: 配对差值表、95% CI 和边界说明
    acceptance: seed_count=30 且数值误差不超过 1e-3
```

确定性检查只验证：

1. ID 唯一且引用存在。
2. 每个必修目标至少被一个练习或验收覆盖。
3. 练习声明了学习证据和判定标准。
4. 评测要求的认知动作不低于目标动作。

Bloom 层级分类、目标和练习的语义一致性仍由人工复核；句向量只允许给候选排序。

### 评测完整性与答案隔离

`worked example`、`starter`、`exercise` 和 `solution` 必须标明角色。独立练习不能在题面后
直接给出完整实现、完整图或标准结论；至少保留一项未见输入、迁移题或隐藏测试。完整答案
放到完成后对照的 reference 区，学习者产物单独保存。机器发现“练习”标题不等于评测有效，
覆盖率统计必须注明可能存在答案泄漏。

## 四、代码块角色

当前教材同时包含完整程序、局部片段、伪代码、错误示例和预期输出。没有角色标记前，
不得把全部围栏交给执行器。新增或重写代码块时，在围栏前加入机器可读注释：

````markdown
<!-- textbook-code: role=runnable env=python-3.13 network=off -->
```python
print("verified example")
```
````

允许的 `role`：

| role | 含义 | 自动化处理 |
|---|---|---|
| `runnable` | 独立可运行示例 | 进入允许清单，可由 pytest-codeblocks 执行 |
| `fragment` | 依赖上下文的局部片段 | 只做语法或人工复核 |
| `pseudocode` | 解释流程，不承诺具体语言语义 | 不执行 |
| `counterexample` | 故意错误或危险示例 | 不执行成功断言；必须有修复说明 |
| `output` | 预期输出 | 与对应 runnable 示例绑定 |

Shell、Docker、数据库、Kubernetes 和 GPU 命令即使标为 runnable，也只能在隔离环境和
显式允许清单中运行。代码块执行器不提供沙箱。

## 五、来源与新鲜度

高风险主张应建立正文级映射：

```yaml
claims:
  - id: F05-C4-1
    statement: modified duration 给出小收益率冲击的一阶价格变化近似
    source_id: treasury-pricing
    locator: price-yield relationship
    last_verified: 2026-07-11
    review_after: 2027-01-11
```

来源优先级：

1. 标准、RFC、PEP、监管机构和项目官方文档。
2. 原始论文、正式教材和公开大学课程。
3. 高质量二级资料，用于解释而非替代原始定义。
4. 博客、岗位页和产品宣传只能校准场景，不能单独支撑公式或普遍结论。

DOI 可解析、网页可访问或域名权威，都不等于来源真正支持正文主张。公式、统计方法、
安全边界和金融口径必须人工核对 locator。

## 六、图表和可访问性

图表用于降低解释成本，不按数量考核。以下内容优先图解：

- 时间顺序、信息可用时间和数据泄漏。
- 状态机、重试、人工确认和失败恢复。
- 请求跨 API、队列、worker、数据库的链路。
- 资源拓扑、调度准入和容器网络。
- 公式变量关系和完整重估对近似的误差。

每张图必须有相邻文字解释；颜色不能是唯一编码；表头、单位和时间方向必须明确。
Mermaid 必须通过 CLI 渲染。数值曲线优先使用可复算的真实图表，不用 Mermaid 模拟。

现行 Mermaid 还必须提供 `accTitle` 和 `accDescr`；ASCII/text 流程要有等价的有序文字说明。
公式不能只依赖代码字体表达，发布态应使用带 Assistive MathML 的数学渲染或提供等价朗读。
原始 Markdown 只能做静态候选检查；生成确定性 HTML 后，再用 axe-core/Pa11y 和至少一次
键盘、放大、读屏人工检查验证 WCAG 相关行为。

## 七、状态、复现与来源许可

- 状态按范围使用明确真源：模块/实验成熟度来自模块状态表，整改任务来自 110 项矩阵及状态
  证据，逐文件清单只证明覆盖与分诊。教材不得把已有 verified reference 写成“尚未创建”，
  也不得把 reference 写成学习者已完成；任务完成也不得自动升级整本教材成熟度。
- 实验记录至少保存 argv、解释器/OS/硬件、依赖锁哈希、输入/config、源码 commit 和
  dirty 状态。dirty run 若保留为 reference，必须附 source patch/snapshot 和哈希，并在
  clean checkout/snapshot 重跑前明确降级。
- 教材正文、代码、图表、数据和第三方摘录分别记录 license、copyright、source、locator
  和 attribution。REUSE/ScanCode/SBOM 只能验证声明和发现候选，不能替作者选择许可证或
  证明已获授权。
- 资料索引补语言、字幕、登录/付费、地区限制、预计时长和离线可用性；URL 可访问不等于
  学习者可取得。

## 八、学习者证据与数据生命周期

静态审计不能证明教材有效。进入 `learner-validated` 前，核心路径至少要有一次目标学习者试学，记录
用时、卡点、错误类型、求助次数和未见迁移任务表现；成熟阶段再做前测/后测和延迟保持检查。

涉及 RAG/Agent/金融数据时，安全章节还要覆盖数据最小化、保留期、脱敏、删除请求，以及
原文、chunk、embedding、缓存、日志和 artifact 的级联删除验证。访问控制通过不等于完整
数据生命周期合规。

## 九、门禁分级

### 阻断

- 代码围栏、Markdown 结构或正式 WikiLink 破损。
- 当前版本/字段/API 与已验证 reference 存在事实冲突。
- 可执行实验缺入口、依赖、固定输入或成功判据。
- 权限、密钥、未来信息、统计单位或金融公式存在高置信错误。
- 把设计稿、reference 或单次 smoke 写成学习者完成或科研结论。

### 必须人工复核

- 目标与练习可能错配。
- 章节缺显式来源、反例、视觉或章末压缩总结。
- 近重复、长句、术语密度和认知负荷候选。
- LanguageTool、Zhlint、句向量、NLI 或 LLM 事实检查结果。

### Advisory

- 不影响语义的排版偏好。
- 403、429、超时和协议错误等未确认断链。
- 教学中有意的螺旋复用和小段代码重复。

## 十、章节状态完成定义

一个章节只有同时满足下面条件，才可标为 `content-reviewed`：

- [ ] 关键概念、适用边界和不适用场景经过人工专业复核。
- [ ] 公式、版本、统计和安全主张有正文级来源。
- [ ] 代码语义与现行接口一致，可执行块已在匹配环境验证。
- [ ] reference 与学习者本人完成状态没有混用。
- [ ] 结构、链接、围栏、术语和 Mermaid 检查通过。
- [ ] 图表读屏元数据、来源/许可和实验 provenance 已按适用范围记录。
- [ ] 人工复核确认没有用工具分数替代概念正确性判断。

一个已经 `content-reviewed` 的章节只有再满足下面条件，才可标为 `instructional-ready`：

- [ ] 目标、先修、边界、练习和验收对齐。
- [ ] 关键概念至少有直觉解释、机制说明、worked example 和机制型反例。
- [ ] 独立练习有固定输入、可判定证据和反馈/排错路径。
- [ ] 清理、章末压缩总结和下一步衔接完整。
- [ ] 独立练习不直接暴露完整答案；题目、参考答案和验收标准相互一致。

`learner-validated` 另按第八节记录，不进入前两条作者侧状态轴的通过条件。全库“适合直接
自学”仍是路径级综合结论，不能仅由某一个状态数量推出。

## 十一、自学规范依据与采用边界

本契约不是声称通过某个外部认证，而是把公开、可核对的课程与教材质量原则转成仓库内规则：

- [Quality Matters Higher Education Rubric](https://www.qualitymatters.org/qa-resources/rubric-standards/higher-ed-rubric)：
  采用目标、评测、材料、活动和技术之间的 alignment 原则；不声称完成 QM 官方评审。
- [IES/WWC: Organizing Instruction and Study to Improve Student Learning](https://ies.ed.gov/ncee/wwc/PracticeGuide/1)：
  采用间隔学习、worked example 与问题求解交替、抽象/具体表示连接和测验促进提取等建议。
- [CAST UDL Guidelines](https://udlguidelines.cast.org/)：采用多种参与、表征和行动/表达方式，
  以及目标意义、挑战和支持的明确化；不把“提供多个链接”误写成 UDL 合规。
- [Open Textbook Library Review Rubric](https://open.umn.edu/opentextbooks/reviews/rubric)：采用完整性、
  准确性、清晰度、一致性、模块化、组织、界面和可访问性等审查维度。
- [WCAG 2.2](https://www.w3.org/TR/WCAG22/)：用于发布态可访问性；原始 Markdown 静态检查不能
  单独证明 WCAG conformance。

因此本库只报告“满足哪些本地证据项、哪些仍缺失”，不使用“符合国际标准”“通过认证”等
超出证据的表述。
