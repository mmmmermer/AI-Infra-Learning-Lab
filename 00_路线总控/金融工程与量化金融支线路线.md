# 金融工程与量化金融支线路线

路线资料来源和建设优先级先看：[[00_路线总控/金融工程高质量资料与路线审计|金融工程高质量资料与路线审计]]。

## 1. 定位

金融工程支线不是 M12 的扩写，也不是投资建议路线。

它的定位是：

```text
用数学、统计、编程和金融市场知识，
理解金融数据、风险、资产定价、投资组合、衍生品和回测实验，
为金融科技、量化研究、金融 AI 场景和 P03 金融 workload 提供底层领域能力。
```

这条路线和 AI Infra 主线并行。

```text
AI Infra 主线：系统如何承载、调度、监控 AI workload。
金融工程支线：金融任务本身如何定义、计算、评估和控制风险。
交叉线：把金融任务变成 P03 可运行的 workload。
```

## 2. 为什么不能放进 M12

M12 的职责是：

```text
金融文档 RAG / Agent / 引用 / 风险提示 / P03 workload
```

金融工程的职责是：

```text
市场机制 / 概率统计 / 时间序列 / 投资组合 / 风险管理 / 衍生品定价 / 回测工程
```

两者交叉，但不是一回事。

如果把金融工程塞进 M12，会导致：

- M12 从 AI 场景模块变成金融大杂烩。
- P03 主线被投资组合、期权、回测等内容冲散。
- 学习模块再次变得像台账。
- 金融工程本身也学不扎实。

所以金融工程必须单独成支线。

## 4. 模块设计

### F00 金融市场与资产基础

目标：建立金融市场和主要资产类别的基本语言。

学习内容：

- 股票、债券、基金、衍生品。
- 市场参与者和交易机制。
- 收益率、风险、流动性、杠杆。
- 利率、期限结构、信用风险。
- 金融数据来源和数据口径。

不做：

- 投资建议。
- 高频交易细节。
- 复杂市场微观结构。

实验出口：

```text
GF00-01 收益率和波动率计算
GF00-02 股票/债券/指数数据字段对比
```

项目出口：

```text
Q01 金融数据分析与风险指标实验
```

### F01 概率统计与数学基础

目标：补金融工程所需的概率、统计、线性代数和优化基础。

学习内容：

- 随机变量、期望、方差、协方差和常见分布（当前正文已有基础）。
- 估计、置信区间、假设检验与回归诊断（计划补强；当前只覆盖局部直觉）。
- 矩阵、协方差矩阵、特征值与条件数（协方差已有；后两项计划补强）。
- 面向定价/风险的微积分、数值求根和误差控制（计划补强）。
- 约束最优化与数值诊断（计划补强；当前不计首次教学）。

不做：

- 纯数学证明大课。
- 高阶测度论。
- 复杂随机分析。

实验出口：

```text
GF01-01 收益率分布与正态性检查
GF01-02 协方差矩阵与相关性热力图
```

### F02 Python 金融数据与时间序列

目标：能用 Python 处理金融时间序列。

学习内容：

- pandas 时间索引。
- 收益率、对数收益率。
- rolling window。
- 缺失值、复权、频率转换。
- 数据来源、数据版本、point-in-time 口径和字段 lineage。
- 可视化和基本统计。
- statsmodels 时间序列入门（计划补强；当前 F02 正文尚未承担首次教学，不计入已覆盖能力）。

不做：

- 过早预测股价。
- 无验证的模型调参。
- 复杂深度学习时间序列模型。

实验出口：

```text
GF02-01 金融时间序列清洗
GF02-02 rolling volatility 与 drawdown
```

项目出口：

```text
Q01 金融数据分析与风险指标实验
```

### F03 投资组合与风险管理

目标：理解组合、风险、收益和约束之间的关系。

学习内容：

- 两资产组合收益与方差（当前正文已有首次教学）。
- Markowitz 多资产均值-方差、有效前沿与约束优化（计划补强；当前不计首次教学）。
- CAPM、beta 和 alpha（计划补强；当前不计首次教学）。
- Sharpe ratio 与其边界（当前正文已有基础）；beta/alpha 随 CAPM 章补入。
- 最大回撤、波动率、VaR。
- 两资产权重（当前已有）；多资产约束与优化器（计划补强）。

不做：

- 承诺收益。
- 实盘交易。
- 复杂组合优化器。

实验出口：

```text
GF03-01 两资产组合风险收益曲线
GF03-02 多资产组合有效前沿（实验候选；等待 F03 多资产优化主章后升级）

```

项目出口：

```text
Q02 投资组合与风险管理实验
```

### F05 固定收益与利率基础

- 关键利率和情景利率风险（计划补强；当前不计首次教学）。

- look-ahead bias。
- survivorship bias。

不做：

- 实盘交易系统。

### F07 金融机器学习与模型风险

## 5. 推荐学习顺序

金融工程支线不要一次全开。建议四阶段，并且先读教材，再按触发条件做实验。

### 阶段 A：金融数据、统计和风险直觉

```text
F00 金融市场与资产基础
-> F01 概率统计与数学基础
-> F02 Python 金融数据与时间序列
-> F03 投资组合与风险管理
```

目标：能读懂金融数据，能计算收益、波动、相关性、VaR、回撤和两资产组合风险。

合格标准：

- 能用 pandas 处理价格序列。
- 能计算 return、volatility、drawdown、historical VaR。
- 能解释权重、协方差、相关性和组合风险。
- 能解释数据频率、缺失值和时间索引。
- 能写一份简单金融数据分析记录。

触发实验：

```text
GF02-01 -> GF02-02 -> GF03-03 -> GF03-01
```

### 阶段 B：量化研究和金融 ML 工程

```text
F06 量化研究与回测工程
-> F07 金融机器学习与模型风险
```

目标：理解从数据、信号、仓位、成本、切分、baseline 到失败分析的研究闭环。

合格标准：

- 能写一个最小回测。
- 能说明交易成本、数据泄漏和过拟合风险。
- 能做 baseline/challenger 对比。
- 能写模型卡和失败样例。

### 阶段 C：定价、利率和数值计算

```text
F04 衍生品定价
-> F05 固定收益
```

目标：理解模型价格、参数假设、Monte Carlo 误差、现金流、YTM、久期和利率敏感性。

合格标准：

- 能实现 Black-Scholes 或 Monte Carlo 定价最小例子。
- 能记录 seed、simulation_count、runtime_ms。
- 能计算债券现值、YTM、久期和利率变化影响。

### 阶段 D：金融任务接入 P03

```text
F08 金融工程任务与 P03 接口
```

目标：能把金融工程任务变成可复现、可记录、可调度的 workload。

合格标准：

- 能区分 input_json、result_json、metrics、review_status。
- 能把 risk_metric_task / backtest_task / pricing_task / model_risk_task 写成字段草图。
- 能说明 M05/M06/M08/P03 分别负责什么。
