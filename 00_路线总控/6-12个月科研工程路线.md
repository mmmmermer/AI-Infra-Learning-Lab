# 6-12 个月科研工程路线

## 总目标

把两个月项目从“可运行工程”升级成“科研训练项目”。

## 2-4 个月：平台工程 v2

目标：让项目像真实企业系统。

学习与建设：

- 用户与权限
- metadata 权限过滤
- Redis 缓存
- 异步任务队列
- Prometheus / Grafana
- Locust 或 k6 压测
- 成本统计
- RAG 评估集
- 错误处理和日志

产出：

- 压测报告
- 监控面板截图
- RAG 质量评估表
- 简历项目描述 v1

## 4-8 个月：科研训练 v3

目标：形成明确研究问题和实验方法。

研究问题示例：

> 在 RAG/Agent 请求类型差异较大的场景下，如何通过任务调度降低尾延迟并提升 worker 利用率？

学习与建设：

- baseline：FIFO / Priority / SJF / Random
- 变量：任务长度、并发量、worker 数量、优先级分布
- 指标：平均等待时间、P95/P99、吞吐、失败率、资源利用率
- 论文阅读：云原生、Serverless、推理服务调度、K8s AI workload
- 实验报告写作

产出：

- 3-5 张实验图
- 一篇实验报告
- 一页研究兴趣说明
- 联系导师材料 v2

## 8-12 个月：AI Infra v4

目标：从应用平台靠近推理平台。

学习与建设：

- Kubernetes 部署
- HPA
- Kueue / Volcano 了解
- vLLM
- Ray Serve
- Triton
- 模拟 GPU worker
- 请求路由
- autoscaling
- 冷启动与预热

产出：

- K8s 部署版 demo
- vLLM 或模拟推理 worker
- 推理指标实验记录
- 论文局部复现记录

## 长期能力目标

- 工程上：能独立设计、实现、部署、压测一个小型 AI 服务平台。
- 科研上：能提出问题、设置 baseline、做实验、解释结果、写报告。
- 就业上：能对齐 RAG/Agent、AI Platform、LLMOps、Inference Infra 岗位。
