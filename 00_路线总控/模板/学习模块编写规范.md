# 学习模块编写规范

## 适用范围

这份规范用于 `10_学习模块/` 下的所有学习模块，也适用于对应的 `20_资料库/模块资料索引/`、`40_实验练习/` 和 `50_项目产出/` 的联动写法。

它以 M00 为基准抽象出来，目标是让后续模块统一做到：可学、可练、可落地、可回看。

## 从 M00 抽取出的核心架构
M00 不是“计算机基础大全”，而是基础薄弱阶段的最小工程地基。

它的真实职责是：先让你具备独立创建、运行、调试、保存和复盘一个 Python 小项目的能力，再进入后续的后端、调度、RAG、容器和科研训练。

M00/M01 给出的基础标准是：一个学习模块不能靠堆文件显得完整，而要靠清楚的入口、稳定的教材正文、真实资料、可执行实验和项目承接形成闭环。

对大多数学习模块来说，主动文件只保留两个：

1. `学习地图`
2. `适配教材`

对少数核心实操模块，可以扩展为三个主动文件：

1. `学习地图`
2. `章节教材`
3. `复现推进表`

只有满足下面条件的模块，才允许使用三文件结构：

- 模块直接支撑核心项目或科研问题。
- 模块需要多轮手写代码、实验观察和复盘。
- 单个适配教材已经不足以承载“章节学习 + 复现执行”。
- 已经明确哪份文件是主教材，哪份文件是执行表。

M05 的经验说明：核心模块可以写深，但入口必须轻。旧导读、改造计划、临时规划、草稿和过期版本不能长期和主教材并列，应该进入 `99_归档/`。

一个模块通常还会配套三个跨目录组件：

1. `20_资料库/模块资料索引/` 下的资料索引
2. `40_实验练习/` 下的实验练习
3. `50_项目产出/` 下的项目主页或项目工作台

所以“模块写作标准”不是让一个目录长出许多主动文件，而是让模块通过少量主入口，去串起资料、实验和项目。

M00/M01/M05 共同抽出的职责边界是：

- 学习地图：定义边界、问题、目标和入口。
- 适配教材/章节教材：把概念讲清楚并接到项目上。
- 复现推进表：只用于核心实操模块，安排亲手复现顺序。
- 资料索引：记录权威来源和转化状态。
- 实验与项目：把知识变成可运行产出。
- 归档目录：保存旧导读、改造计划、草稿和历史版本，不能作为正式学习入口。

## M00 的最低学习目标

M00 完成后，学习者应该能：

- 独立创建并运行一个 Python 小项目
- 理解当前工作目录、绝对路径、相对路径和环境变量
- 知道为什么 `python -m pip` 比裸 `pip` 更稳
- 独立创建 `.venv` 并安装依赖
- 用 Git 保存、回退和定位学习进度
- 用 logging 记录关键过程
- 用 pytest 保护最小规则
- 理解 HTTP、JSON 和 API 的基础交互
- 把一个小脚本整理成可维护的小工程

## 后续所有模块的共同要求
1. 先有学习地图，再有教材正文，再有资料索引，再有实验。
2. 每个模块必须有明确的学习目标和边界，不写成大而全百科。
3. 每个模块必须能落到至少一个实验或项目功能。
4. 每个模块必须说明哪些内容先不学深。
5. 每个模块必须有检查标准，能判断是否真的学会。
6. 资料不是收藏，必须能被转化成卡片、实验、项目或表达。
7. 主动入口必须克制：普通模块最多 2 个主动文件，核心实操模块最多 3 个主动文件。
8. 任何旧版导读、改造计划、临时规划、重复教材和草稿都必须进入 `99_归档/`，不得和正式入口并列。
9. 学习地图不能写成长教材，教材正文不能写成路线清单，复现推进表不能替代教材讲解。
10. 每个教材章节必须同时满足“讲得懂、能练习、能验收、能连接项目”。

## 防偏移规则

M05 的偏移风险来自三个地方：内容越写越多、入口越建越多、旧规划没有及时归档。后续所有模块必须遵守下面规则。

### 入口防偏移

- 学习模块目录中只能出现正式主动文件和 `99_归档/`。
- 普通模块正式主动文件：`学习地图`、`适配教材`。
- 核心实操模块正式主动文件：`学习地图`、`章节教材`、`复现推进表`。
- 如果新增了“改造计划”“补充计划”“旧版教材”“草稿”，完成后必须移入 `99_归档/`。
- 学习地图里的推荐学习顺序只能指向正式入口，不能让用户先读归档文件。

### 内容防偏移

- 不因为某个技术热门就提前扩展。
- 不把后续模块内容塞进当前模块，例如 M05 不承担 FastAPI、数据库、Kubernetes 源码、真实 GPU 调度。
- 不把项目 README、简历表达、成果包装提前当作学习成果。
- 不把资料索引里的外部资料大段搬成教材。
- 不把实验记录写成教材，也不把教材写成实验台账。

### 质量防偏移

每个模块补强时必须检查五项质量：

| 维度 | 要求 | 不合格表现 |
|---|---|---|
| 完整性 | 有地图、教材、资料、实验、项目承接和检查标准 | 只有学习地图，不能开始学 |
| 可读性 | 章节像讲义，先解释为什么，再讲概念和例子 | 只列清单、术语堆叠、跳步太大 |
| 高质量 | 使用官方文档、高校课程、权威项目资料，不虚构来源 | 链接不明、资料泛泛、没有出处 |
| 实用性 | 每章能落到代码、实验、配置、项目功能或复盘 | 只讲概念，没有练习和验收 |
| 连通性 | 能从路线连到模块、资料、实验、项目、复盘 | 文档孤立，读完不知道下一步 |

## 学习地图标准

### 作用

学习地图是模块的导航图和任务边界，不负责把知识讲透。

它回答的是：

- 这个模块在总路线里的位置是什么
- 它要解决什么问题
- 它包含哪些内容
- 它对应哪些资料、卡片、实验和项目
- 学完后应该达到什么程度
- 哪些东西暂时不学深

### 必须包含的部分

- 在总路线中的位置
- 要解决的问题
- 学习内容
- 对应资料
- 对应知识卡片
- 对应实验
- 对应项目
- 检查标准
- 不需要一开始学很深的内容

### 写作要求

- 只写导航和边界，不展开长篇讲解
- 每个学习内容都要和实际能力挂钩
- 每个“对应资料”都要是真实来源
- 每个“对应实验”都要能实际执行
- 每个模块都要明确服务哪个主项目或科研问题
- 内容密度适中，避免变成流水账

## 适配教材标准
适配教材和章节教材都属于“教材正文”。区别只在深度和执行方式：普通模块使用适配教材，核心实操模块可以使用章节教材。

教材正文必须能让你之后真正用来学习，而不是只让知识库看起来完整。判断标准是：一个基础薄弱的学习者打开本章后，能知道为什么学、学什么、怎么练、怎么验收、和项目有什么关系。


### 作用

适配教材是模块的正文教材，不是路线说明，也不是资料台账。

它的任务是：把一个模块真正讲明白，让学习者可以按章节学习、按实验练习、按项目落地。

### 教材应当具备的特征

- 有明确读者：基础薄弱但要做项目的人
- 有明确目标：解决什么能力缺口
- 有解释、有例子、有场景
- 有最小代码或图示
- 有常见错误和排查方式
- 有和 P01 / 当前项目的对应关系
- 有练习和检查
- 有外部资料入口，但不直接堆资料

### 每章推荐结构

1. 为什么学这一章
2. 核心概念是什么
3. 最小示例或图示
4. 在项目里怎么用
5. 常见错误是什么
6. 练习或检查是什么
7. 相关资料去哪里查

### 必须包含的部分

- 编写说明
- 分章正文
- 项目贯通案例
- 学习检查
- 外部资料索引
- 暂时不要深入的内容

### 写作要求

- 不写成目录搬运
- 不写成待办列表
- 不写成只讲概念不落地的说明文
- 每章至少有一个与项目有关的例子
- 难点要讲直白，不要只给术语
- 允许使用代码块、表格和 Mermaid 图
- 允许指向权威资料，但正文必须能独立成立

## 编写说明

### 内容深度要求

M00 证明了一个原则：教材要“足够讲清楚”，但不要把自己写成整套教材合集。

所以后续模块的深度标准是：

- 能支撑当前项目推进
- 能解释核心机制
- 能做出最小实验
- 能回答为什么不是别的做法
- 不追求一次讲完整个领域

## F 系列教材适配要求

F 系列是金融工程与量化金融第二主线。它必须继承 M 系列教材标准，但不能写成金融百科、投资建议、资料清单或量化项目包装。

F 系列教材的读者仍然是：基础相对薄弱，但希望通过自学、代码、实验和项目逐步建立工程能力的人。

### F 系列教材的基本定位

F 系列教材要回答四个问题：

1. 这个金融概念为什么重要？
2. 它在数据、代码或实验里长什么样？
3. 它如何变成 GF 实验、Q 项目或 P03 task？
4. 它有哪些数据口径、模型假设和风险边界？

例如，讲 VaR 时不能只写“VaR 是风险价值”，而要讲：

- 为什么组合风险需要一个尾部损失指标。
- 输入数据是价格还是收益率。
- 时间窗口、置信水平和频率如何影响结果。
- 最小 Python 计算流程是什么。
- 记录表里要保存哪些字段。
- 为什么结果不构成投资建议。
- 未来如何映射到 `risk_metric_task`。

### 每章必须包含的内容

F 系列每章除了通用教材结构外，还必须尽量包含：

1. `金融直觉`：用日常语言解释概念解决什么问题。
2. `数学/统计口径`：只讲当前需要的公式、变量和假设，不做纯证明大课。
3. `数据口径`：说明数据来源、字段、频率、缺失、复权、时间范围和 point-in-time 限制。
4. `最小代码/流程`：给出可以手写或复述的最小计算流程。
5. `实验连接`：明确对应 GF 实验候选或后续具体实验页。
6. `项目连接`：明确对应 Q01/Q02/Q03/Q04/X02 或 P03 task。
7. `常见误区`：尤其是数据泄漏、过拟合、忽略交易成本、把回测当收益承诺。
8. `记录字段`：说明实验或 task 至少要记录哪些字段。
9. `风险提示`：明确不构成投资建议，不虚构结果，不包装成已完成项目。

### F 系列章节推荐模板

```markdown
## 第 X 章：章节名

### X.1 本章解决什么问题

用金融场景说明为什么要学这一章。

### X.2 金融直觉

用非术语化语言解释概念，不先堆公式。

### X.3 数学/统计口径

只写当前实验需要的公式、变量、假设和限制。

### X.4 数据口径

说明输入数据是什么、从哪里来、频率是什么、哪些字段必须记录。

### X.5 最小代码或计算流程

给出最小 Python 代码、伪代码或表格流程。

### X.6 工程场景

说明它在 GF 实验、Q 项目或 P03 task 里怎么使用。

### X.7 常见误区

列出学习和工程实现中最容易犯的错误。

### X.8 小练习

给出一个能亲手做的练习，最好能落到记录表。

### X.9 检查标准

说明什么叫学会本章。

### X.10 推荐资料

只列本章真正要用的权威资料小节。
```

### F 系列教材不能写成什么

- 不能写成金融词典：只解释术语，不教计算和实验。
- 不能写成资料台账：只罗列课程、网站、书和链接。
- 不能写成投资建议：不预测行情，不承诺收益，不给买卖判断。
- 不能写成高级数学大纲：第一轮不深挖随机微积分、测度论或复杂优化证明。
- 不能写成模型堆叠清单：金融 ML 第一轮重在 baseline、时间切分、泄漏检查和失败分析。
- 不能写成项目成果包装：候选实验、AI 辅助材料和未复现内容必须标明状态。

### F 系列和 M 系列的交叉写法

F 系列教材不能只停留在金融概念层。每个重要章节都要说明它如何进入 M 系列工程系统：

| F 系列内容 | M 系列连接 | P03 候选任务 |
|---|---|---|
| 金融时间序列清洗 | M01/M06/M08 | `data_quality_task` |
| 收益率、波动率、回撤、VaR | M06/M08 | `risk_metric_task` |
| 投资组合权重与约束 | M05/M08 | `portfolio_task` |
| 期权和 Monte Carlo 定价 | M05/M08 | `pricing_task` |
| 回测流程 | M05/M06/M08/M11 | `backtest_task` |
| 金融 ML 验证 | M08/M11 | `model_risk_task` |
| 金融文档理解 | M03/M04/M12 | `finance_rag_query` |

### F 系列教材合格线

一份 F 系列适配教材发布前，至少要满足：

- [ ] 有学习地图和资料索引承接。
- [ ] 每章有解释、例子、练习和检查标准。
- [ ] 至少连接一个 GF 实验候选。
- [ ] 至少连接一个 Q 项目或 P03 task。
- [ ] 明确数据口径、模型假设和暂不深入范围。
- [ ] 没有虚构数据结论、收益结果或项目成果。
- [ ] 读者能按章节学习，而不是只看到一堆清单。

## 资料索引标准

### 作用

资料索引是来源管理，不是摘要大全。

它负责回答：

- 这条资料是什么
- 为什么收录
- 属于哪个模块
- 当前状态是什么
- 能转化成什么

### 每条资料必须标注

- 资料名称
- 来源链接
- 类型
- 适合阶段
- 对应模块
- 对应实验或项目
- 当前状态
- 为什么收录
- 能转化成什么

### 状态建议

- 候选：真实存在，暂未开始处理
- 必读：当前阶段必须读
- 选读：遇到问题或进阶时再读
- 查阅：作为工具文档使用
- 已转化：已经变成卡片、实验或项目功能
- 暂缓：当前不投入时间

### 收录规则

- 优先官方文档、高校课程、权威项目文档和经典论文
- 资料必须和模块目标有关
- 每周只激活少量资料
- 资料必须有转化出口，不能只收藏

### 推荐模板

```markdown
# 资料标题

## 来源

- 链接：
- 类型：官方文档 / 课程 / 论文 / 技术博客 / 开源项目 / JD
- 状态：候选 / 必读 / 选读 / 查阅 / 已转化 / 暂缓

## 为什么收录

这条资料和当前路线的关系是什么？

## 适用模块

- [[学习模块]]

## 可转化内容

- 知识卡片：
- 实验：
- 项目功能：

## 摘要

- 
- 
- 
```

## 实验联动标准

每个实验页必须具备：

- 实验定位
- 前置阅读
- 实验目标
- 输入或测试数据
- 步骤
- 验收标准
- 记录区
- 关联模块和项目

实验不是附属品，而是教材落地的证明。

## 项目联动标准

每个学习模块都要回答：

- 它最终支撑哪个项目
- 它能在项目里变成什么功能
- 它能给项目提供什么指标、测试或说明

### 普通学习模块目录

## 最后检查清单

在发布一个学习模块前，至少确认：

- [ ] 位置和边界清楚
- [ ] 学习目标清楚
- [ ] 资料真实且有关联
- [ ] 教材不是台账
- [ ] 教材有项目例子
- [ ] 实验能执行
- [ ] 项目能接住
- [ ] 检查标准可验证
- [ ] 有明确的暂不深入内容
- [ ] 文档之间的链接闭环成立
