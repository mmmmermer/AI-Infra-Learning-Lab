# 学习模块建设状态表

## 用途

本表管理学习模块的真实成熟度。它不再用“有教材页”“有实验页”代替可执行和已验证状态。

状态与证据见：

- [[00_路线总控/审计与整改/00_整改总控|整改总控]]
- [[00_路线总控/审计与整改/11_教材内容优化任务矩阵.csv|110 项内容优化任务矩阵（整改任务真源）]]
- `00_路线总控/审计与整改/artifacts/governance/optimization_matrix_status_evidence.csv`
- [[00_路线总控/审计与整改/01_逐文件整改矩阵.csv|教材与可执行源文件覆盖及自动分诊清单（非发布清单、非成熟度真源）]]
- [[00_路线总控/审计与整改/01_逐文件整改矩阵_历史审计快照.csv|历史逐文件审计快照]]
- [[00_路线总控/审计与整改/03_验证清单|验证清单]]

覆盖清单由生成器从 Markdown、源码、配置和锁文件集合重建，不继承首轮审计中的旧 blocker；
CSV/JSON/SVG/压缩研究证据等发布文件由结构解析、校验和、编码及 Git 发布候选门禁另行覆盖。未进入人工复核的文件
必须保持 `unreviewed + unassessed/unknown`，不得自动推导为 `planned`、低风险或非阻断；历史
快照只用于追溯。模块成熟度以本表为准，整改任务状态以 110 项矩阵及其证据表为准。

## 状态维度

| 维度 | 状态 |
|---|---|
| 内容成熟度 | `planned` / `draft` / `content-reviewed`；`reviewed` 仅作历史别名 |
| 实现 | `not-applicable` / `absent` / `partial` / `runnable-task` / `executable` |
| Reference 验证 | `unverified` / `verified` |
| 教学就绪 | `not-assessed` / `partial` / `instructional-ready` |
| 归属 | `reference` / `learner-reproduced` / `learner-owned` |
| 学习者验证 | `not-evaluated` / `pending` / `learner-validated` |

各状态轴独立判断：`content-reviewed` 不自动变成 `instructional-ready`，reference 的 `verified` 不自动
变成 `learner-validated`。“当前可进入”只表示可以按修订后的入口学习，不表示项目或实验已经由
学习者完成。

## 升格规则

模块或实验只有满足以下条件，才能从 `draft/partial/unverified` 升格：

1. 概念、代码语义和来源经过校订。
2. 干净环境能按文档命令运行。
3. 正常、边界和错误路径测试通过。
4. 安全与权限在最小版本中成立。
5. 有可追溯的运行记录和验收证据。
6. AI 参考与本人完成边界明确。
