## 第二阶段总原则

1. 不新增 M00-M12 主入口。
2. 不继续扩写大段教材。
3. 优先补实验页，而不是继续补路线说明。
4. 实验页必须包含目标、步骤、数据、指标、记录表和验收标准。
5. P03 已拆出项目工作台，后续按 v0.1 执行并沉淀真实实验记录。
6. 脱敏合规 AI 案例只作为案例参照，不能照搬包装。
7. 科研和简历表达必须等待真实复现、实验记录和项目结果。

## 优先级 1：补 E03 RAG 实验详情

### 为什么先补

E03 直接支撑 P03 的第一个真实 AI workload：RAG 请求。

没有 E03，M03 教材虽然可读，但 P03 的 `RagTask`、引用、延迟、失败类型和评估指标缺少练习来源。

### 需要补什么

- `E03-01 chunk 大小对检索效果的影响`
- `E03-02 top-k 对回答质量和延迟的影响`
- `E03-03 metadata 权限过滤实验`

### 合格标准

- 每个实验有测试数据、步骤、观察指标和记录表。
- 能记录 `retrieved_sources`、`has_citation`、`latency_ms`、`token_count`、`error_type`。
- 能说明实验结果如何影响 P03 的 RagTask 字段。
- 可以接入 M12 的金融/合规短文档样例，但必须标注来源或自造测试数据。

## 优先级 2：补 E06 数据库异步任务实验详情

### 为什么第二

P03 不能长期停留在内存脚本。RAG/Agent 请求必须能保存、恢复、异步执行和查询状态。

E06 是 P03 从 demo 走向工程服务的地基。

### 需要补什么

- `E06-01 任务状态持久化`
- `E06-02 文档解析或 RAG 请求异步化`
- `E06-03 embedding 或检索结果缓存实验`

### 合格标准

- 能形成 `任务入库 -> 状态更新 -> 异步执行 -> 失败重试 -> 查询状态` 闭环。
- 能解释 `pending / queued / running / succeeded / failed / retrying / cancelled`。
- 能给出最小任务表字段。
- 能说明 M05 队列和 worker 如何进入真实服务。
