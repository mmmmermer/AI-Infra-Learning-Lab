## 2. 岗位能力概述

下文是为校准学习路线而写的必要能力概述，不是 JD 原文转载，也不保留完整逐句摘录。来源按
`平台 + 日期 + 岗位名称` 定位；岗位内容可能随时间变化。

### 代码与算法质检能力

- 读懂 Python、SQL 等主流代码，理解基础算法、数据结构。
- 质检 AI 代码内容，精准识别语法错误、逻辑漏洞、边界缺失、安全风险。
- 准确解读技术文档、API 说明、开源项目内容，做出专业判断。

### 金融知识素养

- 熟悉三大财务报表、基础金融术语、估值模型与金融市场规则。
- 校验 AI 在金融计算、数据分析、投资建议等内容的准确性、合规性，识别内容幻觉。
- 了解 Wind、Bloomberg 等金融数据接口概念者优先。

### 数据评测与规则能力

- 拥有数据标注、大模型评测实操经验，熟悉数据集全生产流程。
- 掌握 Rubric（评分标准/标注规范），可独立拆解评分维度、撰写细则，并根据问题推动规则迭代优化。
- 快速吃透标注 SOP，高密度工作中保持对数据精度、单位、时间逻辑等细节的敏感度。

### 逻辑思维与判断能力

- 具备批判性思维，不盲从 AI 输出，精准识别逻辑漏洞、事实错误、因果倒置等问题。
- 多版本内容对比分析，筛选最优解，并出具清晰的质量判定依据。

### 大模型使用能力

- 深度使用 ChatGPT、Claude、Gemini 等主流前沿大模型，熟悉模型能力边界与常见错误。
- 有 CodeBuddy、WorkBuddy、OpenClaw 等 Agent 类产品使用经验优先。
- 可评估 Agent 任务拆解、工具调用、代码执行全链路的合理性，区分典型成功/失败场景。
- 主动发现规则盲区，整理问题并反馈算法团队，参与规则共创迭代。

## 3. 岗位一句话判断

这个岗位的核心不是“会用 AI 生成内容”，而是：

```text
在金融、代码和 Agent 场景中，判断 AI 输出是否正确、可信、合规、可复核，并把问题沉淀成 Rubric、SOP 和评测规则。
```

它不是典型 AI Infra 工程岗，但会反向提示当前学习路线需要补齐一条能力线：

```text
AI 输出质量评测与人工复核能力线
```

## 13. 能力域重构方案

### 模块结构重构

- M03 增加“RAG 输出质量评测”主线提示。
- M04 增加“Agent 轨迹评测”主线提示。
- M11 增加“Rubric / 标注 / 人工评测 / 规则迭代”能力线。
- M12 增加“金融事实核验 Rubric”。

### 教材内容重构

- M03 补充 supported claim、unsupported claim、faithfulness、citation check。
- M04 补充 tool_call_correctness、trajectory、step-level review。
- M11 补充 Rubric 设计、标注一致性、人工复核记录、规则迭代。
- M12 补充金融事实、时间、单位、口径、来源核验。

### 实验链路重构

- E03 增加 RAG 回答质量记录字段，不一定新增文件。
- E04 增加 Agent step review 字段，不一定新增文件。
- 后续可新增 E11-01 Rubric 评分一致性实验，但本轮先不新建入口。

### P03 项目重构

- 增加轻量 `EvaluationTask` 设计。
- 增加 `ReviewRecord` / `Rubric` 字段说明。
- 增加质量指标：`unsupported_claim_count`、`hallucination_type`、`logic_error_type`、`tool_call_error`、`rubric_score`、`review_status`。

### 资料索引重构

- M03 加入 Ragas / LangSmith RAG eval 资料。
- M04 加入 OpenAI Agent evals / LangSmith trajectory evals。
- M11 加入 OpenAI eval best practices / eval-driven design。
- M12 保留 SEC/CFA 金融资料，并明确它们服务事实核验。

### 学习顺序重构

第一阶段不变：

```text
M00 -> M01 -> M05 -> P01
```

P03 阶段改为：

```text
M03 RAG 最小链路
-> M06 状态持久化
-> M04 Agent 可控工作流
-> M11 Rubric 和人工评测
-> M08 质量 + 性能指标
-> M12 金融场景核验
```

### 表达边界重构

当前只能表达为：

```text
正在围绕 RAG/Agent 输出质量评测设计 Rubric、人工复核字段和金融事实核验流程。
```

完成真实实验后才可表达：

