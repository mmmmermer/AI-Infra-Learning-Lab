# GF02-01 金融时间序列清洗

> 校订状态：本实验使用 `(symbol, date)` 作为唯一键，并包含同日双资产数据。仅按日期去重会误删另一资产，属于验收失败。

## 实验定位

本实验是 F02 的第一个具体实验页，目标不是分析真实市场，而是训练一个可复现的金融时间序列清洗流程。

它解决的问题是：

```text
拿到一张价格表后，如何判断它能不能用于收益率、波动率、回撤和后续风险指标计算？
```

本实验使用自造测试数据，只用于学习流程，不构成真实市场结论，也不构成投资建议。

## 为什么这个实验有意义

很多金融分析错误不是发生在模型阶段，而是发生在数据进入系统的第一步。

如果这一步没有做好：

- 收益率可能按错误顺序计算。
- 重复日期可能让同一天价格出现两个版本。
- 缺失价格可能让 rolling 指标出现大片空值。
- 使用了错误价格字段，后续回测和风险指标都会失真。
- 没有记录数据口径，未来无法复现实验。

所以 GF02-01 的重点不是“把表洗干净”，而是建立一套可解释、可追踪的清洗记录。

### 步骤 3：制定清洗策略

本实验采用保守策略：

| 问题 | 处理方式 | 为什么 |
|---|---|---|
| 日期未排序 | 按 `symbol,date` 升序排序 | 收益率和 rolling 依赖时间顺序 |
| 重复日期 | 保留最后一条 | 模拟“后下载版本覆盖前版本”，真实项目需查来源 |
| `adjusted_close` 缺失 | 删除该行 | 第一轮收益率计算依赖复权价格 |
| `close` 与 `adjusted_close` 同时存在 | 使用 `adjusted_close` | 更适合长期收益率练习 |

注意：这些策略不是唯一正确答案。实验必须记录策略，方便未来复查。

### 步骤 4：执行清洗

```python
df_sorted = df.sort_values(["symbol", "date"])
df_dedup = df_sorted.drop_duplicates(subset=["symbol", "date"], keep="last")
df_clean = df_dedup.dropna(subset=["adjusted_close"]).copy()
df_clean = df_clean.set_index(["symbol", "date"]).sort_index()

assert not df_clean.index.duplicated().any()
```

本实验不是观察收益率高低，而是观察数据质量。

## 常见错误

- 直接计算收益率，不先检查日期排序。
- 删除缺失值后不记录删除了多少行。
- 把 `close` 和 `adjusted_close` 混用。
- 对重复日期直接平均，但不说明原因。
- 用自造数据写真实市场解释。
- 只保存最终图表，不保存清洗步骤。
