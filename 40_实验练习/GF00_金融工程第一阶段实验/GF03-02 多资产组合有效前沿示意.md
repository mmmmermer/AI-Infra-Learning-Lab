## 为什么这个实验有意义

两资产实验能展示权重变化。多资产实验能展示一个更接近实际组合问题的现象：

```text
很多不同权重组合，会形成一片风险收益散点。
```

所谓有效前沿，可以粗略理解为：在这些候选组合里，同等风险下收益更高、或同等收益下风险更低的一批点。

但第一轮只做示意，不做严格优化。因为严格有效前沿依赖：

- 资产收益估计。
- 协方差估计。
- 权重约束。
- 是否允许卖空。
- 优化算法。
- 样本稳定性。

这些条件一变，前沿就会变。

## 知识点与难点

### 多资产组合收益

```text
r_p = w_1 r_1 + w_2 r_2 + ... + w_n r_n
```

### 多资产组合风险

```text
portfolio_variance = w^T Sigma w
portfolio_volatility = sqrt(portfolio_variance)
```

### 随机权重

第一轮用随机权重生成候选组合：

```text
weights >= 0
sum(weights) = 1
```

这不是优化，只是采样观察。

### 工程痛点

- 资产顺序必须和权重顺序一致。
- 协方差矩阵必须和收益率列顺序一致。
- 随机种子要记录，否则结果不可复现。
- 采样点数量影响散点形态。
- “前沿点”只是样本和候选权重里的相对结果。
- 不同数据窗口会得到不同图形。

## 实验步骤

### 步骤 1：读取多资产收益率数据

```python
from io import StringIO
import numpy as np
import pandas as pd

returns_csv = """date,A,B,C
2024-01-03,0.012,0.004,0.006
2024-01-04,-0.014,0.002,-0.003
2024-01-05,0.023,0.006,0.010
2024-01-08,0.011,-0.001,0.004
2024-01-09,-0.016,0.003,-0.006
2024-01-10,0.004,0.001,0.002
2024-01-11,-0.031,-0.004,-0.012
2024-01-12,0.018,0.005,0.007
2024-01-15,-0.008,0.000,-0.002
2024-01-16,0.007,0.002,0.003
"""

returns_df = pd.read_csv(StringIO(returns_csv))
returns_df["date"] = pd.to_datetime(returns_df["date"])
returns_df = returns_df.sort_values("date").set_index("date")
asset_order = ["A", "B", "C"]
```

### 步骤 2：计算均值和协方差矩阵

```python
mu = returns_df[asset_order].mean().values
cov = returns_df[asset_order].cov().values
```

记录：

```text
covariance_method: pandas sample covariance
annualized_or_not: false
```

### 步骤 3：生成随机权重

```python
rng = np.random.default_rng(seed=42)
n_portfolios = 200

raw_weights = rng.random((n_portfolios, len(asset_order)))
weights_matrix = raw_weights / raw_weights.sum(axis=1, keepdims=True)
```

记录：

| 字段 | 记录 |
|---|---|
| random_seed | 42 |
| n_portfolios | 200 |
| long_only | true |
| weight_sum | 1.0 |

### 步骤 4：计算每组权重的风险收益

```python
rows = []

for weights in weights_matrix:
    portfolio_return = float(weights @ mu)
    portfolio_variance = float(weights @ cov @ weights)
    portfolio_volatility = portfolio_variance ** 0.5
    rows.append(
        {
            "weight_A": float(weights[0]),
            "weight_B": float(weights[1]),
            "weight_C": float(weights[2]),
            "portfolio_return": portfolio_return,
            "portfolio_volatility": portfolio_volatility,
        }
    )

portfolio_points = pd.DataFrame(rows)
```

### 步骤 5：做一个简单前沿示意

第一轮不做严格优化，只做粗略标记：

```python
portfolio_points["vol_bin"] = pd.qcut(
    portfolio_points["portfolio_volatility"],
    q=10,
    duplicates="drop"
)

frontier_like = (
    portfolio_points
    .sort_values("portfolio_return", ascending=False)
    .groupby("vol_bin", observed=True)
    .head(1)
    .sort_values("portfolio_volatility")
)
```

解释：

```text
frontier_like 只是每个波动率分箱里收益较高的候选点，不是严格有效前沿。
```

## 观察指标

| 指标 | 含义 | 解释边界 |
|---|---|---|
| `portfolio_return` | 给定权重下的样本平均收益 | 不代表未来收益 |
| `portfolio_volatility` | 给定权重下的样本组合波动 | 受样本和协方差影响 |
| `n_portfolios` | 随机采样组合数量 | 数量越多散点越密 |
| `frontier_like` | 分箱中收益较高的候选点 | 不是严格有效前沿 |
| `random_seed` | 控制可复现 | 不记录会导致结果变化 |

## 常见错误

- 把随机采样结果当成严格有效前沿。
- 不记录 random seed。
- 不记录资产顺序，导致权重解释错位。
- 忘记权重归一化。
- 用小样本散点图做投资结论。
- 不说明是否允许卖空和杠杆。
- 把“示意图”写成“最优资产配置系统”。
