# E00-02 Git 保存学习进度

## 实验定位

这个实验训练你用 Git 保存一次真实学习进度。

Git 不是网盘，也不只是上传 GitHub 的工具。当前阶段你先把它理解成：给项目拍可追踪的快照。以后代码写坏了、实验结果变了、AI 改乱了文件，都可以通过 Git 看清楚发生了什么。

## 实验目标

在 `task_sorter` 项目里完成一次 Git 提交。

你要理解三件事：

- working tree：你正在修改的文件
- staging area：准备提交的文件
- repository：已经保存下来的提交历史

## 第 1 步：进入项目目录

```powershell
cd task_sorter
pwd
ls
```

确认能看到：

```text
main.py
.venv/
```

注意：`.venv` 一般不应该提交进 Git。

## 第 2 步：初始化 Git 仓库

```powershell
git init
```

查看状态：

```powershell
git status
```

你应该能看到未跟踪文件。

## 第 3 步：创建 `.gitignore`

在项目根目录创建 `.gitignore`：

```text
.venv/
__pycache__/
.pytest_cache/
```

然后再次查看：

```powershell
git status
```

你应该看到 `.venv` 不再作为要提交的内容出现。

## 第 4 步：添加文件到暂存区

```powershell
git add main.py .gitignore
```

查看状态：

```powershell
git status
```

此时文件应该进入 staged 状态。

## 第 5 步：完成第一次提交

```powershell
git commit -m "create minimal task sorter project"
```

查看提交历史：

```powershell
git log --oneline
```

预期能看到一条提交记录。

## 第 6 步：做一个小修改并查看差异

修改 `main.py`：

```python
print("task_sorter is running")
print("ready to sort tasks")
```

查看差异：

```powershell
git diff
```

你应该能看到新增了一行输出。

## 验收标准

- [ ] 能执行 `git init`
- [ ] 能用 `.gitignore` 排除 `.venv`
- [ ] 能解释 `git status` 里 staged / unstaged / untracked 的区别
- [ ] 能完成一次 commit
- [ ] 能用 `git log --oneline` 找到提交
- [ ] 能用 `git diff` 查看修改

## 常见错误

### 错误 1：把 `.venv` 提交进 Git

`.venv` 里是本地依赖环境，通常很大，也不适合提交。

如果 `git status` 里看到 `.venv/`，先检查 `.gitignore` 是否写对。

### 错误 2：commit message 太模糊

不推荐：

```text
update
```

推荐：

```text
create minimal task sorter project
```

commit message 应该说明这次提交做了什么。

### 错误 3：不知道当前有没有东西要提交

先执行：

```powershell
git status
```

不要靠感觉判断。
