# E00-04 写一个带日志的小脚本

## 实验定位

这个实验训练你用 logging 观察脚本运行过程。

工程项目里，很多 bug 不是语法错误，而是数据在某一步变成了你没想到的样子。日志的作用就是把关键过程留下来，让你能定位问题，而不是盲目改代码。

## 实验目标

给一个任务排序脚本加入日志，记录：

- 输入任务数量
- 使用的排序规则
- 排序前后的任务顺序
- 异常发生的位置

## 推荐代码位置

```text
task_sorter/
└─ task_sorter.py
```

## 第 1 步：准备任务数据

```python
tasks = [
    {"id": "task-001", "created_at": 0, "priority": 2, "duration": 5},
    {"id": "task-002", "created_at": 1, "priority": 1, "duration": 2},
    {"id": "task-003", "created_at": 2, "priority": 3, "duration": 1},
]
```

## 第 2 步：配置 logging

```python
import logging

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s",
)
```

当前阶段不用深入 handler、filter、formatter。先会用 `logging.info` 记录关键过程。

## 第 3 步：给排序函数加日志

> 环境要求：使用 `py -3.13` 创建并激活项目 `.venv`。若 `python --version` 显示 3.8.6，说明环境未激活，不要通过修改类型标注继续运行。

```python
def sort_by_priority(tasks: list[dict]) -> list[dict]:
    logging.info("sort_by_priority input_count=%s", len(tasks))
    logging.info("before_sort order=%s", [task["id"] for task in tasks])

    sorted_tasks = sorted(tasks, key=lambda task: task["priority"])

    logging.info("after_sort order=%s", [task["id"] for task in sorted_tasks])
    return sorted_tasks
```

运行时应该看到类似：

```text
INFO sort_by_priority input_count=3
INFO before_sort order=['task-001', 'task-002', 'task-003']
INFO after_sort order=['task-002', 'task-001', 'task-003']
```

## 第 4 步：人为制造错误

把某个任务的 `priority` 改成拼错的字段，例如：

```python
{"id": "task-002", "created_at": 1, "prio": 1, "duration": 2}
```

再次运行脚本，观察报错。

你应该能定位：排序函数访问 `task["priority"]` 时失败。

## 第 5 步：用日志写复盘

记录：

- 哪个输入有问题
- 程序在哪一步失败
- 日志提供了什么线索
- 下次如何避免

## 验收标准

- [ ] 日志能显示输入任务数量
- [ ] 日志能显示排序前后顺序
- [ ] 能制造一次错误并定位
- [ ] 能写出一段复盘说明
- [ ] 能说明 logging 和 print 的区别

## 常见错误

### 错误 1：日志只写“开始了”“结束了”

这种日志帮助不大。好的日志要包含关键变量，例如任务数量、任务 id、策略名。

### 错误 2：把异常吞掉

不要这样写：

```python
try:
    ...
except Exception:
    pass
```

这会让错误消失，但问题还在。

### 错误 3：日志太多但没有信息

当前阶段只记录关键节点：输入、策略、输出、错误。

## 和 P01 的关系

P01 的调度器后续需要观察：任务进入队列、排序策略、worker 分配、指标计算。没有日志，实验结果异常时很难判断是策略问题、数据问题还是代码问题。
