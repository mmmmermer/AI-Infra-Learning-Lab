# E05-06 项目复盘与证据表达

## 状态与边界

- 类型：学习者复盘任务。
- 状态：`runnable task / learner-unverified`。
- 前置：E05-R0-R1、E05-01 至 E05-05，以及
  [[10_学习模块/M05_任务队列与调度/教材章节/12_项目总结与复盘|第 12 章]]。
- 目标：只使用本人实际复现产生的代码、测试和 artifact 形成项目表达。

P01 当前是 reference。没有完成的实验、没有亲手写出的实现和只看过的结果，不得改写成
“我实现了”或“实验表明”。

## 交付物

在个人练习目录中建立：

```text
README.md
evidence.md
limitations.md
reproduce.ps1
artifacts/
```

`reproduce.ps1` 必须从固定输入运行测试和实验，不能只打印已有 CSV。

## 主张与证据表

先写证据表，再写 README：

| 主张 | 本人证据 | reference 证据 | 限制 | 可以使用的措辞 |
|---|---|---|---|---|
| 实现 FIFO | 本人源码提交和测试日志 | P01 对照文件 | 仅确定性模拟 | “实现并测试了最小 FIFO 模拟器” |
| 比较调度策略 | 本人固定 workload 与逐任务结果 | P01 示例 artifact | 非真实 RAG 服务 | “在固定合成 workload 上比较” |
| 多 worker 扩容 | E05-05 原始结果 | P01 参考结果 | 未计入进程/网络成本 | “观察到本次夹具中的变化” |

表中没有本人证据的行只能写入“后续工作”，不能进入成果描述。

## README 必含内容

1. 问题和范围：解决的最小调度问题，以及明确不解决什么。
2. 数据模型：Task、Worker、Queue、Result 的字段与时间语义。
3. 策略：FIFO、Priority、SJF，以及每种策略的优化目标和代价。
4. 验证：测试命令、固定输入、逐任务 artifact 和汇总方法。
5. 结果：只引用本人跑出的数字，并给出 workload/hash/run id。
6. 局限：合成任务、单机模拟、无真实服务资源竞争、无生产故障模型。
7. 复现：从干净练习目录开始的完整命令。

## 禁止表述

- “显著提升”但没有预先定义指标、重复实验和不确定性。
- “适用于 RAG/Agent 生产系统”但只跑了合成模拟。
- “实现了 Kubernetes 调度”但没有 Kubernetes reference。
- 把 P01 的代码、图或结果改成第一人称成果。
- 只报告平均值，隐藏 P95/P99 或受损任务组。

## 复盘问题

```text
哪一段核心逻辑可以不看 P01 重新写出？

哪个结论最依赖固定 workload？

哪个任务组在当前策略下受损？

如果预测耗时错误，SJF 结论如何变化？

真实服务还需要哪些时间戳、资源指标和故障实验？
```

## 验收

- [ ] README 中每个结果数字都能回到本人 artifact。
- [ ] `evidence.md` 区分本人证据和 reference 证据。
- [ ] `reproduce.ps1` 能重新生成测试与实验结果。
- [ ] `limitations.md` 明确合成模拟和外推边界。
- [ ] 没有把 P01 reference 写成个人完成记录。
- [ ] 至少请另一位读者按证据表检查一次措辞。
