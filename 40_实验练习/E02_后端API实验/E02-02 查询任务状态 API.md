# E02-02 查询任务状态 API

## 目标

在 E02-01 的同一 app 和 repository 上实现 `GET /tasks/{task_id}`。本实验不允许重新声明 `TASKS = {}`。

## 正确链路

```text
POST /tasks + Alice Principal
-> middleware(request_id/deadline)
-> principal dependency
-> TaskService.create_task(...)
-> repository.create(...)
-> GET /tasks/{task_id}
-> TaskService.get_task(...)
-> repository.get_for_principal(task_id, principal)
```

## 错误契约

未知任务与跨 owner 任务统一返回 RFC 9457 风格、可机器判定的 Problem Details：

```json
{
  "type": "urn:e02:problem:task_not_found",
  "title": "Task not found",
  "status": 404,
  "detail": "The task does not exist or is not visible to this principal.",
  "code": "task_not_found",
  "request_id": "trace-example-001"
}
```

HTTP 状态码为 404，不返回空对象，也不自动创建脏任务。Bob 查询 Alice 已存在的 task ID 也得到
相同 `status/code`，避免用响应差异枚举其他 owner 的资源；`request_id` 每次请求可以不同。

## 验收

- [x] 能查询刚刚通过 POST 创建的任务。
- [x] 查询结果与创建结果一致。
- [x] 未知 task id 返回稳定 404 错误。
- [x] 跨 owner task id 与未知 id 返回相同 404。
- [x] 创建与查询端点来自同一个 `app.main:app`。
