# E02-01 创建任务 API

## 目标

在共享项目 `e02_service/` 中实现 `POST /tasks`。服务端从 bearer fixture 形成 `Principal`，生成
`task_id` 和 UTC `created_at`，并按 principal 保存所有权；客户端不能提交或覆盖身份、权限或
服务端接收时间。

## 关键文件

- `e02_service/app/models.py`
- `e02_service/app/repository.py`
- `e02_service/app/security.py`
- `e02_service/app/main.py`
- `e02_service/tests/test_api.py`

## 请求

```json
{
  "task_type": "rag_query",
  "priority": 2,
  "estimated_duration_ms": 1500
}
```

请求使用 `Authorization: Bearer alice-fixture`。该短字符串是公开、非秘密的测试凭据，只用于把
确定的服务端 principal 带入实验。未知字段被拒绝；`priority` 和耗时有明确范围。第一版创建状态为
`pending`，因为 E02 尚未接入真实队列。

## 验收

```powershell
cd e02_service
.\.venv\Scripts\Activate.ps1
python -m pytest -q
```

- [x] 返回 HTTP 201。
- [x] `task_id` 不是固定字符串。
- [x] `created_at` 是服务端生成的带时区 UTC 时间。
- [x] 创建结果写入共享 repository。
- [x] 客户端 `submit_time` 和未知字段被拒绝。
- [x] 无凭据返回 401，缺 `tasks:write` 返回 403。
- [x] 客户端伪造 `tenant_id/user_id/owner_id/permission_group` 返回 422。
