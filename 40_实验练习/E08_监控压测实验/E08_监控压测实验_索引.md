# E08 监控压测实验索引

> 状态（2026-07-11，当前契约 v0.3.1）：P03 已补 run-id 隔离指标、Locust CSV/JSON、500ms 队列时序、worker 容器 CPU/内存时序，以及随机化顺序的 1/2/4 worker × 3 次 `mock_rag` reference 和 95% Student t 区间；另有 18-task BM25 `rag_retrieval` 单轮 smoke。学习者复现、长时稳态、warm-up、重复真实 RAG/LLM 和多主机验证仍未完成；本机 reference 不是容量或科研结论。

## 定位

E08 用于把 M08 的日志、metrics、tracing、压测和报告方法落到 P03。它不是为了搭一个复杂监控平台，而是训练你用数据回答：

```text
P03 慢在哪里？
调度策略是否改善了 P95/P99？
worker 数量是否足够？
队列是否堆积？
错误率是否升高？
```

## 实验闭环

```text
M05/P01 调度指标
-> M08 适配教材
-> P03 暴露 metrics
-> Locust/k6 生成负载
-> 记录 P95/P99、吞吐、错误率、队列长度、worker utilization
-> 写压测报告
```

## 最小练习路线

```text
先压测 API 是否能稳定接收请求
-> 再压测 RAG 任务是否会排队
-> 记录 queue_wait / task_runtime / total_latency
-> 对比 1/2/4 worker
-> 写一份瓶颈判断报告
```

## 必须记录的指标

| 指标 | 用途 |
|---|---|
| average_latency | 总体平均体验 |
| p95_latency / p99_latency | 尾延迟 |
| throughput | 单位时间处理能力 |
| error_rate | 稳定性 |
| broker_queue_length | Redis consumer group 的 pending + lag；不是数据库全部 queued task 数 |
| pending/queued/running | 数据库任务状态，不能与 Redis backlog 混为一谈 |
| p95_queue_wait | 排队尾延迟 |
| task_runtime | worker 执行耗时 |
| worker_utilization | worker 是否忙满 |
| worker_cpu_percent / worker_memory_mib | 容器资源占用，与 task-processing utilization 分开 |
