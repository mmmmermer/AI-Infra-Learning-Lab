# GF05-03 利率曲线数据读取与说明

## 实验定位

本实验不要求立即调用真实 API，而是训练公开利率数据的字段口径记录能力。

核心流程：

```text
series_id
-> source_url
-> frequency
-> units
-> date_range
-> limitation note
```

## 实验目标

- [ ] 能说明利率数据来源。
- [ ] 能记录 series_id、频率、单位和时间范围。
- [ ] 能说明公开数据的限制。
- [ ] 能区分利率数据说明和投资结论。

## 推荐数据源

第一轮使用 FRED 文档作为公开数据源参照：

```text
source: FRED
docs: https://fred.stlouisfed.org/docs/api/fred/
```

可以先不调用 API，只记录字段口径。后续学习时再选择具体 series。

## 执行步骤

### 步骤 1：选择一个公开利率序列

先从 FRED 文档或 F05 资料索引里选择一个 series，并记录它的 `series_id`、名称和官方说明。第一轮可以只做资料阅读，不强制调用 API。

### 步骤 2：填写数据口径

把频率、单位、样本范围、下载时间和访问限制写入记录模板。不要只写“来自 FRED”。

### 步骤 3：写限制说明

说明这个公开序列是否可能修订、是否代表 point-in-time 数据、是否适合直接和股票日频数据合并。

## 记录模板

```yaml
data_source: FRED
source_url: ""
series_id: ""
series_name: ""
frequency: ""
units: ""
date_range: ""
download_time: ""
license_or_access_note: ""
point_in_time_note: ""
not_investment_advice: true
```

## 字段说明

| 字段 | 为什么重要 |
|---|---|
| series_id | 数据序列唯一标识 |
| frequency | 日频、月频、周频会影响合并 |
| units | 百分比、小数、指数值不能混用 |
| date_range | 样本范围影响指标 |
| download_time | 数据可能更新或修订 |
| point_in_time_note | 是否代表当时可见数据 |

## 常见错误

- 只写“数据来自 FRED”，不写 series_id。
- 不记录单位。
- 不记录下载时间。
- 把公开利率数据图写成投资建议。

## 验收标准

- [ ] 已记录 source_url。
- [ ] 已记录 series_id 或说明暂未选择。
- [ ] 已记录 frequency 和 units。
- [ ] 已写 point_in_time_note。
- [ ] 明确不构成投资建议。
