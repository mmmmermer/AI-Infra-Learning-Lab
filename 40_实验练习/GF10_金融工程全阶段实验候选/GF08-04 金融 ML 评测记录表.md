# GF08-04 金融 ML 评测记录表

## 实验定位

本实验把 F07/GF07 的金融 ML 评测和模型风险记录抽象成 P03 候选 `model_risk_task` 或 `evaluation_task`。它只记录评测与风险，不把模型结果包装成投资能力，也不构成投资建议。

重点不是模型训练，而是记录结构：

```text
label/features/split
-> baseline/model metrics
-> confusion_matrix
-> failure_examples
-> leakage_check
-> model_card
-> review_status
```

## 实验目标

- [ ] 能设计 `model_risk_task` 的字段。
- [ ] 能区分模型指标和人工复核字段。
- [ ] 能记录 baseline、failure_examples 和 leakage_check。
- [ ] 能设计 review_status。
- [ ] 能说明模型评测结果不等于投资能力。

## 执行步骤

### 步骤 1：选择来源实验

优先使用 GF07-04 或 GF07-05 的模型卡、失败样例和压力测试结果。

### 步骤 2：整理输入字段

记录 label 定义、feature 列表、时间切分规则、baseline 规则和模型配置。

### 步骤 3：整理结果字段

分开保存 baseline metrics、model metrics、confusion matrix、failure examples、leakage check 和 limitations。

### 步骤 4：设计人工复核字段

用 `review_status` 和 `review_items` 记录是否有 baseline、是否检查泄漏、是否保留失败样例，以及是否出现不合规投资表达。

## 指标分离

| 类型 | 字段 |
|---|---|
| 模型指标 | accuracy、precision、recall、F1、confusion_matrix |
| 质量/风险指标 | leakage_flags、failure_examples、review_status |
| 性能指标 | task_runtime_ms、queue_wait_ms、data_rows |

## 常见错误

- 把 accuracy 当成投资能力。
- 不记录 baseline。
- 不记录失败样例。
- review_status 只写 reviewed，却没有 review_items。
- 把模型评测记录写成已完成金融 ML 项目。
