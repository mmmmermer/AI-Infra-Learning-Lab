# GF08-02 回测任务进入队列

## 实验定位

本实验把 F06/GF06 的最小回测抽象成 P03 候选 `backtest_task`，重点观察任务生命周期：

```text
pending
-> queued
-> running
-> succeeded / failed
```

本实验不实现真实队列，只设计字段和状态流，服务后续 P03/X02。它不评价策略好坏，不构成投资建议。

## 实验目标

- [ ] 能设计 `backtest_task` 的 input_json。
- [ ] 能设计状态流转。
- [ ] 能记录 queue_wait_ms、task_runtime_ms。
- [ ] 能定义失败类型。
- [ ] 能说明回测任务为什么适合异步执行。

## 执行步骤

### 步骤 1：从 GF06 选择一个回测结果

优先选择 GF06-01 或 GF06-02 的最小回测输出，记录数据范围、信号规则、成本模型和限制说明。

### 步骤 2：把回测输入写成 `input_json`

把 `data_source`、`asset_list`、`price_fields`、`signal_rule`、`execution_assumption` 和 `cost_model` 写入任务草图。

### 步骤 3：设计状态流和指标

按 `pending -> queued -> running -> succeeded/failed` 设计任务生命周期，并记录 `queue_wait_ms`、`task_runtime_ms`、`total_latency_ms`。

### 步骤 4：设计失败类型

至少考虑 `invalid_signal_rule`、`missing_price_data`、`lookahead_risk`、`cost_model_error`、`calculation_error`、`worker_error`。

## 状态流转设计

| 状态 | 含义 | 记录字段 |
|---|---|---|
| pending | 已创建，未入队 | created_at |
| queued | 已入队，等待 worker | queued_at |
| running | worker 正在执行 | started_at |
| succeeded | 成功完成 | finished_at、result_json |
| failed | 执行失败 | finished_at、error_type |
| retrying | 准备重试 | retry_count |
| cancelled | 取消执行 | cancelled_at |

## 常见错误

- 只设计回测结果，不设计状态流。
- `failed` 没有 `error_type`。
- 不记录 queue_wait_ms。
- 不保存 leakage_check。
- 把任务设计写成已经实现的 P03 功能。
