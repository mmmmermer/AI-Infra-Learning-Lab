# GF07-04 模型卡与失败样例记录

## 实验定位

本实验承接 GF07-01 到 GF07-03。它不继续追求更高指标，而是把一个金融 ML 实验整理成可复查的模型卡。

核心目标：

```text
task
-> data
-> label
-> features
-> split
-> baseline
-> model
-> metrics
-> failure_examples
-> leakage_check
-> limitations
```

模型卡不是论文，也不是简历成果。它是学习阶段防止“只记住分数、不记得实验口径”的记录工具。

- [ ] 能写清楚模型任务、数据、label、feature 和 split。

## 实验步骤

### 步骤 1：补齐任务说明

回答：

| 问题 | 记录 |
|---|---|
| 模型预测什么？ |  |
| 预测发生在什么时候？ |  |
| label 来自哪个未来区间？ |  |
| feature 是否只用当时可见数据？ |  |

### 步骤 2：补齐数据和切分

记录：

```text
data_source
date_range
sample_size
train_date_range
test_date_range
shuffle=false
```

### 步骤 3：补齐 baseline 和模型指标

把 GF07-02 的结果放进模型卡：

```text
baseline_metrics
model_metrics
confusion_matrix
```

### 步骤 4：写失败样例说明

失败样例至少记录：

```text
date
true_label
pred_label
feature_snapshot
failure_note
```

失败分析可以这样写：

```text
模型在这些样本上预测错误，可能因为短期收益方向噪声较大、特征过少、样本量过小。该结果不能说明模型有稳定预测能力。
```

### 步骤 5：写限制和边界

必须包含：

```text
学习样本
小样本
没有交易执行
没有真实成本模型
不构成投资建议
不能写成已完成金融 ML 项目成果
```

## 常见错误

- 只写模型分数，不写 label 定义。
- 只写模型，不写 baseline。
- 只写成功样例，不写失败样例。
- 不写 train/test 日期范围。
- 把模型卡包装成已完成金融 ML 项目。

## 验收标准

- [ ] leakage_check 和 limitations 已写。
- [ ] 明确不构成投资建议。
