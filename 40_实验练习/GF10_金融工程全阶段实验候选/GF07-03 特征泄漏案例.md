
## 泄漏案例 1：future_return 作为 feature

危险写法：

```python
bad_features = ["return_lag_1", "future_return"]
```

原因：

```text
future_return 是 label 的来源，预测时不可能提前知道。它进入 feature 后，模型实际上看到了答案。
```

## 泄漏案例 2：rolling 特征没有 shift

危险写法：

```python
df["rolling_vol_3_bad"] = df["return"].rolling(3).std()
```

更合理的教学写法：

```python
df["rolling_vol_3"] = df["return"].rolling(3).std().shift(1)
```

原因：

```text
如果预测发生在 t 时刻，rolling 特征必须只使用 t 之前已经可见的数据。
```

## 泄漏案例 3：全样本标准化

危险写法：

```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
scaled_all = scaler.fit_transform(dataset[features])
```

更合理：

```python
scaler = StandardScaler()
train_scaled = scaler.fit_transform(train[features])
test_scaled = scaler.transform(test[features])
```

原因：

```text
scaler 如果用全样本 fit，就让测试集统计信息提前进入训练流程。
```

## 常见错误

- 看到泄漏模型指标高，就误以为模型更好。
- 不检查 feature 列名。
- 在切分前做全样本标准化。
- rolling 特征忘记 shift。
- 没有把泄漏案例写进失败分析。
