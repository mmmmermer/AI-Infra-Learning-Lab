# E10 推理服务实验索引

> 状态（2026-07-10）：3 个实验页已补齐。E10-01/02 的确定性 Python 参考实现已通过 7 个测试；E10-03 因缺少独立 Linux/WSL vLLM 环境保持 `blocked/unverified`。

## 定位

E10 当前用于训练合成 inference workload、请求长度、TTFT、TPOT 和 tokens/s 口径。E10-01/02
不加载模型、不使用 GPU，也不执行真实 continuous batching 或 KV 分配；只有 E10-03 在独立
Linux/WSL 环境完成后，才能作为 vLLM 最小服务证据。当前没有 Triton 可执行实验。

第一轮不是大模型集群部署实验，而是把 M10 导论教材落到最小可观察实验：

```text
模拟推理 worker
-> 请求长度影响延迟
-> 队列和调度影响 P95/P99
-> 记录 TTFT / TPOT / tokens/s
-> 后续 vLLM 最小服务
```

## 最小练习路线

```text
先模拟 prompt_tokens / output_tokens
-> 计算 prefill / decode / runtime
-> 加 queue_wait
-> 比较 FIFO / SJF / Cost-aware
-> 记录 TTFT / TPOT / tokens/s
-> 再尝试 vLLM 最小服务
```

## 必须记录的指标

| 指标 | 用途 |
|---|---|
| prompt_tokens | 输入长度 |
| output_tokens | 输出长度 |
| queue_wait_ms | 外层排队 |
| ttft_ms | 首 token 等待 |
| tpot_ms | 每 token 耗时 |
| total_latency_ms | 总耗时 |
| tokens_per_second | 吞吐 |
| p95 / p99 | 尾延迟 |
| error_type | timeout、rate_limited、oom 等 |

模拟指标必须带 `simulated` 或 `estimated` 语义；只有真实 streaming 服务记录了 token 到达时间，才能报告 actual TTFT/TPOT。
