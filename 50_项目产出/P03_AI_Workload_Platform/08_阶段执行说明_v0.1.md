## 本阶段目标

用最小工程闭环跑通：

```text
FastAPI
-> POST /tasks（task_type=rag_retrieval）
-> 创建 task
-> 入队
-> worker 执行 mock RAG
-> 回写状态和 metrics
-> GET /tasks/{task_id}
```

v0.1 不追求完整平台，先让项目可以运行、可以记录失败并具备自动测试。可执行参考入口为 `p03_service/`。

## 卡点处理

### 任务一直 queued

优先检查 worker 是否启动、queue 是否一致、worker 是否能连接 store。

### metrics 为空

优先检查时间戳是否在状态流转时写入。

### Docker 下 API 能启动但 worker 不处理任务

优先检查 `.env`、`REDIS_URL`、worker command、compose network。

### RAG 结果为空

v0.1 不先纠结模型效果，先记录 `empty_result`，确保失败路径可解释。
