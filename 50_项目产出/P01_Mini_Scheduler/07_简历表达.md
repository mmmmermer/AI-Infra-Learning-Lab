## 项目一句话

Mini Scheduler 是一个面向 AI / RAG / Agent workload 的任务调度模拟器，用纯 Python 实现任务建模、调度策略、指标统计和实验对比，用于分析不同调度策略、成本权重、等待保护和 worker 数量对平均等待时间、P95/P99 尾延迟和资源利用率的影响。

## 简历 Bullet 版本

### 偏工程实现

- 实现 Mini Scheduler 调度模拟器，支持 FIFO、Priority、SJF、Cost-aware、aging / 最大等待保护等策略，完成单 worker 与多 worker 任务执行模拟，并统计平均等待时间、P95/P99、最大队列长度和 worker 利用率。

### 偏实验分析

- 设计高峰负载、权重敏感、多 worker 扩容等实验，比较不同调度策略在平均等待时间、P95/P99 尾延迟和资源利用率上的取舍；实验显示 worker 从 1 增至 8 可将 FIFO P95 从 97.70 降至 3.50，但利用率从 0.97 降至 0.30。

### 偏 AI Infra / 调度方向

- 构建面向 AI workload 的成本感知调度实验框架，将 `estimated_duration`、`token_count`、`priority` 纳入调度分数，并通过 task_type 分组分析发现不同权重会系统性影响任务等待时间分布。

### 偏科研训练

- 围绕 AI workload 调度尾延迟问题，建立从任务建模、策略对比、负载生成、指标统计到实验结论的可复现实验链路，分析平均等待时间、P95/P99、最大等待、资源利用率之间的权衡。

## 面试讲法

### 1. 项目背景

我做这个项目是为了理解 AI Infra 里任务调度的基本问题：当 RAG、Agent、embedding、长上下文等任务同时到来时，系统应该先执行谁，怎么衡量策略是否更好。

所以我没有一开始接 Kubernetes 或真实 GPU，而是先用纯 Python 做了一个最小调度器，把任务、worker、队列、策略和指标拆清楚。

### 2. 做了什么

项目里我实现了：

- Task / Worker 数据模型
- FIFO / Priority / SJF / Cost-aware 策略
- Aging / 最大等待保护
- 单 worker 和多 worker 模拟
- 高峰负载任务流
- Cost-aware 权重敏感任务流
- 平均等待时间、P95/P99、最大队列长度、worker 利用率
- 按 task_type 分组的等待时间分析

### 3. 关键实验结论

第一，高峰负载下平均值不够用。SJF / Cost-aware 可以降低平均等待时间，但可能让部分任务的 P99 更差。

第二，Cost-aware 权重会改变谁被牺牲。在当前固定 fixture 中，token_dominant 下高 token 短任务 `short_high_token` 的平均等待为 88.40；priority_dominant 下 `urgent_medium` 的平均等待为 6.20。这里描述的是确定性样例中的数值差异，不是统计显著性结论。

第三，aging / 最大等待保护能降低极端尾延迟。strong_aging 将 token_dominant 下的 P99 从 110.80 降到 104.40，并把 `short_high_token` 的平均等待从 88.40 降到 52.40，但会让一些原本很快的任务等待增加。

第四，扩容能降尾延迟，但会牺牲利用率。FIFO 下 worker 从 1 增加到 8，P95 从 97.70 降到 3.50，但利用率从 0.97 降到 0.30。

### 4. 我从项目里学到什么

这个项目让我理解到调度不是简单排序，而是在平均效率、尾延迟、公平性和资源利用率之间做权衡。一个策略整体平均值更好，不代表每类任务都受益；所以必须做分组分析和尾部指标分析。

## 可量化结果

| 实验 | 结果 |
|---|---|
| 高峰负载 | SJF 平均等待时间 26.04，低于 FIFO 的 48.71，但 P99 更高 |
| Cost-aware 权重 | token_dominant 的 P99 为 110.80，高于 duration_dominant 的 104.40 |
| Aging 保护 | strong_aging 将 P99 从 110.80 降至 104.40 |
| 分组分析 | priority_dominant 将 `urgent_medium` 平均等待降至 6.20，但牺牲低优先级任务 |
| Worker 扩容 | FIFO P95 从 97.70 降至 3.50，但利用率从 0.97 降至 0.30 |
