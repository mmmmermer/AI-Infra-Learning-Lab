# P01 需求与任务拆解

## 项目一句话

P01 Mini Scheduler 是一个用纯 Python 实现的最小任务调度器。它不追求一开始就像生产系统，而是先把 AI/RAG/Agent 请求背后的通用问题拆出来：任务来了以后，谁先执行、分配给哪个 worker、等待多久、整体效率如何。

## 为什么第一阶段只做纯 Python

第一阶段先不接 FastAPI、数据库、Docker、RAG 和 Kubernetes。原因很简单：如果任务、worker、调度策略和指标都还没想清楚，提前上框架只会让问题变得更难排查。

纯 Python 阶段要解决的是系统内核问题：

- 如何表示一个任务
- 如何表示一个 worker
- 如何按不同策略给任务排序
- 如何模拟任务被执行
- 如何计算等待时间、完成时间和 worker 利用率
- 如何用 pytest 保证规则没有写错

## 第一阶段范围：v0.1

v0.1 的目标不是做完整平台，而是做出一个可以运行、可以测试、可以解释结果的调度器核心。

### 必须实现

- Task 数据结构
- Worker 数据结构
- FIFO 排序
- Priority 排序
- SJF 排序
- 单 worker 或多 worker 的最小模拟
- 等待时间计算
- 完成时间计算
- worker 利用率的基本计算
- 至少 5 个 pytest
- 一个命令行 demo

### 暂时不实现

- FastAPI 接口
- 数据库存储
- 异步队列
- Docker 部署
- 真实 RAG 请求
- 真实 GPU worker
- Kubernetes 调度

## 核心概念

### Task
Task 是被调度的对象。后续 RAG 请求、Agent 工具调用、推理请求，都可以抽象成 Task。

第一版字段：

| 字段 | 类型 | 含义 | 示例 |
|---|---|---|---|
| `id` | `str` | 任务唯一标识 | `task-001` |
| `task_type` | `str` | 任务类型 | `rag_query` |
| `priority` | `int` | 优先级，数字越小越优先 | `1` |
| `estimated_duration` | `float` | 预计执行耗时 | `3.5` |
| `submit_time` | `float` | 提交时间 | `0.0` |
| `start_time` | `float / None` | 开始执行时间 | `2.0` |
| `finish_time` | `float / None` | 完成时间 | `5.5` |
| `status` | `str` | 任务状态 | `pending` |

### Worker
Worker 是执行任务的资源。现在可以理解为一个 Python 模拟对象，后续可以映射到 CPU worker、RAG worker、推理 worker 或 GPU 实例。

第一版字段：

| 字段 | 类型 | 含义 | 示例 |
|---|---|---|---|
| `id` | `str` | worker 唯一标识 | `worker-1` |
| `available_at` | `float` | worker 什么时候空闲 | `0.0` |
| `current_task_id` | `str / None` | 当前任务 | `task-001` |
| `total_busy_time` | `float` | 累计忙碌时间 | `12.0` |

### Scheduler

Scheduler 不是一开始就要写成复杂类。第一阶段可以先把调度策略做成函数，再逐步合并成调度器。

核心问题：

- FIFO：谁先来，谁先执行
- Priority：谁更重要，谁先执行
- SJF：谁预计耗时短，谁先执行

## 输入样例

第一阶段可以先把任务写在 Python 代码里，后续再改成读取 JSON。

```python
tasks = [
    {"id": "task-001", "task_type": "rag_query", "priority": 2, "estimated_duration": 5.0, "submit_time": 0.0},
    {"id": "task-002", "task_type": "agent_tool", "priority": 1, "estimated_duration": 2.0, "submit_time": 1.0},
    {"id": "task-003", "task_type": "embedding", "priority": 3, "estimated_duration": 1.0, "submit_time": 2.0},
]
```

## 输出样例

demo 运行后至少要能看出每个任务的排序结果和指标。

```text
strategy=fifo
order=task-001, task-002, task-003
average_wait_time=3.00
average_completion_time=5.67
worker_utilization=1.00
```
